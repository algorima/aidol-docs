# AIdol Docs 문서 리뷰 가이드

이 레포지토리는 코드가 아닌 기획/설계 문서를 관리합니다.
리뷰 시 문서의 명확성, 일관성, 실행 가능성을 중심으로 검토하세요.

## 필수 체크리스트

### 1. Atomic PR

- 1 PR이 1개 주제만 다루고 있는지 확인. 여러 주제가 섞여 있으면 분리 요청
- 5분 안에 읽을 수 있는 분량인지 확인
- 변경 배경/이유가 PR 설명에 명시되어 있는지 확인

### 2. 문서 일관성

- 같은 기능이 다른 문서에서 다르게 설명되어 있지 않은지 확인
  - 기능 명세서 ↔ ERD ↔ API 명세서 간 일관성
  - Sprint 간 같은 기능의 설명 일관성
- 용어가 문서 전체에서 통일되어 있는지 확인

### 3. 기능 명세서 품질

- 기능 설명이 개발자가 구현할 수 있을 정도로 구체적인지 확인
- 데이터 흐름이 명확한지 확인 (데이터의 출처, 필터링 조건, 표시 규칙)
- 모호한 표현("적절히", "필요에 따라" 등)이 있으면 구체화 요청
- UI 동작만 기술하고 데이터 흐름을 생략한 경우 반드시 지적

### 4. 포맷

- 번호 체계가 계층적으로 일관되게 사용되었는지 확인 (1. 1.1 1.1.1)
- 기능 명세서의 표 형식이 기존 문서와 동일한지 확인
- 이모지가 문서 내에 사용되지 않았는지 확인

### 5. 충돌 감지

- 기존 문서의 내용과 충돌하는 부분이 있으면 반드시 지적
- 충돌이 있을 경우, PR 설명에 해결 방안이 제시되었는지 확인
