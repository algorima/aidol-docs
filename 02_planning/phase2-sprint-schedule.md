## 3. 듀얼 트랙 구조

```
Discovery (1 Sprint 선행)           Delivery (현재 Sprint 구현)

Sprint 4 디자인 (2/23~2/27)    ->   Sprint 4 개발 (2/26~3/4)
Sprint 5 디자인 (2/28~3/4)     ->   Sprint 5 개발 (3/5~3/11)
Sprint 6 설계 (3/5~3/7)        ->   Sprint 6 작업 (3/12~3/18)
```

Discovery가 와이어프레임/Figma를 완성해야 Delivery가 개발을 시작할 수 있다. Discovery는 항상 다음 Sprint를 미리 디자인한다.

## 11. 커뮤니케이션 규칙

### 11.1 PR 리뷰

- 문서 PR: 리뷰 요청 후 **24시간 이내** 리뷰 완료
- 코드 PR: 리뷰 요청 후 **24시간 이내** 리뷰 완료
- 초과 시 리뷰어 변경 또는 소연이 직접 머지 판단

### 11.2 차단 요인 (Blocker) 처리

- 차단 요인 발생 시 **즉시** Slack 공유
- Sprint Review까지 기다리지 않기
- 24시간 내 해결 안 되면 소연에게 에스컬레이션

### 11.3 문서 작업

- 기능 명세서, ERD, API 명세서는 모두 aidol-docs PR로 관리
- ERD/API 변경은 반드시 영욱 리뷰 필수
- 기능 명세서는 영욱 리뷰 후 관련 개발자 공유
