- 목적: 개발 스프린트와 사용자 리서치가 병렬로 진행되는 전체 일정 파악
- 상태: draft
- 작성일: 2026-03-05

## 1. 병렬 트랙 개요

| 트랙 | 담당 | 목적 |
|------|------|------|
| 개발 트랙 | 영욱, 시영, 제형, 지영, 은재 | 기능 구현 및 배포 |
| 리서치 트랙 | 수지 | 사용자 인터뷰, UT 설계/진행, 데이터 분석 |

| AI/LLM 트랙 | 채현 | 프롬프트 설계, LLM 파이프라인 |
| 비즈니스 트랙 | 소연 | IR, 제품 방향, 의사결정 |

## 3. 수지 (PM/리서치) 상세 일정

### 3.1 스프린트당 리서치 사이클

각 스프린트마다 아래 사이클을 반복:

| 단계 | 기간 | 산출물 |
|------|------|--------|
| 1. UT 설계 | 스프린트 시작 ~3일 | UT 가이드 (가설, 인터뷰 질문, 세그먼트) |
| 2. 리크루팅 | 1~2일 (설계와 병렬) | 피실험자 확보 (세그먼트별 3~5명) |
| 3. UT 진행 | 3~5일 | 인터뷰 녹음, 녹취록 |
| 4. 분석 정리 | 2~3일 | 가설 검증 보고서 (ut-transcript-analyzer 에이전트 활용) |
| 5. 팀 공유 | 1일 | Slack 공유, 다음 스프린트 반영 사항 제안 |

### 3.2 Phase 1 리서치 실적

| 스프린트 | 피실험자 | 핵심 발견 | PRD 반영 |
|---------|---------|----------|---------|
| Sprint 1 | U1~U7 (7명) | 캐스팅은 선택 행위로 인지되나 애착 미형성, 경쟁/희소성 유효 | 프리셋 확대, 데뷔 서사 필요성 |
| Sprint 1-2 | U8~U10 (3명) | 캐스팅 몰입 O, 육성 요소 없이는 결제 전환 X | 다마고치형 육성 요소 Phase 2 반영 |
| Sprint 2 | U8 (1명) | 관계성 설정 기대감 O, 결과물 반영 확인 욕구 | 하이라이트 콘텐츠 개인화 |

### 3.3 Phase 2 리서치 계획

| 스프린트 | 검증 가설 | 목표 인원 | 세그먼트 |
|---------|----------|----------|---------|
| Sprint 4-5 | PMF 신호, NPS, 유료 전환 의향, 육성 욕구 | 10명+ | A(팬), B(창작), C(일반) |
| Sprint 6 | 리텐션, 재방문 동기, 콘텐츠 소비 패턴 | 5~8명 | 기존 유저 재인터뷰 |

## 4. 리서치 → PRD/IR/사업계획서 반영 프로세스

```
UT 녹취록 (raw)
    |
    v
ut-transcript-analyzer 에이전트로 분석
    |
    +--→ 가설 검증 보고서
    |        |
    |        v
    |    스프린트 회고 (팀 공유)
    |        |
    |        v
    |    PRD/기능명세서 업데이트
    |
    +--→ IR 데이터 시트
    |        |
    |        v
    |    IR Deck 업데이트 (06_ir-master/)
    |    - 03-traction.md: PMF 지표, UT 누적 인원
    |    - 01-problem.md: 사용자 Pain Point 발화
    |    - 02-solution.md: Aha Moment 발화
    |    - 05-business-model.md: WTP/결제 의향 데이터
    |
    +--→ 사업계획서 근거
             |
             v
         지원사업 작성 시 활용
         - 고객 검증 결과 (인원, 세그먼트, 지표)
         - 제품 차별성 근거 (경쟁사 대비 발화)
         - 시장 반응 데이터 (전환율, NPS)
```

### 4.1 반영 기준

| 분석 결과 | 액션 |
|----------|------|
| 가설 검증됨 | 다음 스프린트에서 해당 기능 확장 |
| 부분 검증 | UX 개선 또는 조건부 진행, PRD에 근거 기록 |
| 기각 | 기능 축소/제거, PRD에 기각 사유 기록 |
| 미검증 | 다음 UT에서 재검증 설계 |

### 4.2 산출물 위치

| 산출물 | 경로 |
|--------|------|
