## 1. Phase 2 목표

Phase 2의 목표는 AI 아이돌 생성 경험에 "실재감(Reality)"을 부여하는 것이다.

### 1.1 현재 UX 흐름

```
아이돌 생성 → 완료 화면 → 종료
```

사용자는 아이돌을 생성하지만 그 이후 아이돌이 활동하는 세계를 경험하지 못한 채 경험이 종료된다.

### 1.2 Phase 2 목표 흐름

```
아이돌 생성 → 내 그룹 → 콘텐츠 → 탐색 → 상호작용
```

이를 통해 다음 두 가지 경험을 구축한다.

1. 아이돌이 실제 활동하는 것처럼 보이는 경험
2. 다른 유저와 연결된 아이돌 세계 경험

## 2. 문제 정의

### 2.1 사용자 경험 문제

현재 서비스는 아이돌 **생성 경험**까지만 제공하며 아이돌 **활동 경험**은 제공하지 못하고 있다.

**근거: UT 관찰 (Sprint 1-2, U8 김현조)**

사용자는 캐스팅 과정에서 높은 몰입을 보였으나, 생성 이후 다음 행동이 명확하지 않았다.

> "뭔가 오랜만에 몰입해 본 것 같아요."
> — U8 김현조 (Sprint 1-2 UT, 캐스팅 완료 후 전반 소감)

> "캐스팅을 하고 물론 이제 계속 메일로 어떤 스토리가 있는지 보여지겠지만 뭔가 제가 직접 트레이닝하고 좀 키우는 느낌은 조금 부족하다는 생각은 있었어요."
> — U8 김현조 (Sprint 1-2 UT, 서비스 평가 발화)

즉 사용자는 생성 이후 **육성 또는 다음 활동으로 이어지는 경험**을 기대하지만 현재 서비스에는 해당 흐름이 없다.

### 2.2 멘토 피드백

**D.Camp Office Hours — 성낙환 팀장**

> "지금 느낌은 프린세스 메이커 같은 게임 느낌이다."

> "가상의 아이돌이지만 실제 존재하는 것처럼 보이게 하는 장치가 필요하다."

> "채팅 외에도 공연이나 라이브 같은 다른 인터랙션 채널이 있어야 한다."

**핵심 인사이트**: 사용자가 원하는 것은 아이돌 **생성** 경험이 아니라 아이돌이 **실제 활동하는 것처럼 보이는** 경험이다.

### 2.3 사용자 테스트 주요 인사이트

#### 2.3.1 아이돌 몰입 포인트

**관찰 데이터 (Sprint 1 UT)**

- U6 김민지: 직접 생성한 멤버의 얼굴 퀄리티에 불만을 표현하며 프리셋 멤버 위주로 선택. 얼굴 확인 행동이 반복적으로 관찰됨 (행동 관찰 기록)
- U4 배나영: 완성 그룹 카드를 보고 캡처해서 보여주고 싶다고 발화. 포토카드처럼 소장하고 싶다는 욕구를 함께 표현함

> "캡쳐해서 보여주고 싶었다."
> — U4 배나영 (Sprint 1 UT, 그룹 완성 카드 확인 시)

**해석**: 사용자는 캐릭터 디자인, 얼굴, 멤버 설정 등 아이돌 자체에는 높은 몰입을 보인다. 그러나 아이돌 생성 이후 캐릭터와 상호작용하거나 활동을 볼 수 있는 경험이 부족하다.

#### 2.3.2 탐색 UX 문제

현재 캐스팅 리스트에 계약 완료 연습생이 계속 노출되는 UX 문제가 관찰됐다.

**관련 관찰 (Sprint 1 UT 및 2/22 버그 리포트)**

- U5 차유경: 캐스팅 리스트에서 블러 처리된(계약 완료) 연습생이 상단에 노출되는 문제를 지적 (행동 관찰 기록)
- U7 최지윤: 선택 가능한 연습생 풀이 제한적이어서 "퍼즐 끼워 맞춘 느낌"이라고 발화

> "퍼즐 끼워 맞춘 느낌"
> — U7 최지윤 (Sprint 1 UT, 캐스팅 과정 평가)

- 2/22 버그 리포트: 계약 완료가 아니어야 하는 멤버가 계약 완료로 표시되는 버그 확인

> "왜 계약 완료야? 계약 완료되면 안 되는데 원래 저렇게"
> — 2/22 UT 참여자 (사용자 테스트 버그 리포트)

**문제 요약**: 캐스팅 상태 인지 실패로 인해 탐색 혼란과 탐색 비용 증가가 발생하는 UX 문제.

#### 2.3.3 상호작용 필요성

**관찰 데이터 (Sprint 1 UT)**

- U7 최지윤: 즉각적인 피드백 또는 리워드가 없는 구조에서 이탈 경향 관찰

> "이게 키우는 게임이면 이메일 작성 했을 것"
> — U7 최지윤 (Sprint 1 UT, 전환 조건 제안)

> "매일보다는 다음날처럼 더 빠른 리워드가 좋다"
> — U7 최지윤 (Sprint 1 UT, 리텐션 조건 제안)

- U1/U2 찬찬: 다른 유저와의 비교/경쟁 심리가 강하게 작동하는 것이 관찰됨

> "다른 유저에게 가면 아쉽다."
> — U1/U2 찬찬 (Sprint 1 UT, 캐스팅 완료 후)

**인사이트**: 사용자는 다른 유저 존재, 경쟁 요소, 즉각적 반응 인터페이스를 통해 서비스에 더 오래 머무르는 경향을 보인다.

## 3. Phase 2 제품 전략

Phase 2에서는 다음 3가지 전략을 기반으로 제품을 개선한다.

### 3.1 Strategy 1 — 생성 → 콘텐츠 경험 연결

아이돌 생성 이후 사용자가 콘텐츠 공간으로 자연스럽게 이동하도록 UX를 재설계한다.

| 현재 | 변경 |
|------|------|
| 완료 → 종료 | 완료 → 내 그룹 페이지 → 콘텐츠 피드 |

**근거**: U8 김현조 UT에서 캐스팅 완료 후 다음 행동 단서 부재가 확인됨. U1/U2, U7 UT에서 "데뷔를 해야 의미 있다", "연습생 갖고는 할 게 없어" 발화가 관찰되어 생성 이후 활동 경험으로의 연결이 핵심 과제임을 확인.

### 3.2 Strategy 2 — 아이돌 몰입 강화

사용자가 만든 아이돌을 더 크게 보고, 더 자세히 탐색하고, 더 오래 머물 수 있도록 아이돌 정보 UX를 확장한다.

**근거**: U4 배나영의 소장/캡처 욕구, U6 김민지의 얼굴 퀄리티 확인 행동 반복 관찰.

### 3.3 Strategy 3 — 아이돌 세계 구축

다른 유저가 만든 그룹을 탐색하고 콘텐츠를 소비할 수 있는 구조를 만든다.

이는 서비스 경험을 "나 혼자 만드는 인형놀이"에서 "아이돌 세계에서 활동하는 경험"으로 확장하는 핵심 요소다.

**근거**: U1/U2의 희소성/경쟁 심리 관찰, D.Camp 멘토 피드백.

## 8. Phase 2 Experiment — IP 육성 니즈 검증

Phase 2에서는 AI 아이돌 IP 육성 모델의 사용자 니즈를 검증하는 UX 실험을 함께 진행한다.

이 실험은 기능 출시를 목적으로 하는 것이 아니라 서비스 방향을 검증하기 위한 Discovery 단계 실험이다.

### 8.1 실험 배경

Phase 1 사용자 테스트에서 다음 패턴이 반복적으로 관찰됐다.

사용자는 아이돌 캐릭터 자체에 높은 관심을 보였으며, "내가 만든 아이돌"이라는 개입 요소에서 더 높은 몰입을 보였다.

**근거 발화**

> "캡쳐해서 보여주고 싶었다."
> — U4 배나영 (Sprint 1 UT, 그룹 완성 카드 확인 시)

> "내가 키우는 내 아이라는 생각이 들었다."
> — U6 김민지 (Sprint 1 UT, 캐스팅 평가)

> "다마고치면은 제가 맨날 밥 주고 좀 케어해 주면서 신경을 쓰게 되는데 현재까지는 그런 요소는 살짝 조금 없었던 것 같아가지고 그런 부분이 좀 있으면은 이제 뺏기면 안 되겠다 생각이 좀 더 들 것 같은"
> — U8 김현조 (Sprint 1-2 UT, 육성 요소 부재 관련)

> "이게 키우는 게임이면 이메일 작성 했을 것"
> — U7 최지윤 (Sprint 1 UT, 전환 조건 제안)

이러한 행동 패턴은 다음 가설을 생성한다.

### 8.2 실험 가설

아이돌과의 단순 채팅 경험보다 아이돌 IP에 **개입**하는 경험이 더 높은 몰입과 지불 의향을 만든다.

즉, 다음 두 방향 중 어느 쪽이 더 강한 제품 경험인지 검증한다.

- AI 채팅 서비스
- AI 아이돌 육성 서비스

### 8.3 실험 방식 (Fake Door Test)

Phase 2에서는 실제 육성 기능을 개발하지 않고 Fake Door 방식으로 사용자 반응을 측정한다.

채팅 인터페이스에서 다음 CTA를 노출한다.

```
다음 활동 컨셉을 정해보시겠어요?

[컨셉 정하기]  [그냥 대화하기]
```

사용자가 컨셉 선택 버튼을 누르면 다음 중 하나를 노출한다.

- **알림 등록형**: "프로듀서 기능은 준비 중입니다. 출시 알림을 받으시겠어요?"
- **지불 의향형**: "프로듀서 기능 / 월 3,000원" (Paywall)

### 8.4 측정 지표

| 지표 | 내용 |
|------|------|
| CTA 클릭률 | CTA 노출 대비 클릭 비율 |
| 알림 등록률 | CTA 클릭 후 알림 등록 전환율 |
| 지불 의향 | Paywall 도달 후 반응 |
| 인식 변화 | 사용자 설문: 키운 느낌, 애착도, 재방문 의향 |
| 행동 차이 | 채팅 경험 유저 vs IP 개입 CTA 클릭 유저 행동 비교 |

### 8.5 실험 의도

이 실험의 목적은 다음 질문에 답하는 것이다.

AI 아이돌 서비스의 핵심 경험은 **채팅인가, IP 육성인가?**

실험 결과에 따라 Phase 3에서 다음 방향을 결정한다.

- AI 채팅 중심 서비스
- AI 아이돌 IP 육성 플랫폼

### 8.6 실험 위치

이 실험은 Sprint 5 또는 Sprint 6에서 진행한다.

**이유**: Sprint 4에서는 아이돌 실재감 UX 기반 구축이 우선이기 때문이다.

## 10. Phase 2 UX Loop

Phase 2 이후 서비스 핵심 UX 루프:

```
아이돌 생성
→ 내 그룹
→ 콘텐츠 소비
→ 다른 그룹 탐색
→ 좋아요 / 상호작용
→ 다시 탐색
```

이 루프를 통해 서비스는 **아이돌 생성 서비스**에서 **아이돌 세계 플랫폼**으로 확장된다.
