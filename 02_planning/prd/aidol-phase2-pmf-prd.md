## 1. 개요

### 1.1 Phase 2 목적

Phase 2는 Phase 1 MVP 운영 데이터와 사용자 테스트(UT)를 통해 **시드 라운드 IR에서 투자자에게 보여줄 PMF 신호**를 확보하는 단계다. 본 문서는 전체 플랫폼 전략의 'Phase 2' 중 첫 단계인 **PMF 검증(Sprint 4-6)**에 대한 요구사항을 정의하며, platform-strategy-prd에서 정의한 'Phase 2'(6-12개월 UGC 플랫폼 전환) 전체를 다루지는 않는다.

**서비스 정체성 전환:**

Phase 1에서는 "AI 아이돌을 만들고 대화하는 경험"을 검증했다. Phase 1 UT에서 핵심 발견은 **"내 새끼 심리"**다. 사용자가 캐스팅 과정에서 전략적 고민을 하면서 소유감이 형성되고, 이것이 공유 의향의 핵심 트리거로 작동했다 ("내가 키우는 그룹이니까 마케팅 해주고 싶다" - U3). 그러나 Phase 1 서비스에는 "키울 수 있다"는 느낌이 전혀 없었다.

Phase 2에서는 서비스 정체성을 **"네가 캐스팅해서 키우는 AI 아이돌 그룹"**으로 명확히 전환한다. 풀 육성 시스템(레벨업, 친밀도 등)은 Phase 5에서 구축하되, Phase 2에서는 UX 라이팅, 최소 육성 체험(컨셉 디벨롭), 설문을 통해 이 방향의 사용자 반응을 검증한다.

투자자가 시드 단계에서 확인하고 싶은 것:

1. **사용자가 이 제품을 진짜 좋아하는가?** (만족도, 재사용 의향)
2. **다른 사람에게 퍼질 수 있는가?** (공유 의향, 바이럴 가능성)
3. **돈을 낼 의향이 있는가?** (유료 전환 의향)
4. **시장 수요가 실제로 있는가?** (자연 유입, 트래픽)

Phase 2는 이 4가지 질문에 대한 정량 데이터를 UT + 인앱 설문 + 행동 분석으로 확보한다.

### 1.2 Phase 1과의 관계 및 인증 전환

Phase 1 (Sprint 1-3)은 "나만의 아이돌을 만들고 대화하는 경험이 사용자를 만족시키는가"를 검증하는 단계다.

**Phase 1 인증 구조 (anonymous):**

Phase 1 MVP에는 로그인/회원가입 기능이 없다. 소유권은 `anonymous_id` (UUID)를 localStorage에 저장하여 관리한다. 이로 인해 가입자 수 정확 측정, D7 Retention 추적, 멀티 디바이스 동기화가 불가능하다.

**Phase 2에서 로그인/회원가입 도입:**

Phase 2에서는 로그인/회원가입 기능을 도입한다. 핵심 이유:

1. **웹-모바일 동기화**: Phase 2에서 모바일 앱 출시가 예정되어 있으며, 웹에서 익명으로 만든 아이돌 데이터를 모바일에서 이어서 사용하려면 계정 기반 인증이 필수다 (PR #13 ywkim 피드백).
2. **정확한 리텐션 측정**: 계정 기반으로 D7 Retention을 신뢰성 있게 추적할 수 있다.
3. **팔로우 시스템**: 다른 그룹 탐색/팔로우 기능에 userId가 필요하다.
4. **데이터 마이그레이션**: Phase 1에서 생성한 anonymous 데이터를 계정으로 전환하는 마이그레이션 API가 필요하다.

이로 인해 Phase 1에서의 측정 제약(anonymous_id 기반 추적 한계)이 Phase 2에서 해소된다.

**Phase 1 Go 기준 (Phase 2 진입 조건):**

| 항목 | Go 기준 | 측정 방법 | 비고 |
|------|---------|----------|------|
| 고유 방문자 수 | 1,000명 이상 | PostHog distinct_id 기준 | 로그인 없이 측정 가능한 트래픽 규모 |
| 그룹 생성 완료율 | 70% 이상 | PostHog 퍼널: 캐스팅 시작 -> 그룹 완성 | 핵심 경험의 첫 관문 |
| 첫 대화 시작률 | 80% 이상 | PostHog 이벤트: 그룹 완성 -> 첫 채팅 메시지 발송 | 두 번째 핵심 기능 도달률 |

**참고:** 40-week-sprint-plan에서는 "가입자 1,000명+"과 "D7 Retention 15%+"을 Phase 1 Go 기준으로 명시하고 있으나, Phase 1은 로그인이 없으므로 위 지표로 대체하여 판단한다. D7 Retention은 Phase 2에서 로그인 도입 후 정확하게 측정한다.

## 2. 목표와 가설

### 2.1 Phase 2 핵심 질문

Phase 2에서 답해야 할 질문은 **"시드 IR에서 투자자를 설득할 수 있는 PMF 데이터가 있는가?"**이다.

이를 위해 Phase 1에서 시작한 검증을 이어서, 아래 4가지 가설을 정량적으로 확인한다.

### 2.2 가설과 검증 지표

**가설 1: 사용자가 이 제품을 필수적으로 느낀다**

Phase 1에서 "만드는 재미"와 "대화하는 재미"를 확인했다면, Phase 2에서는 그 경험이 일회성이 아닌 **없으면 아쉬운 수준**인지 확인한다.

| 검증 지표 | 목표 | 측정 방법 |
|----------|------|----------|
| "매우 실망" 응답 비율 | 40% 이상 | UT 설문 (Sean Ellis 질문) |
| 재사용 의향 | 4.0/5.0 이상 | UT 설문: "이 서비스를 계속 사용하고 싶다" |
| 세션당 채팅 메시지 수 | 10개 이상 | PostHog 이벤트 추적 |

**투자자에게 의미:** "매우 실망" 40% 이상은 Sean Ellis가 수백 개 스타트업을 분석해 도출한 PMF 기준이다. Superhuman이 이 방법으로 PMF를 찾아 유니콘이 되었다. 시드 단계에서 이 수치를 보여줄 수 있다면, 제품의 핵심 가치가 검증되었다는 강력한 근거다.

**가설 2: 다른 사람의 AI 아이돌 콘텐츠에 관심이 있고, 보면 직접 만들고 싶어진다**

이 가설은 Phase 1 platform-strategy-prd에서 정의한 Chicken-and-Egg 검증과 직결된다. 향후 UGC 플랫폼으로의 확장 가능성을 증명한다.

| 검증 지표 | 목표 | 측정 방법 | 조작적 정의 |
|----------|------|----------|------------|
| "다른 그룹" 콘텐츠 클릭률 | 40% 이상 | PostHog 이벤트 | 홈 화면 노출된 "다른 그룹" 카드 중, 최소 1개를 클릭(탭)한 세션 비율. 분자: "다른 그룹" 카드를 1회 이상 클릭한 세션 수. 분모: 홈 화면에 "다른 그룹" 카드가 1개 이상 노출된 전체 세션 수 |
| 세션당 콘텐츠 탐색 개수 | 3개 이상 | PostHog 이벤트 | 한 세션 내에서 클릭하여 열어본 "다른 그룹" 콘텐츠(채팅 UI 스토리)의 고유 개수. 같은 콘텐츠를 여러 번 열어도 1개로 집계. 세션 = PostHog 기본 세션 (30분 비활동 시 새 세션) |
| 콘텐츠 완료율 | 70% 이상 | PostHog 이벤트 | 열어본 "다른 그룹" 채팅 UI 콘텐츠 중, 마지막 메시지까지 스크롤한 비율. 분자: `content_completed` 이벤트 발생 수. 분모: `content_opened` 이벤트 발생 수 |
| 소비 -> 생성 전환율 | 30% 이상 | PostHog 퍼널 | "다른 그룹" 콘텐츠를 1개 이상 열어본 방문자(distinct_id) 중, 이후 자신의 그룹 생성을 시작한 비율. 분자: 콘텐츠 열람 후 `group_creation_started` 이벤트가 발생한 distinct_id 수. 분모: `content_opened` 이벤트가 1회 이상 발생한 전체 distinct_id 수. 기간 제한 없음 (같은 세션이 아니어도 됨) |

**투자자에게 의미:** 전환율 30%+는 Chicken-and-Egg 문제가 해결 가능하다는 증거다. "남의 AI 아이돌 보고 -> 나도 만들고 싶어짐"이 작동하면, UGC 플랫폼으로의 확장이 현실적이다.

**가설 3: 사용자가 자발적으로 공유하고 추천한다**

시드 IR에서 바이럴 가능성을 증명하는 핵심 데이터다. 이미 TikTok 마케팅 $0로 500K 조회 + 500명 가입을 확인했으므로, 제품 내에서도 공유 행동이 발생하는지 확인한다.

| 검증 지표 | 목표 | 측정 방법 | 조작적 정의 |
|----------|------|----------|------------|
| 공유 의향 점수 | 4.0/5.0 이상 | UT 설문: "내 AI 아이돌 콘텐츠를 다른 사람에게 보여주고 싶다" | 5점 리커트 척도 평균 |
| 공유 버튼 클릭률 | 20% 이상 | PostHog 이벤트 | 그룹 프로필 페이지를 1회 이상 방문한 distinct_id 중, 공유 버튼을 클릭한 비율 |
| NPS | 30 이상 | UT 설문: "이 서비스를 친구에게 추천할 가능성은?" (0-10점) | (추천자 비율 %) - (비추천자 비율 %). 추천자(Promoter): 9-10점, 중립(Passive): 7-8점, 비추천자(Detractor): 0-6점 |

**바이럴 보상 루프 (Phase 2 검증 대상):**

Phase 1 UT에서 "내가 키우는 그룹이니까 마케팅 해주고 싶다"는 자발적 공유 동기가 확인되었다. Phase 2에서는 이 동기를 제품 내 공유 메커니즘으로 구체화한다:

```
사용자가 아이돌 그룹 URL을 SNS에 공유
  -> 외부 유입자가 좋아요/반응
  -> 내 아이돌의 인지도 지표 상승 (향후 Phase 5에서 성장 시스템과 연동 예정)
  -> 사용자가 더 공유하고 싶어짐 (선순환)
```

Phase 2에서는 이 루프의 첫 단계인 **공유 행동 발생 여부**를 검증한다. UX 라이팅으로 "네 아이돌을 세상에 알려봐"라는 가이드를 제공하고, 공유 버튼 클릭률과 실제 공유 완료율을 측정한다.

**투자자에게 의미:** NPS 30+는 B2C 서비스 기준 우수하다. 공유 의향 + 실제 공유 행동이 함께 높으면, 유료 마케팅 없이도 자연 성장 가능성이 있다.

**가설 4: 유료 전환 의향이 있다**

Phase 3(유료 전환 검증)으로 넘어가기 전, 설문을 통해 유료 의향을 사전 확인한다.

| 검증 지표 | 목표 | 측정 방법 |
|----------|------|----------|
| 유료 전환 의향 | 10% 이상 | UT 설문: "월 3,000~4,900원에 프리미엄 기능을 사용하시겠습니까?" |
| 기대 가격대 | 3,000~4,900원/월 | UT 설문: "적정 가격은?" (자유 입력) |

**투자자에게 의미:** 공모전에서 50% 유료 전환율(12명 중 6명, 10-12만원 티켓)과 Phase 1 UT에서 3,000원/월 자발적 가격 제안을 확인했다. 일반 사용자 대상으로 10%+를 보여줄 수 있다면, freemium 평균 전환율 2-5%를 크게 상회하는 수치다. K-pop 팬 평균 연 소비 ~$1,400(Gitnux, 2025) 대비 AIoIA 구독료는 4% 수준으로 가격 저항이 낮을 것으로 예상된다.

### 2.3 투자자 설득을 위한 데이터 패키지 (목표)

Phase 2 종료 시 아래 데이터를 시드 IR 덱에 포함할 수 있어야 한다:

**이미 확보된 데이터 (Phase 1):**

```
1. 트랙션: TikTok 500K 조회, 5,000 리드, 500+ 가입 (CAC $0.005, 마케팅 $25)
2. 유료 검증: 공모전 50% 전환율 (10-12만원 티켓, 선택 편향 포함)
3. UT 몰입도: 프로듀서 역할 인지 6/6, 캐스팅 몰입 4-5/5, 경쟁 심리 5/6
4. 감정적 연결: AI Boyfriend 실험에서 75%가 진심 어린 애정 표현
5. 바이럴 잠재력: 대학생 19명이 무료로 콘텐츠 제작 -> 400명+ 방문자 유도
```

**Phase 2에서 추가로 확보할 데이터:**

```
1. PMF 신호: "매우 실망" 응답 40%+ (인앱 설문, N=최소 30명)
2. NPS: 30+ (추천 의향)
3. 전환 증명: 콘텐츠 소비 -> 생성 전환율 30%+ (PostHog 퍼널)
4. 공유 행동: 인앱 공유 버튼 클릭률 20%+ (PostHog 이벤트)
5. 수익화 의향: 유료 전환 의향 10%+ (인앱 설문)
6. 감정적 연결 정량화: 세션당 채팅 10개+ 메시지 (PostHog 이벤트)
```

**Phase 1 정성 + Phase 2 정량의 결합**이 시드 IR의 설득력을 만든다. 정성 데이터(UT 발화, 감정 반응)는 "왜 좋아하는가"를, 정량 데이터(설문 수치, 행동 이벤트)는 "얼마나 좋아하는가"를 증명한다.

| `share_completed` | 실제 공유 완료 시 (링크 복사 등) | `share_target` | 실제 공유율 |

| `survey_shown` | 설문 모달 노출 시 | - | 설문 응답률 분모 |
| `survey_completed` | 설문 전체 완료 시 | 각 문항 응답값 | 설문 응답률 분자 |

### 3.9 컨셉 디벨롭 체험 (최소 육성 경험)

## 5. Go/Stop 기준

### 5.1 Phase 3 (유료 전환 검증) Go 기준

Phase 3으로 진행하려면 아래 조건을 **모두** 충족해야 한다:

| 항목 | Go 기준 | 근거 |
|------|---------|------|
| "매우 실망" 응답 | 40% 이상 | Sean Ellis PMF 기준 |
| NPS | 30 이상 | B2C 우수 기준 |
| 소비 -> 생성 전환율 | 30% 이상 | Chicken-and-Egg 해결 가능성 증명 |
| D7 Retention | 15% 이상 | Phase 2 로그인 도입 후 계정 기반 정확 측정 |
| 유효 설문 응답 수 | 30건 이상 | UT 규모에서의 최소 유의미 표본 |

**해석:**
- "매우 실망" 40%+: 사용자에게 필수 서비스가 되었다
- NPS 30+: 자발적 추천 의향이 있다
- 전환율 30%+: UGC 플랫폼 확장의 근거가 있다
- D7 Retention 15%+: 재방문 습관이 형성되었다 (업계 평균 13% 대비)
- -> **시드 IR을 시작하고, 유료 전환 검증에 투자할 가치가 있다**

### 5.2 Stop 기준

아래 조건 **중 하나라도** 해당되면 Stop이다:

| 항목 | Stop 기준 | 의미 |
|------|----------|------|
| "매우 실망" 응답 | 25% 미만 | 사용자에게 필수가 아님 |
| NPS | 0 미만 | 비추천자가 추천자보다 많음 |
| 소비 -> 생성 전환율 | 15% 미만 | Chicken-and-Egg 해결 불가 |
| D7 Retention | 5% 미만 | 재방문 습관 형성 실패 |
| 설문 응답률 | 10% 미만 | 유저 참여 자체가 부족 |

### 5.3 Pivot 옵션 (Stop 시)

| 옵션 | 설명 | 해당 상황 |
|------|------|----------|
| Option A: 제품 개선 후 재측정 | "다소 실망" 응답자 피드백 분석 -> 핵심 개선 -> 2주 후 재측정 | "매우 실망" 25-39% (PMF에 근접) |
| Option B: B2C SaaS 전환 | 개인 AI 컴패니언 서비스로 pivot. 1:1 채팅 경험만 강화, 플랫폼 포기 | 소비->생성 전환율 15% 미만이지만 세션당 채팅 메시지 수 15개 이상, 재사용 의향 4.0+/5.0일 때 |
| Option C: B2B 도구 전환 | 소속사/MCN 대상 AI 아이돌 생성 도구 | 그룹 생성 완료율 70%+ 유지되지만 콘텐츠 클릭률 20% 미만, "매우 실망" 25% 미만일 때 (만드는 도구로는 가치 있으나 소비 플랫폼으로는 부족) |
| Option D: 종료 | 사업 종료 | 모든 지표 Stop 기준 미달 |
