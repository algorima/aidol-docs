## 1. 개요

### 1.1 Phase 2 목적

Phase 2는 Phase 1에서 출시한 MVP를 기반으로, **"사용자가 이 서비스 없이는 매우 실망한다"**는 것을 검증하는 단계다. Sean Ellis의 PMF 측정법을 적용하여 Product-Market Fit 달성 여부를 확인하고, Phase 3(유료 전환 검증)으로의 진행 근거를 마련한다.

### 1.2 Phase 1과의 관계

Phase 1 (Sprint 1-3)은 핵심 경험 검증 단계로, "나만의 아이돌을 만들고 대화하는 경험이 사용자를 만족시키는가"를 검증했다. Phase 2는 Phase 1의 Go 기준을 달성한 후에만 진입한다.

**Phase 1 Go 기준 (Phase 2 진입 조건):**

| 항목 | Go 기준 | Stop 기준 |
|------|---------|----------|
| 가입자 수 | 1,000명 이상 | 300명 미만 |
| D7 Retention | 15% 이상 | 7% 미만 |
| 생성 완료율 | 70% 이상 | 30% 미만 |

추가로, platform-strategy-prd에서 정의한 Phase 2 전환 조건도 충족해야 한다:

| 항목 | 기준 |
|------|------|
| 세션당 콘텐츠 탐색 | 3개 이상 |
| 소비 -> 생성 전환율 | 30% 이상 |
| 공유 의향 점수 | 4.0/5.0 이상 |

### 1.3 기간

Sprint 4-6, 약 3주 (Phase 1 완료 직후)

## 2. 목표와 가설

### 2.1 핵심 가설

**"사용자가 이 서비스 없이는 매우 실망한다"**

Phase 1에서 생성의 재미, 소비 욕구, 전환 가능성, 공유 의향을 검증했다면, Phase 2에서는 이 경험이 사용자에게 **필수적인 서비스**가 되었는지를 측정한다.

### 2.2 Sean Ellis PMF 측정법

Sean Ellis는 수백 개 스타트업을 분석하여 PMF를 정량적으로 측정하는 방법을 제시했다.

**핵심 질문:**
> "이 제품을 더 이상 사용할 수 없다면 어떤 기분이겠습니까?"

**응답 선택지:**
1. 매우 실망할 것이다
2. 다소 실망할 것이다
3. 별로 실망하지 않을 것이다
4. 해당 없음 (이미 사용하지 않음)

**판단 기준:**
- "매우 실망" 응답이 **40% 이상**이면 PMF 달성으로 판단
- 40% 미만이면 제품 개선이 필요
- Superhuman은 이 방법으로 PMF를 찾아 유니콘이 되었다

**참고:** PMF 설문에서 의미 있는 결과를 얻으려면 최소 1,000명의 표본이 필요하다 (신뢰구간 95%, 오차범위 3%).

### 2.3 North Star Metric

**"매우 실망" 응답 비율 40% 이상**

이 지표가 달성되면 성장에 집중해도 된다는 신호다.

## 3. 기능 요구사항

### 3.1 PMF 설문 시스템 (Sean Ellis)

**유저 스토리:**
```
As a 2주 이상 사용한 활성 유저,
I want to 서비스에 대한 솔직한 피드백을 전달하면,
So I can 서비스 개선에 기여할 수 있다.
```

**기능 상세:**

| 항목 | 내용 |
|------|------|
| 설문 대상 | 최소 2주 이상 사용, 핵심 기능(생성 + 채팅) 경험 완료한 유저 |
| 트리거 조건 | 가입 후 14일 경과 AND 채팅 3회 이상 완료 |
| 설문 형식 | 인앱 모달 (페이지 전환 없이) |
| 핵심 질문 | "이 제품을 더 이상 사용할 수 없다면 어떤 기분이겠습니까?" |
| 응답 선택지 | 매우 실망 / 다소 실망 / 별로 실망하지 않음 / 해당 없음 |
| 후속 질문 (매우 실망 선택 시) | "이 서비스의 가장 큰 가치는 무엇인가요?" (자유 텍스트) |
| 후속 질문 (별로 실망하지 않음 선택 시) | "어떤 점이 개선되면 더 자주 사용하시겠습니까?" (자유 텍스트) |
| 노출 빈도 | 유저당 1회 (재측정은 30일 이후 가능) |
| 닫기/스킵 | 허용 (스킵 시 3일 후 1회 재노출) |
| 우선순위 | P0 |

**기술 요구사항:**
- PostHog 커스텀 이벤트로 응답 수집
- 설문 응답 데이터는 서버에 저장 (유저 익명 ID 연결)
- 설문 상태 관리: `survey_shown`, `survey_completed`, `survey_skipped`

### 3.2 NPS 측정 기능

**유저 스토리:**
```
As a 활성 유저,
I want to 서비스를 주변에 추천할 의향을 표현하면,
So I can 서비스 품질에 대한 의견을 전달할 수 있다.
```

**기능 상세:**

| 항목 | 내용 |
|------|------|
| 설문 대상 | 최소 1주 이상 사용한 활성 유저 |
| 트리거 조건 | 가입 후 7일 경과 AND 세션 3회 이상 |
| 설문 형식 | 인앱 바텀시트 또는 모달 |
| 핵심 질문 | "이 서비스를 친구나 동료에게 추천할 가능성은?" |
| 응답 형식 | 0-10 점 슬라이더 |
| 후속 질문 | "그 이유는 무엇인가요?" (자유 텍스트, 선택) |
| NPS 계산 | (추천자 비율) - (비추천자 비율) / 추천자: 9-10점, 비추천자: 0-6점 |
| 노출 타이밍 | PMF 설문과 다른 날에 노출 (최소 3일 간격) |
| 우선순위 | P1 |

**기술 요구사항:**
- PostHog 커스텀 이벤트로 응답 수집
- NPS 점수 자동 계산 및 대시보드 반영

### 3.3 피드백 수집 UI

**유저 스토리:**
```
As a 서비스에 의견이 있는 유저,
I want to 언제든 피드백을 보낼 수 있으면,
So I can 불만이나 개선 아이디어를 바로 전달할 수 있다.
```

**기능 상세:**

| 항목 | 내용 |
|------|------|
| 위치 | 설정/프로필 메뉴 내 "피드백 보내기" 항목 |
| 입력 형식 | 카테고리 선택 (버그 / 기능 요청 / 기타) + 자유 텍스트 |
| 제출 후 | "소중한 의견 감사합니다" 확인 메시지 |
| 데이터 저장 | PostHog 이벤트 + 서버 DB |
| 우선순위 | P1 |

### 3.4 데이터 분석 대시보드 (PostHog)

**유저 스토리:**
```
As a 프로덕트 매니저,
I want to 핵심 지표를 실시간으로 모니터링하면,
So I can 데이터 기반 의사결정을 할 수 있다.
```

**기능 상세:**

| 항목 | 내용 |
|------|------|
| 플랫폼 | PostHog 대시보드 |
| 필수 차트 | PMF 설문 결과 (파이 차트), NPS 추이 (라인 차트), 리텐션 코호트, 핵심 퍼널 (가입 -> 생성 -> 채팅 -> 재방문) |
| 세그먼트 필터 | 가입 기간, 활동 빈도, 생성/미생성 유저 |
| 알림 설정 | 설문 응답 100건 도달 시 Slack 알림 |
| 이벤트 트래킹 추가 | `pmf_survey_shown`, `pmf_survey_completed`, `nps_survey_completed`, `feedback_submitted`, `share_clicked`, `share_completed` |
| 우선순위 | P0 |

**기술 요구사항:**
- PostHog SDK 이벤트 정의서 작성
- 대시보드 템플릿 사전 구성
- 주요 지표 일간/주간 자동 리포트 설정

## 5. Go/Stop 기준

### 5.1 Phase 3 (유료 전환 검증) Go 기준

Phase 3으로 진행하려면 아래 조건을 **모두** 충족해야 한다:

| 항목 | Go 기준 |
|------|---------|
| "매우 실망" 응답 | 40% 이상 |
| NPS | 30 이상 |
| 자발적 공유 | 20% 이상 |
| 설문 유효 응답 수 | 100건 이상 (통계적 유의성) |

**해석:**
- "매우 실망" 40% 이상: 사용자에게 필수 서비스가 되었다
- NPS 30 이상: 자발적 추천 의향이 있다
- 자발적 공유 20% 이상: 바이럴 가능성이 있다
- -> **유료 전환 검증에 투자할 가치가 있다**

### 5.2 Stop 기준

아래 조건 **중 하나라도** 해당되면 Stop이다:

| 항목 | Stop 기준 | 의미 |
|------|----------|------|
| "매우 실망" 응답 | 25% 미만 | 사용자에게 필수가 아님 |
| NPS | 0 미만 | 비추천자가 추천자보다 많음 |
| 설문 응답률 | 10% 미만 | 유저 관심/참여 자체가 부족 |
| D7 Retention | 7% 미만 (Phase 1 대비 하락) | 지속 사용 동력 없음 |

### 5.3 Pivot 옵션 (Stop 시)

| 옵션 | 설명 | 조건 |
|------|------|------|
| Option A: 제품 개선 후 재측정 | "다소 실망" 응답 분석 -> 개선점 도출 -> 2주 후 재측정 | "매우 실망" 25-39% |
| Option B: B2C SaaS 전환 | 개인 AI 컴패니언 서비스로 pivot (플랫폼 포기) | 소비 지표 낮지만 채팅 만족도 높을 때 |
| Option C: B2B 도구 전환 | 소속사/MCN 대상 AI 아이돌 생성 도구 | 크리에이터 세그먼트만 PMF 달성 시 |
| Option D: 종료 | 사업 종료 | 모든 지표 Stop 기준 미달 |
