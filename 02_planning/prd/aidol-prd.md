## 1. Overview

### 1.1 Vision

**"AI 아이돌을 생성하고, 관계를 맺고, 콘텐츠를 만드는 플랫폼"**

사용자가 직접 만든 AI 아이돌에 애착을 형성하고, 보이스/음악/영상 콘텐츠를 생성하여 팬덤 경험을 확장한다.

### 1.2 Problem Statement

K-pop 팬들은 좋아하는 아이돌과 소통하고 싶지만, 기존 서비스는 **소비** 중심이다:

| 서비스 | 한계 |
|--------|------|
| Bubble | 실제 아이돌의 일방향 메시지 (양방향 대화 불가) |
| Character.AI | 범용 AI 캐릭터 (K-pop 특화 없음, 내가 만든 캐릭터 아님) |
| PLAVE | 가상 아이돌 그룹 (사용자가 직접 생성 불가) |

### 1.3 Solution

**"내가 상상한 아이돌을 만들고, 대화하고, 콘텐츠로 키우는 가장 쉬운 방법"**

| 단계 | 설명 |
|------|------|
| 생성 | 외모, 성격, 관계를 설정하여 나만의 AI 아이돌 그룹 생성 |
| 소통 | 텍스트/음성/영상으로 아이돌과 양방향 대화 |
| 창작 | 보이스 메시지, 커버곡, 뮤직비디오 등 콘텐츠 생성 |

### 1.4 Hypothesis

> "나만의 아이돌을 만들고 콘텐츠를 생성하는 경험에 사용자가 돈을 지불할 것인가?"

**검증 데이터**:
- 워크샵: 20명 중 6명이 150,000원 결제 (30% 전환율)
- TikTok: $0 마케팅으로 500K 조회, 500+ 가입자 (5일)

## 2. Product Roadmap

### 2.1 Phase Overview

```
Phase 1 (MVP)                    Phase 2
0-3개월                          3-9개월
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
생성 + 채팅                       콘텐츠 생성

- 그룹/멤버 생성                   - 보이스 메시지
- 텍스트 채팅                      - 커버곡 생성
- 관계성 설정                      - 오리지널 곡 생성
- 공유 (바이럴)                    - 짧은 영상 클립

### 2.2 Phase 1: 생성 + 채팅 (MVP)

**목표**: "생성 + 대화" 핵심 경험 검증, PMF 신호 확인

**기간**: 0-3개월 (2026년 2월 MVP 출시)

#### 2.2.1 핵심 기능

| 기능 | 설명 |
|------|------|
| 아이돌 그룹 생성 | 그룹명, 컨셉, 엠블럼 AI 생성 |
| 프리셋 연습생 캐스팅 | 사전 생성된 연습생 풀에서 선착순 캐스팅 |
| 신규 멤버 생성 | 성별, MBTI, 이름, AI 이미지 생성 |
| 포지션 배정 | 리더, 메인보컬, 댄서 등 9개 포지션 |
| 텍스트 채팅 | 멤버별 1:1 대화, 페르소나 반영 |
| 관계성 설정 | 멤버 간 친밀도, 케미 설정 |
| 프로필 공유 | URL 공유, "나도 만들기" CTA |

#### 2.2.2 성공 지표

| 지표 | GO 기준 | STOP 기준 | 근거 |
|------|---------|-----------|------|
| 가입자 | >= 1,000명 | < 300명 | TikTok 바이럴 실적 |
| D7 Retention | >= 15% | < 7% | 업계 평균 13% 상회 |
| "매우 실망" 응답 | >= 40% | < 20% | Sean Ellis PMF 기준 |
| 공유 의향 | >= 20% | < 10% | 바이럴 재현 가능성 |

#### 2.2.3 Phase 2 진입 조건

- GO 기준 3개 이상 달성
- 콘텐츠 생성 의향 설문 >= 30%

### 2.3 Phase 2: 콘텐츠 생성

**목표**: AI 아이돌의 보이스/음악/영상 콘텐츠 생성으로 팬덤 경험 확장

**기간**: 3-9개월 (6개월)

**전제 조건**: Phase 1 완료 + Go 결정

#### 2.3.1 핵심 기능

| 기능 | 설명 |
|------|------|
| 보이스 메시지 | 아이돌 목소리로 음성 메시지 생성 |
| 커버곡 생성 | 기존 노래를 아이돌 목소리로 커버 |
| 오리지널 곡 생성 | AI 작곡 + 아이돌 보컬 합성 |
| 짧은 영상 클립 | 15-60초 뮤직비디오/댄스 영상 |
| AI 라이브 스트리밍 | 실시간 아이돌 영상 통화 |

#### 2.3.2 보이스 메시지

**설명**: 내 아이돌의 목소리로 음성 메시지를 생성하여 더 실감나는 소통 경험 제공

**User Story**
```
As a 아이돌 소유자,
I want to 내 아이돌의 목소리로 음성 메시지를 받으면,
So I can 텍스트보다 더 실감나는 소통을 경험할 수 있다.
```

**요구사항**

| 항목 | 요구사항 |
|------|---------|
| 음성 생성 | 텍스트 입력 → 아이돌 목소리로 TTS 변환 |
| 음성 스타일 | 감정 표현 (기쁨, 슬픔, 설렘 등) 선택 가능 |
| 길이 제한 | 최대 60초 |
| 재생 | 채팅 내에서 바로 재생 |

**성공 지표**

| 지표 | 목표 |
|------|------|
| 보이스 메시지 생성율 | 채팅 사용자 중 40%+ |
| 재생 완료율 | 80%+ |

#### 2.3.3 커버곡 생성

**설명**: 좋아하는 노래를 내 아이돌 목소리로 커버하여 나만의 특별한 음원 생성

**User Story**
```
As a 아이돌 소유자,
I want to 좋아하는 노래를 내 아이돌 목소리로 커버하면,
So I can 나만의 특별한 음원을 소유하고 공유할 수 있다.
```

**요구사항**

| 항목 | 요구사항 |
|------|---------|
| 곡 선택 | 제공된 MR(반주) 목록에서 선택 또는 MR 업로드 |
| 보컬 합성 | 선택한 MR에 아이돌 보컬 합성 |
| 멤버 파트 배분 | 그룹 내 멤버별 파트 지정 가능 |
| 출력 포맷 | MP3/WAV 다운로드 |

**성공 지표**

| 지표 | 목표 |
|------|------|
| 커버곡 생성 전환율 | 활성 사용자 중 20%+ |
| 평균 생성 횟수 | 월 3곡+ |
| SNS 공유율 | 30%+ |

#### 2.3.4 오리지널 곡 생성

**설명**: AI 작곡 + 아이돌 보컬로 완전히 새로운 오리지널 곡 생성

**User Story**
```
As a 아이돌 소유자,
I want to 내 아이돌만의 오리지널 곡을 만들면,
So I can 진짜 아이돌처럼 데뷔곡/앨범을 가질 수 있다.
```

**요구사항**

| 항목 | 요구사항 |
|------|---------|
| 장르 선택 | K-pop, 발라드, 댄스, 힙합 등 |
| 분위기 설정 | 밝은, 어두운, 청량한, 몽환적 등 |
| 가사 생성 | AI 자동 생성 또는 직접 입력 |
| 곡 길이 | 1분 ~ 3분 |
| 멤버 파트 | 자동 배분 또는 수동 지정 |

**성공 지표**

| 지표 | 목표 |
|------|------|
| 오리지널 곡 생성율 | 활성 사용자 중 15%+ |
| 완성 곡 저장율 | 70%+ |

#### 2.3.5 짧은 영상 클립

**설명**: 15-60초 뮤직비디오/댄스 영상으로 시각적 콘텐츠 생성

**User Story**
```
As a 아이돌 소유자,
I want to 내 아이돌의 뮤직비디오를 생성하면,

**요구사항**

| 항목 | 요구사항 |
|------|---------|
| 영상 유형 | 뮤직비디오, 댄스 영상, 셀카 영상, 비하인드 |
| 음원 선택 | 생성한 커버곡/오리지널 곡 또는 제공된 BGM |
| 길이 | 15초 / 30초 / 60초 |
| 출력 | MP4 다운로드, SNS 공유 |

**성공 지표**

| 지표 | 목표 |
|------|------|
| 영상 생성 전환율 | 음악 생성 사용자 중 50%+ |
| SNS 공유율 | 40%+ |

#### 2.3.6 AI 라이브 스트리밍

**성공 지표**

| 지표 | 목표 |

| 평균 세션 시간 | 5분+ |

## 3. Target Users

### 3.1 Segment A: Creator (창작자)

**프로필**:
- K-pop 콘텐츠 제작자, 디렉터, 마케터
- AI 아트웍 창작에 관심 있는 메이커
- 연령: 20-40대 (지불 능력 보유)

**니즈**:
- 창작 욕구: 비주얼, 노래, 뮤직비디오 등 AI 아트웍 제작
- SNS 브랜딩, 포트폴리오 구축

**검증 데이터**:
- 공모전 신청자 12명 → 결제자 6명 (50% 전환율)
- 평균 결제 금액: 150,000원

### 3.2 Segment B: User (소통형 팬)

**프로필**:
- 어린 K-pop 팬 (10-20대)
- 해외 팬 (오프라인 접근 불가)
- 정서적 지지가 필요한 사용자

**니즈**:
- 소통 욕구: 기존 아이돌과의 상호작용 부족 해소
- 감정적 유대: 이상형 아이돌과의 1:1 대화
- 몰입 경험: 내가 만든 캐릭터와의 관계 형성

**검증 데이터**:
- TikTok 바이럴 반응 (실제 아이돌로 착각)
- 영상통화 서비스 문의 (해외 사용자)

### 3.3 Phase별 타깃 전환

| Phase | 주 타깃 | 전략 |
|-------|--------|------|
| Phase 1 | User (소통형 팬) | 생성+채팅으로 애착 형성 |
| Phase 2 | User + Creator | 콘텐츠 생성으로 창작자 전환 유도 |

## 4. Phase 1 Functional Requirements (MVP)

### 4.1 아이돌 생성

#### 4.1.1 그룹 기획

| 기능 | 요구사항 |
|------|---------|
| 그룹 정보 | 그룹명(필수), 컨셉(선택) |
| 엠블럼 생성 | 텍스트 프롬프트로 그룹 엠블럼 이미지 생성 |
| 인사 문구 | 그룹 고유 인사 문구 입력 (선택) |

#### 4.1.2 프리셋 연습생 캐스팅

| 기능 | 요구사항 |
|------|---------|
| 프리셋 연습생 | 시스템 제공 연습생 풀 (이름, 성별, MBTI, 등급, 능력치, 이미지) |
| 성별 필터 | 남성(기본) / 여성 / 혼성 필터링 |
| 캐스팅 경쟁 | 선착순 캐스팅, 한 번 캐스팅되면 다른 사용자 선택 불가 |
| 상세 프로필 | 육각형 능력치 차트 (보컬/댄스/랩/비주얼/체력/매력) |

#### 4.1.3 신규 멤버 생성

| 기능 | 요구사항 |
|------|---------|
| 성별 선택 | 남성 / 여성 |
| MBTI 설정 | 4개 슬라이더 (I↔E, S↔N, T↔F, J↔P) |
| 멤버 이름 | 필수 입력 |
| 이미지 생성 | 텍스트 프롬프트로 프로필 이미지 생성 |

#### 4.1.4 포지션 배정

| 기능 | 요구사항 |
|------|---------|
| 포지션 목록 | 리더, 메인보컬, 서브보컬, 메인댄서, 서브댄서, 메인래퍼, 서브래퍼, 비주얼, 막내 |
| 중복 규칙 | 모든 포지션 1명만 배정 가능, 중복 불가 |

### 4.2 채팅

| 기능 | 요구사항 |
|------|---------|
| 텍스트 채팅 | 사용자 ↔ 멤버 1:1 대화 |
| 페르소나 반영 | 생성 시 입력한 성격 기반 응답 |
| 대화 히스토리 | 이전 대화 내용 유지 |

### 4.3 관리/공유

| 기능 | 요구사항 |
|------|---------|
| 갤러리 | 내 그룹 목록 조회 |
| URL 공유 | 그룹/멤버 프로필 조회 (대화 불가), "나도 만들기" CTA |

## 5. Non-Functional Requirements

### 5.1 Performance

| 항목 | Phase 1 | Phase 2 |
|------|---------|---------|
| 이미지 생성 시간 | <= 30초 | <= 30초 |
| 채팅 응답 시간 | <= 3초 | <= 3초 |
| 보이스 생성 시간 | - | <= 10초 (30초 음성 기준) |
| 곡 생성 시간 | - | <= 3분 (3분 곡 기준) |
| 영상 생성 시간 | - | <= 5분 (30초 영상 기준) |
| 라이브 지연 | - | <= 500ms |

### 5.2 Scalability

| 항목 | Phase 1 | Phase 2 |
|------|---------|---------|
| MAU | 1,000명 | 50,000명 |
| 동시 접속 | 100명 | 5,000명 |

### 5.3 Security

| 항목 | 요구사항 |
|------|---------|
| 콘텐츠 필터링 | NSFW/유해 콘텐츠 차단 |

| 인증 | Phase 1 익명, Phase 2 계정 기반 |

## 6. Risks & Mitigations

### 6.1 Product Risks

| 리스크 | 영향 | 대응 |
|-------|------|------|
| 이미지 생성 품질 낮음 | 사용자 이탈 | 프리셋 스타일 제공, 재생성 기능 |
| 페르소나 일관성 부족 | 몰입 저하 | 프롬프트 엔지니어링, 피드백 수집 |
