# Phase 2 방향 및 역할 브리핑

- 작성일: 2026-03-03
- 대상: 수지 (UX 리서치 / 사용자 테스트)
- 작성자: 소연

이 문서는 미진이 합류하면서 기능 명세서 작성을 맡게 된 배경, Phase 2가 왜 이 방향으로 가고 있는지, 그리고 수지가 앞으로 무엇에 집중하면 되는지를 정리한 브리핑입니다.

## 1. Phase 2 방향: 왜 "실재감"인가

Phase 2의 핵심 과제는 "AI 아이돌에 실재감 부여"입니다. 이 방향은 D.Camp 오피스아워 피드백과 MARU 입주 지원서에서 일관되게 나온 결론입니다.

### 1.1 D.Camp 오피스아워 피드백 (2026-02-23, 성낙환 팀장)

성낙환 팀장은 서비스를 처음 보고 이렇게 말했습니다.

> "지금의 느낌은 약간 게임... 프린세스 메이커의 느낌이긴 하죠."

> "가상의 아이돌이지만 얘네가 어느 정도까지는 뭔가 실제하는 것처럼 보여주는 그런 장치들을 내리시는 게 좋은 것 같아요."

> "채팅만으로는 부족하고, 공연이나 라이브 방송처럼 채팅 외 인터랙션 채널이 있어야 몰입 후 현타가 안 온다."

이는 팀 내부에서 이미 공감하던 지점이었습니다. 사용자 테스트에서도 동일하게 확인된 패턴으로, 아이돌을 만들고 나서 "그래서 뭐?"라는 반응이 나오는 구간이 존재합니다. 그룹 생성 완료 후 콘텐츠로 바로 연결되지 않으면 실재감이 생기지 않는다는 것이 D.Camp 피드백 이후 팀의 공통된 인식입니다.

팀 내부에서 합의한 해결 방향 3가지:

1. 그룹 생성 직후 콘텐츠가 바로 보이도록 플로우 변경 (소연 제안)
2. AI 영상 생성으로 아이돌 실재감 강화 (기술 가능성 검토 중)
3. 다른 유저가 만든 그룹을 볼 수 있는 피드/투표 기능 추가 (제이 제안)

### 1.2 D.Camp 팀 내부 디브리핑 (2026-02-24)

오피스아워 다음 날 팀 전체가 피드백을 공유하고 공감한 내용입니다.

- 서비스 완성 후 "인형놀이에 불과하구나"라는 느낌을 어떻게 극복할 것인가가 핵심 질문
- 콘텐츠 자동 생성 플로우, AI 영상(인사 영상) 생성, 타 유저 노출 및 투표/팔로우 기능을 Phase 2 아이디어 후보로 확정
- 이 세 방향 중 기술적 가능성 검토 후 실행 순서 결정하기로 합의

### 1.3 MARU 입주 지원서에서 약속한 방향

MARU 지원서에 기재한 2026년 5~7월 목표:

- 모바일 앱 출시 (iOS/Android)
- 유료 전환 검증
- 육성 시스템 (친밀도/레벨업/일일 미션) 출시로 장기 리텐션 확보
- MRR $500+, 유료 전환 의향 10%+, D30 Retention 10%+ 달성

지원서에 작성한 차별점:

> "MVP(아이돌 생성/관계 설정/채팅) 기반 사용자 테스트에서 캐스팅 몰입도 4.7/5점, '오랜만에 뭔가에 몰입했다'는 정성 피드백 확인"

> "사용자 테스트를 매 Phase마다 진행하며 정성적 데이터에서 PMF 신호를 잡아내는 방법을 체계화했습니다."

MARU에 제출한 우리 팀의 강점은 "리서치 기반으로 제품 방향을 검증하는 팀"입니다. 이것이 수지의 역할이 왜 핵심인지와 직결됩니다.

## 2. PR #95: 이 방향의 첫 결과물

### 2.1 작성 배경

2월 25일 Sprint 3 중간 점검 미팅에서 Phase 2 Sprint 4의 기능 범위를 논의했습니다. D.Camp 피드백과 팀 내부 디브리핑에서 합의된 방향을 첫 스프린트에 반영한 것입니다.

### 2.2 누가 작성했나

미진(모바일 앱 개발자)이 2026-03-02부터 2026-03-03까지 초안을 작성해 PR #95로 제출했습니다.

미진은 개발자 시각에서 기능 명세를 직접 작성하고, 백엔드 API 응답 예시, Frontend UI 처리 방식, Backend SQL 수정 범위까지 구체적으로 포함했습니다. 디자인이 필요한 항목에는 `(@jeonjay0323)` 태그로 제이에게 요청을 명시했습니다.

### 2.3 포함된 6가지 기능

| 번호 | 기능 | 핵심 변경 내용 |
|------|------|----------------|
| 1 | 그룹 기획 완료 플로우 변경 | "프로필 공유" 버튼 → "내 그룹 확인하기"로 변경, 생성 직후 콘텐츠로 바로 연결 |
| 2 | 멤버 상세 화면 개선 | 케미 화면을 멤버 상세 화면으로 대체, 이미지 크게 보기 추가 |
| 3 | 피드 탭 추가 | 내 그룹 페이지에 피드 탭 신설, 현재는 "곧 업로드됩니다" 안내 |
| 4 | 아이돌 생성 상태 유지 | 탭 이동 시 생성 진행 상태가 초기화되던 문제 해결 |
| 5 | Buppy MCP 서버 | 운영자가 피드에 이미지/영상을 수동 업로드하는 내부 도구 |
| 6 | 계약완료 연습생 미노출 | 홈 화면(캐스팅 리스트)에서 계약완료 연습생 제거, Backend SQL 수정 |

6번 기능은 수지가 진행한 UT 피드백이 직접 반영된 결과입니다. U5, U7, U10, U12 네 명이 공통으로 "왜 앞에 있는지 모르겠다", "블러 처리된 연습생이 부러움이나 경쟁심을 유발하지 않는다"고 피드백했고, 이 근거가 명세서에 명시되어 변경 이유를 추적 가능하게 했습니다.

1번, 2번, 3번 기능은 D.Camp 피드백 이후 합의된 "실재감 강화" 방향의 첫 구현입니다.

## 3. 수지의 역할

### 3.1 달라진 것

미진이 기능 명세서 작성을 맡으면서 수지의 역할이 명확하게 정리되었습니다.

수지가 집중할 일:

- UX 리서치 및 사용자 테스트 설계·진행
- 인사이트 도출 및 문서화
- 인사이트를 PRD와 IR 자료에 반영

더 이상 수지의 역할이 아닌 것:

- 기능 명세서 작성 (미진 담당)
- 스프린트 일정 조율
- 개발팀 기획 검토

### 3.2 수지의 일이 팀에 직결되는 이유

MARU 지원서에 적었듯 우리 팀의 가장 강력한 차별점은 "사용자 테스트 기반으로 제품 방향을 검증하는 팀"입니다. 이 근거를 만드는 사람이 수지입니다.

PR #95의 6번 기능이 좋은 예입니다. 수지의 UT 결과가 기능 결정으로 이어졌고, 그 근거가 명세서에 그대로 남아 있습니다. 앞으로도 이 흐름이 반복됩니다. 수지가 인사이트를 뽑으면 미진이 명세로 만들고, 그 인사이트는 PRD와 IR 자료에도 반영되어 투자자에게 "우리 제품이 왜 작동하는지"를 설명하는 근거가 됩니다.

### 3.3 지금 할 일

| 순서 | 할 일 | 결과물 |
|------|--------|--------|
| 1 | Phase 2 Sprint 4 기준 UT 설계 및 진행 | 인터뷰 가이드, 인터뷰 기록 |
| 2 | UT 결과 분석 및 인사이트 정리 | 인사이트 문서 (GitHub PR) |
| 3 | 인사이트를 PRD 관련 섹션에 반영 | PRD 업데이트 PR |
| 4 | 인사이트 중 IR 트랙션에 활용할 수 있는 내용 소연에게 공유 | 슬랙 공유 또는 메모 |

1번과 2번이 핵심입니다. 3번과 4번은 그 이후에 자연스럽게 이어집니다.
