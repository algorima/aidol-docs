> **작성일**: 2026-03-02
> **상태**: draft

## 개요

Phase 2 Sprint 1은 **실제감 강화**를 목표로 UI/UX 개선과 피드 기능 기반을 구현합니다.
사용자가 생성한 AI 아이돌에 대한 정보 접근성을 높이고, 콘텐츠 확장을 위한 피드 탭을 추가합니다.

---

## 1. 그룹 기획 완료 플로우 변경

그룹 생성 완료 후 사용자가 자연스럽게 내 그룹 페이지로 이동하도록 플로우를 변경합니다.

### 1-1. 플로우 변경

| 항목 | 내용 |
|------|------|
| **Page** | GroupComplete (그룹 기획 완료) |
| **현재 (AS-IS)** | "이 그룹의 다음 행보가 궁금하시다면" → [프로필 공유] [다른 그룹 만들기] |
| **변경 (TO-BE)** | "이 그룹의 다음 행보가 궁금하시다면" → [내 그룹 확인하기] → 내 그룹 페이지로 이동 |
| **변경 이유** | 그룹 완성 후 즉시 콘텐츠로 연결되지 않으면 실제감이 떨어짐 (UT 피드백, 멘토 피드백) |

---

## 2. 멤버 상세 화면 개선 (멤버간 케미 화면 대체)

기존 멤버간 케미 화면을 멤버 상세 정보를 볼 수 있는 화면으로 확장합니다.

### 2-1. 멤버 상세 화면

| 항목 | 내용 |
|------|------|
| **Page** | MemberDetail (기존 MemberChemistry 대체) |
| **설명** | 멤버 상세 정보 + 멤버간 케미 정보를 통합 표시 |

### 2-2. 멤버 이미지 크게 보기

| 항목 | 내용 |
|------|------|
| **기능** | 멤버 이미지 탭 시 전체 화면으로 확대 |
| **트리거** | 멤버 이미지 탭 |
| **동작** | 전체 화면 모달로 이미지 표시, 배경 탭 또는 닫기 버튼으로 닫힘 |

### 2-3. 표시 정보

| 항목 | 내용 |
|------|------|
| **멤버 정보** | 이름, MBTI, 포지션, 프로필 이미지 |
| **멤버간 케미** | 기존 케미 기능 유지 |

## 3. 피드 탭 추가

내 그룹 페이지에 피드 탭을 추가하여 향후 콘텐츠 확장을 준비합니다.

### 3-1. 피드 탭

| 항목 | 내용 |
|------|------|
| **Page** | MyGroup |
| **탭 위치** | 하이라이트 탭 옆 |
| **탭 구성** | [하이라이트] [피드] |

### 3-2. 피드 탭 화면 (임시)

| 항목 | 내용 |
|------|------|
| **화면 내용** | "곧 업로드 됩니다" 안내 문구 표시 |
| **비고** | 실제 피드 콘텐츠는 Buppy MCP 서버를 통해 수동 업로드 후 노출 |

---

## 4. 아이돌 생성 상태 유지

발견 탭에서 아이돌 생성 중 다른 탭으로 이동해도 생성 상태가 유지되도록 변경합니다.

### 4-1. 탭 이동 시 상태 유지

| 항목 | 내용 |
|------|------|
| **대상 탭** | 발견 (가운데 탭) |
| **현재 (AS-IS)** | 홈/내그룹 탭 이동 시 생성 상태 초기화됨 |
| **변경 (TO-BE)** | 홈/내그룹 탭 이동 시에도 생성 상태 유지됨 |

### 4-2. 상세 요구사항

| 항목 | 내용 |
|------|------|
| **유지 대상** | 발견 탭에서 진행 중인 아이돌 생성 데이터 (입력값, 생성 단계) |
| **유지 조건** | 홈/내그룹 탭 이동 시 상태 보존 |
| **복귀 동작** | 발견 탭 재진입 시 이전 생성 단계에서 이어서 진행 |
| **초기화 조건** | 생성 완료 또는 사용자가 명시적으로 취소한 경우 |

## 5. Buppy MCP 서버 (피드 콘텐츠 업로드)

피드 탭에 이미지/영상을 수동으로 업로드하기 위한 운영 도구를 개발합니다.

### 5-1. 개요

| 항목 | 내용 |
|------|------|
| **목적** | 피드 탭에 이미지/영상을 수동으로 업로드하기 위한 운영 도구 |
| **형태** | MCP (Model Context Protocol) 서버 |
| **사용자** | 운영자 (내부) |

### 5-2. 이미지 업로드

| 항목 | 내용 |
|------|------|
| **지원 포맷** | JPG, PNG, WebP |
| **대상 선택** | 특정 그룹/멤버 지정 |
| **메타데이터** | 캡션, 업로드 일시, 태그 (선택) |

### 5-3. 영상 업로드

| 항목 | 내용 |
|------|------|
| **지원 포맷** | MP4, MOV |
| **길이 제한** | TBD |
| **대상 선택** | 특정 그룹/멤버 지정 |
| **메타데이터** | 캡션, 업로드 일시, 태그 (선택) |

### 5-4. MCP 서버 기능

| 기능 | 설명 |
|------|------|
| 콘텐츠 업로드 | 이미지/영상 파일을 피드에 업로드 |
| 그룹/멤버 조회 | 업로드 대상 그룹/멤버 목록 조회 |
| 업로드 이력 조회 | 기존 업로드된 콘텐츠 목록 확인 |
| 콘텐츠 삭제 | 업로드된 콘텐츠 삭제 |

### 5-5. 연동 흐름

```
운영자 (Claude Code 등)
       ↓
   Buppy MCP 서버
       ↓
   AIdol 백엔드 API
       ↓
   피드 탭에 콘텐츠 노출
```
