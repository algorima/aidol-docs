## 개요

Phase 2 Sprint 4는 **실제감 강화**를 목표로 UI/UX 개선과 피드 기능 기반을 구현합니다.
사용자가 생성한 AI 아이돌에 대한 정보 접근성을 높이고, 콘텐츠 확장을 위한 피드 탭을 추가합니다.

---

## 1. 그룹 기획 완료 플로우 변경

그룹 생성 완료 후 사용자가 자연스럽게 내 그룹 페이지로 이동하도록 플로우를 변경합니다.

### 1-1. 플로우 변경

| 항목 | 내용 |
|------|------|
| **Page** | GroupComplete (그룹 기획 완료) |
| **현재 (AS-IS)** | "이 그룹의 다음 행보가 궁금하시다면" → [프로필 공유] [다른 그룹 만들기] |
| **변경 (TO-BE)** | "이 그룹의 다음 행보가 궁금하시다면" → [내 그룹 확인하기] → 내 그룹 페이지로 이동 |
| **변경 이유** | 그룹 완성 후 즉시 콘텐츠로 연결되지 않으면 실제감이 떨어짐 (UT 피드백, 멘토 피드백) |
| **디자인** | 필요 (@jeonjay0323) |

## 2. 멤버 상세 화면 개선 (멤버간 케미 화면 대체)

| 항목 | 내용 |

## 3. 피드 탭 추가

내 그룹 페이지에 피드 탭을 추가하여 향후 콘텐츠 확장을 준비합니다.

### 3-1. 피드 탭

| 항목 | 내용 |
|------|------|
| **Page** | MyGroup |
| **탭 위치** | 하이라이트 탭 옆 |
| **탭 구성** | [하이라이트] [피드] |

### 3-2. 피드 탭 빈 상태 (Empty State)

| 항목 | 내용 |
|------|------|
| **조건** | 피드 데이터 없을 때 |
| **화면 내용** | "곧 업로드 됩니다" 안내 문구 표시 |
| **비고** | 실제 피드 콘텐츠는 Buppy MCP 서버를 통해 수동 업로드 후 노출 |
| **디자인** | 필요 (@jeonjay0323) |

### 3-3. 피드 콘텐츠 표시

| 항목 | 내용 |
|------|------|
| **조건** | 피드 데이터 있을 때 |

| **탭 동작** | 이미지 → 이미지 뷰어 / 영상 → 영상 플레이어 |

## 4. 아이돌 생성 상태 유지

발견 탭에서 아이돌 생성 중 다른 탭으로 이동해도 생성 상태가 유지되도록 변경합니다.

### 4-1. 탭 이동 시 상태 유지

| 항목 | 내용 |
|------|------|
| **대상 탭** | 발견 (가운데 탭) |
| **현재 (AS-IS)** | 홈/내그룹 탭 이동 시 생성 상태 초기화됨 |
| **변경 (TO-BE)** | 홈/내그룹 탭 이동 시에도 생성 상태 유지됨 |

### 4-2. 상세 요구사항

| 항목 | 내용 |
|------|------|
| **유지 대상** | 발견 탭에서 진행 중인 아이돌 생성 데이터 (입력값, 생성 단계) |
| **유지 조건** | 홈/내그룹 탭 이동 시 상태 보존 |
| **복귀 동작** | 발견 탭 재진입 시 이전 생성 단계에서 이어서 진행 |
| **초기화 조건** | 생성 완료 또는 사용자가 뒤로가기 버튼을 눌러 취소한 경우 |
| **복수 DRAFT** | 여러 DRAFT가 동시 존재 시 가장 최신 DRAFT만 불러옴 |
| **디자인** | 불필요 (기존 화면 동작 변경만 해당) |

## 5. Buppy MCP 서버 (피드 콘텐츠 업로드)

피드 탭에 이미지/영상을 수동으로 업로드하기 위한 운영 도구를 개발합니다.

### 5-1. 개요

| 항목 | 내용 |
|------|------|
| **목적** | 피드 탭에 이미지/영상을 수동으로 업로드하기 위한 운영 도구 |
| **형태** | 운영 도구 서버 (코드명: Buppy MCP) |
| **사용자** | 운영자 (내부) |

### 5-2. 이미지 업로드

| 항목 | 내용 |
|------|------|
| **지원 포맷** | JPG, PNG, WebP |
| **대상 선택** | 특정 그룹/멤버 지정 |
| **메타데이터** | 캡션, 업로드 일시, 태그 (선택) |

### 5-3. 영상 업로드

| 항목 | 내용 |
|------|------|
| **지원 포맷** | MP4, MOV |
| **대상 선택** | 특정 그룹/멤버 지정 |
| **메타데이터** | 캡션, 업로드 일시, 태그 (선택) |

### 5-4. MCP 서버 기능

| 기능 | 설명 |
|------|------|
| 콘텐츠 업로드 | 이미지/영상 파일을 피드에 업로드 |
| 그룹/멤버 조회 | 업로드 대상 그룹/멤버 목록 조회 |
| 업로드 이력 조회 | 기존 업로드된 콘텐츠 목록 확인 |
| 콘텐츠 삭제 | 업로드된 콘텐츠 삭제 |

### 5-5. 연동 흐름

```
운영자
       ↓
   Buppy MCP 서버
       ↓
   AIdol 백엔드 API
       ↓
   피드 탭에 콘텐츠 노출
```

## 6. 홈 화면 계약완료 연습생 미노출

홈 화면(캐스팅 리스트)에서 계약완료된 연습생을 노출하지 않도록 변경합니다.

### 6-1. 계약완료 연습생 필터링

| 항목 | 내용 |
|------|------|
| **Page** | Home (데뷔 멤버 캐스팅) |
| **현재 (AS-IS)** | 계약완료 연습생이 모자이크 처리되어 리스트에 함께 노출됨 |
| **변경 (TO-BE)** | 계약완료 연습생은 홈 화면 리스트에서 제외하고, 캐스팅 가능한 연습생만 노출 |
| **변경 이유** | UT 피드백 반영 - 이미 캐스팅된 연습생의 얼굴이 블러 처리되어 보이지 않을 뿐더러, 부러움이나 경쟁심리가 생성되지 않음. 오히려 "왜 앞에 있는지 모르겠음" 등 탐색 비용만 높이는 결과 (U5, U7, U10, U12 공통 피드백) |
| **변경 범위** | 명시적 쿼리 파라미터 |
| **참고 UT** | [Sprint 1 UT U12 이정인 분석](https://github.com/algorima/aidol-docs/pull/87) - 계약완료 연습생 표시 개선 제안, 다른 유저에게 가면 아쉬움 1/5 (전체 유저 최저) |
