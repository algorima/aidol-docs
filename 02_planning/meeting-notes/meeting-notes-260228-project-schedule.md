- 회의 목적: Sprint 4 일정 수립, 콘텐츠 생성 방식 결정, 팀 운영 논의

---

> TL;DR: Sprint 4는 자동 콘텐츠 생성 보류, 수동 생성+업로드 방식으로 전환. 월요일 기획/디자인 확정 후 화요일부터 개발 착수, 토요일 배포 목표.

## 1. 요약

### 1.1 콘텐츠 생성 방식 전환

- Phase 2 핵심: 아이돌 콘텐츠(영상, 노래 등) 생성 및 제공
- 자동 생성(백그라운드 영상/노래 생성)은 비용(API 트래픽 과금)과 개발 리소스 문제로 보류
- 대안: 수동으로 콘텐츠 생성 후 DB에 업로드하는 방식으로 전환
- 사용자 테스트 시 실시간 생성이 안 되더라도, 사용자가 아이돌 구성하면 운영자가 콘텐츠를 수동 생성하여 올리는 워크플로우
- 콘텐츠 업로드용 페이지(어드민) 또는 스크립트 필요 — 백엔드에 요청 예정

### 1.2 Sprint 4 일정

- 월요일(3/2): 재희+미진 기능 범위 확정 미팅, UI 디자인 및 기능 명세서 완료
- 화요일(3/3): 프론트엔드+백엔드 개발 동시 착수
- 토요일(3/7): 개발 완료 및 배포
- 일요일(3/8): 새 기능 포함 사용자 테스트 추가 진행

### 1.3 Sprint 4 개발 범위

- 프론트엔드: UI 플로우 변경, 디자인 토큰 적용, 피드 화면 변경, 생성 중 이어서 할 건지 확인 프롬프트
- 백엔드: 콘텐츠 DB 업로드 기능 구축 (수동 업로드 지원)
- 그룹 채팅은 Sprint 4에서 시간 여유 있으면 포함 검토

### 1.4 팀 운영

- 시영 님은 Phase 2부터 빠지는 것으로 확정
- 프론트엔드: 은재 님 중심으로 운영 (지영 님은 회사 업무로 참여 제한적)
- 백엔드: 재형 님 중심, 영욱이 코드 리뷰 담당
- 일정 관리: 구글 캘린더(회사 워크스페이스)로 이전 중

### 1.5 사용자 테스트 및 마케팅

- 사용자 테스트는 배포 대기 없이 프로토타입/기존 버전으로도 계속 진행
- 미진이 만든 아이돌 영상(애교, 손 흔들기 등)을 틱톡에 업로드 → 팔로우/좋아요 유입 발생 중
- 영상 콘텐츠 결제 가능성 논의: 사용자가 아이돌 생성 직후 인사 영상 보면 결제 의향 높아질 것으로 예상

## 2. 결정 사항

- 자동 콘텐츠 생성 기능은 Sprint 4에서 제외, 수동 생성+업로드 방식으로 전환
- 월요일(3/2)까지 기획 확정 + UI 디자인 완료
- 화요일(3/3)부터 프론트엔드+백엔드 개발 동시 착수
- 토요일(3/7)까지 개발 완료 및 배포, 일요일(3/8)부터 UT 추가 진행
- 시영 님은 Phase 2부터 빠지는 것으로 확정
- 내일(3/1) 소연+미진 오프라인에서 콘텐츠 생성, 기획서 작성, UT 등 작업
