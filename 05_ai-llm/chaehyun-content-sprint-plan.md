## 1. 역할 정의

| 항목 | 내용 |
|------|------|
| 이름 | 채현 (chaeniverse) |
| 타이틀 | LLM 프롬프트 엔지니어 |
| 핵심 담당 | 이미지 생성 프롬프트 설계·실험·검증, LLM 시스템 프롬프트 설계 |

채현님은 프롬프트 설계와 실험에 집중합니다.
최종 이미지·영상 콘텐츠의 큐레이션과 마케팅 채널(TikTok, X) 운영은 제이님 담당입니다.

### 1.1 담당 영역

| 영역 | 설명 | 산출물 |
|------|------|--------|
| 이미지 생성 프롬프트 | 캐스팅 프로필 사진·엠블럼·포즈 이미지 프롬프트 파라미터 설계 및 품질 개선 | 프롬프트 파일, 실험 리포트 |
| 숏폼 영상 생성 | Grok image-to-video API로 멤버별 3~6초 클립 생성 실험 (포즈 이미지 → 영상 변환) | 영상 샘플, 실험 리포트 |
| LLM 프롬프트 | 채팅 시스템 프롬프트(멤버 페르소나), 세계관·서사 초안 LLM 실험 | 프롬프트 파일, 실험 리포트 |

### 1.2 채현↔제이 업무 분기점

| 채현 (프롬프트 설계) | 제이 (콘텐츠 최종 관장) |
|---------------------|------------------------|
| 이미지 생성 프롬프트 파라미터 실험 | 생성 결과물 큐레이션 및 품질 최종 판단 |
| LLM 세계관·서사 텍스트 초안 생성 | 마케팅 채널(TikTok, X)용 이미지·영상 편집·배포 |
| 프롬프트 실험 리포트 PR | 콘텐츠 방향 결정 및 브랜딩 연계 |

### 1.3 채현↔제형 핸드오프 기준

프롬프트 확정 후 제형님에게 전달하는 포맷과 절차입니다.

| 단계 | 채현 | 제형 |
|------|------|------|
| 프롬프트 확정 후 | JSON으로 PR 업로드 (`05_ai-llm/prompts/*.json`) | PR 리뷰 후 백엔드 연동 |
| JSON 포함 항목 | 고정 블록, 매핑 테이블(한국어 UI → 영어 프롬프트), 모델 파라미터 | — |
| API 연동 | 기능별 사용 API·엔드포인트 명시 | 엔드포인트 통합 |

## 2. 콘텐츠 품질 평가 가이드

영역에 따라 평가 방식이 다릅니다.

| 영역 | 평가 방식 | 평가자 |
|------|---------|--------|
| 이미지 생성 결과물 | 아이돌 감성에 맞는지 외부 피드백 | 미진(팬덤·서사 관점) + 제이(비주얼·컨셉 관점) |
| 채팅 시스템 프롬프트·멤버 페르소나 | 논문 리서치 기반으로 채현이 직접 기준 수립 및 자체 검증 | 채현 |
| 세계관·서사 LLM 텍스트 | 논문 리서치 기반으로 채현이 직접 기준 수립 및 자체 검증 | 채현 |

### 2.1 이미지 생성 품질 기준 (미진·제이 평가 적용)

미진과 제이도 Gemini / Grok API를 직접 사용해 이미지·영상을 뽑아보고 채현님 결과물과 함께 비교 검토합니다.

**비주얼 진정성**

| 항목 | 기준 | 탈락 예시 |
|------|------|---------|
| 아이돌 비주얼 | 케이팝 아이돌 특유의 피부 톤, 이목구비 비율, 머리카락 질감 | 인스타 필터 과다, 서양 잡지 모델 느낌 |
| 스타일링 | 칼군무 무대 의상 또는 일상 컨셉 포토 레퍼런스와 일치 | 의상이 판타지/코스프레로 흘러감 |
| 캐릭터 일관성 | 같은 멤버를 다른 프롬프트로 생성해도 동일 인물처럼 보임 | 헤어 컬러·눈 모양이 매번 달라짐 |
| 성별 고정 | 선택한 성별이 100% 유지됨 | 남성 선택 시 여성 분위기 혼재 |

**감성 적합성**

| 항목 | 기준 |
|------|------|
| 그룹 컨셉 반영 | 유저가 설정한 그룹 분위기가 이미지에 반영됨 |
| 포즈·표정 | 아이돌 컨셉 포토에서 자주 쓰는 포즈 (손하트, 브이, 눈 마주침 등) 자연스럽게 구현 |
| 배경·분위기 | 소속사 앨범 컨셉 포토 수준의 배경 처리 (과하지 않고 캐릭터가 살아남) |

**이미지 평가 프로세스**

```
채현 → 샘플 이미지 슬랙 공유
         ↓
미진 (팬덤·서사 관점) + 제이 (비주얼·컨셉 관점) 피드백
         ↓
채현 → 프롬프트 수정 후 재실험
         ↓
수 21:00 중간 점검 미팅에서 최종 확정
```

**Must-Pass 조건 (하나라도 미달 시 Reject)**

| 항목 | 기준 |
|------|------|
| 성별 고정 | 선택한 성별이 100% 유지됨 |
| 캐릭터 일관성 | 같은 멤버를 다른 프롬프트로 생성해도 동일 인물처럼 보임 |

**평가 등급 (Must-Pass 통과 전제)**

| 등급 | 기준 | 다음 단계 |
|------|------|---------|
| Pass | Must-Pass 통과 + 나머지 항목 80% 이상 충족 | 프롬프트 확정, 리포트 PR |
| Revise | Must-Pass 통과, 나머지 항목 1~2개 미달 | 프롬프트 수정 후 재평가 |
| Reject | Must-Pass 미달 또는 방향 자체가 다름 | 미진·제이와 레퍼런스 이미지 재논의 |

### 2.2 LLM 프롬프트 품질 기준 (채현 자체 수립)

채팅 시스템 프롬프트와 멤버 페르소나는 채현님이 관련 논문을 리서치하여 기준을 직접 설계합니다.
미진·제이 평가 대상이 아닙니다.

채현님이 리서치할 주요 영역:
- 페르소나 일관성 유지 (Persona Consistency in LLM)
- 감정 표현 및 공감 대화 (Empathetic Dialogue)
- 캐릭터 기반 대화 생성 (Character-based Conversation Generation)
- 롤플레잉 LLM 평가 방법론

리서치 결과는 `05_ai-llm/evaluations/` 에 문서로 정리하고 PR 업로드합니다.

채팅 시스템 프롬프트는 사용자 경험에 직결되므로, 서비스 연동 전 수지님 UT와 연계하여 최소 1회 UT 검증을 필수로 진행합니다.

### 6.4 Week 4 이후 — 영상 변환 + LLM 프롬프트 (별도 일정 확정 예정)

Week 3에서 이월된 숏폼 영상 변환(Grok image-to-video API로 포즈 이미지 → 3~6초 클립)을 우선 진행합니다. 영상 생성은 현재 품질이 부족하므로 제이·미진이 기존에 테스트한 프롬프트를 먼저 공유받아 참고할 것.
