## 1. UT 기반 프롬프트 개선점 및 해결안

Phase 1에서의 UT 결과에 따른 프롬프트 개선점 및 해결안을 정리하였습니다.

### 1-1. 캐스팅

| 후기                           | 해결안                                                                                                                                                                                                                                                          | 기간     |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------ |
| 신규 캐스팅이 어렵다. 선지가 있으면 좋겠다     | **프롬프트 템플릿/구조화 도입**  <br>- 매핑 테이블 방식: `[고정 블록] + [가변 블록(인종)] + [가변 블록(분위기)] + [가변 블록(인상)] + [가변 블록(헤어)] + [공통 다양성 블록]`  <br>- 추가 파라미터 후보(UT): 눈(눈 크기, 쌍꺼풀 유무 등)/코/입<br>  <br>**구현 방식**  <br>- UI에서 프롬프트 선택 → 백엔드에서 영어 프롬프트로 매핑  <br>- 기대효과: 일관성 향상(추가 실험 필요) | week 1 |
| 성별이 랜덤하게 섞여 나온다<br>          | **자유도 제한 + 프롬프트 보강**  <br>- 선택한 성별이 정확히 반영되도록 제약 강화 <br>- “의도한 출력이 나왔는지” 정량 평가 지표 설계                                                                                                                                                                         | week 2 |
| 이미지 생성속도가 느리다                | 프롬프트/파라미터로 속도 제어 가능성 검토 (step 수, guidance scale 등)                                                                                                                                                                                                                                      | week 2 |
| 실존 인물 연상 + 서비스 의도와 다른 이미지 생성 | **안전장치 강화 + 검증 루프**  <br>- 프롬프트 방어 문구/제약 추가<br>- 정성/정량 평가 지표 설계                                                                                                                                                                                              | week 2 |

### 1-2. 엠블럼
| 개선점                  | 해결안                              | 기간     |
| -------------------- | -------------------------------- | ------ |
| 엠블럼 프롬프트에서 예시 부재로 막힘 | **엠블럼 템플릿** 도입(로고 기반, 입력값 예시 포함) | week 3 |

## 2. 콘텐츠 생성

IP 개입을 위한 콘텐츠 생성에 필요한 AI 모델/프롬프트 방안을 정리하였습니다.

| 제안              | 방안                                                                                                     | 기간  |
| --------------- | ------------------------------------------------------------------------------------------------------ | --- |
| 멤버별 포즈, 인사하는 사진 | 이미지 생성 모델로 포즈 (손흔들기/인사/하트/브이 등) 생성                                                                     | 미정  |
| 세계관 AI 초안 제공    | LLM으로 세계관 초안 생성 → 사용자 수정 (UT)                                                                          | 미정  |
| 멤버별 서사 라인 설정    | - 관계 그래프 UI 제안  <br>- 관계 입력을 구조화해 수집 → LLM으로 멤버별 3막 서사(입덕/갈등/성장) 초안 생성 → 숏폼 프롬프트에 자동 반영                | 미정  |
| 짧은 텍스트 또는 숏폼 영상 | - 세계관/서사(관계) 등의 정보를 프롬프트에 자동 주입<br>- LLM으로 스크립트(JSON) 생성 → 이미지 생성 모델로 컷 생성 → 영상 생성 모델로 컷 기반 3~6초 클립 생성 | 미정  |

### 2-1. 기타 의견 (UT)

- 컨셉포토 AI 생성
- 그룹 한 줄 소개
