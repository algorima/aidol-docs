**오전 (엠블럼 프롬프트 구조 설계)**
- 엠블럼 프롬프트 템플릿 구조 설계
  - 블록 후보: `[그룹명/이니셜]` `[심볼 모티프]` `[스타일]` `[컬러 팔레트]` `[배경]` `[텍스처/질감]`
  - 입력값 예시: 그룹명 "NOVA" → 별·우주 모티프 / 그룹명 "BLAZE" → 불꽃·역동 모티프
  - 아이돌 그룹 엠블럼 레퍼런스와 대조하며 스타일 방향 결정
    - 미니멀 로고형 vs 엠블럼 뱃지형 vs 타이포 중심형

**오후~저녁 (실험 실행)**
- 엠블럼 이미지 생성 (~20장, 1K)
  - 모델: `gemini-3.1-flash-image-preview`
  - 스타일 3방향 × 모티프 2~3종 = 6~9개 조합
  - 고도화 포인트:
    - 텍스트 렌더링 품질 (그룹명이 깨지지 않는지)
    - 심볼과 텍스트의 조화
    - 다양한 배경(흰색/검정/투명감) 대응
- 파일명 규칙: `w3_emblem_{스타일}_{모티프}_{시도번호}.png`

### 3/24 (화) 낮 — 포즈 이미지 실험

**포즈 4종 정의**

| 포즈 코드 | 설명 | 레퍼런스 |
|-----------|------|---------|
| POSE_A | 정면 바스트샷 + 눈 마주침 | 앨범 컨셉 포토 정면 |
| POSE_B | 손하트 / 브이 포즈 | 팬사인회 셀카 |
| POSE_C | 칼군무 정지 컷 (역동적) | 무대 캡처 느낌 |
| POSE_D | 일상 캔디드 (카페/거리) | 사복 파파라치컷 느낌 |

- Week 1~2에서 확정된 캐스팅 프롬프트 블록 구조 기반
- 포즈별 5장씩 = ~20장 (1K)
- 포즈 블록을 기존 구조에 어떻게 붙일지 테스트
  - 옵션 1: 별도 `[포즈]` 블록 추가
  - 옵션 2: `[분위기/컨셉]` 블록에 포즈 정보 통합
- 성별 고정 제약 문구 + 안전장치 적용 상태에서 실험
- 파일명 규칙: `w3_pose_{포즈코드}_{시도번호}.png`

## 엠블럼 프롬프트 템플릿 초안 구조

```
[그룹명/텍스트]  →  유저 입력: 그룹명
[심볼 모티프]    →  유저 선택 or 자동 매칭 (그룹 컨셉 기반)
[스타일]         →  미니멀 / 뱃지형 / 타이포 중심
[컬러 팔레트]    →  그룹 대표 컬러 2~3색
[배경]           →  투명 / 단색 / 그라데이션
[텍스처]         →  메탈릭 / 홀로그램 / 매트 / 글리터
```

매핑 테이블 예시:

| 한국어 UI | 영어 프롬프트 값 |
|-----------|----------------|
| 미니멀 | `"minimalist logo, clean lines, modern typography"` |
| 뱃지형 | `"emblem badge, heraldic style, detailed crest"` |
| 타이포 중심 | `"typographic logo, stylized lettering, bold font"` |
