### 3/16 (월) — 성별 고정 실험 + 안전장치 방식 A

**오전 (실험 설계 확인)**
- 실험 변수 정리:
  - 독립변수: 제약 문구 3종
  - 종속변수: 성별 고정율 (생성 이미지 중 의도한 성별이 유지된 비율)
  - 통제변수: 나머지 블록 파라미터는 Week 1 확정값 고정

**오후~저녁 (실험 실행)**
- 제약 문구 후보 3종 × 10장 = 30장 (1K)
  - 후보 A: 명시적 성별 선언 (예: `"This character is strictly male."`)
  - 후보 B: 성별 특징 강화 (예: 체형·턱선·어깨 폭 등 구체적 서술)
  - 후보 C: A+B 결합형
- 각 이미지에 성별 고정 여부 태깅: ✅ 고정 / ❌ 혼재
- 고정율 수치 계산: (고정 이미지 수 / 전체) × 100

**안전장치 방식 A 병행 실험**
- 방식 A: 명시적 제약 문구
  - `"Do not depict any real person, celebrity, or idol."` 등 문구 삽입
  - 문구 위치별 효과 테스트 (프롬프트 앞/뒤/시스템 프롬프트)
  - 10장 생성 → 실존인물 연상 여부 주관 평가

### 3/17 (화) 낮 — 안전장치 방식 B + 4K 확인

- 방식 B: 네거티브 프롬프트 활용
  - Gemini의 negative prompt 지원 범위 확인
  - `negative_prompt: "real person, celebrity, specific idol, photorealistic portrait of existing person"` 등
  - 10장 생성 → 방식 A와 비교

- 상위 성별 고정 조합 4K 확인
  - 고정율 가장 높은 제약 문구로 4K 생성 (~10장)
  - 4K에서도 성별 고정이 유지되는지 확인

**결과 정리**

|-----------|-----------|-----------|------|
| 후보 A     | __% | __% | |
| 후보 B     | __% | __% | |
| 후보 C     | __% | __% | |
