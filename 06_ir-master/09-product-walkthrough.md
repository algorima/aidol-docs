# 제품 워크스루 (Product Walkthrough)

> AIoIA는 AI 아이돌을 캐스팅하고 성장시키는 시뮬레이션 플랫폼입니다.
> 이 문서는 제품의 전체 사용자 흐름과 핵심 기능을 상세히 설명합니다.

## 1. 전체 사용자 여정

```
접속 → 연습생 캐스팅 → 신규 멤버 생성 → 데뷔조 구성 → 그룹 기획
→ 하이라이트 콘텐츠 → 멤버 관계성 → AI 채팅 → 육성 → 다른 그룹 탐색
```

## 2. 핵심 기능 상세

### 2.1 연습생 캐스팅

사용자는 "K-pop 기획사 대표"로서 연습생을 캐스팅합니다.

| 항목 | 내용 |
|------|------|
| 성별 선택 | 남성 / 여성 / 혼성 그룹 |
| 연습생 풀 | 시스템이 제공하는 프리셋 연습생 프로필 |
| 등급 체계 | A / B / C / F (능력치 기반) |
| 능력치 | 육각형 차트 (보컬, 댄스, 랩, 비주얼, 체력, 매력) |
| 경쟁 캐스팅 | 선착순 시스템 — 한 번 캐스팅되면 다른 사용자 선택 불가 |

경쟁 캐스팅은 희소성을 만들어 사용자 간 긴장감과 소유감을 형성합니다.

### 2.2 신규 멤버 생성

원하는 연습생이 없으면 직접 멤버를 생성할 수 있습니다.

| 단계 | 설명 |
|------|------|
| 1. 성별 선택 | 남성 / 여성 |
| 2. MBTI 설정 | 4개 슬라이더 (I↔E, S↔N, T↔F, J↔P) |
| 3. AI 이미지 생성 | MBTI 기반으로 AI가 프로필 이미지 자동 생성 |
| 4. 이름 / 서사 | 사용자가 직접 이름을 짓고 배경 스토리 설정 |

MBTI 설정은 단순 성격 분류가 아니라 채팅 AI의 페르소나를 결정합니다. ENFP로 설정한 멤버는 채팅에서도 밝고 에너지 넘치는 대화를 합니다.

### 2.3 데뷔조 구성 + 포지션 배정

캐스팅/생성된 멤버들로 데뷔 그룹을 구성합니다.

| 포지션 | 설명 |
|--------|------|
| 리더 | 그룹 대표, 멘트 담당 |
| 메인보컬 | 음악의 핵심 |
| 서브보컬 | 하모니, 파트 분배 |
| 메인댄서 | 안무의 중심 |
| 서브댄서 | 포메이션 구성 |
| 메인래퍼 | 랩 파트 |
| 서브래퍼 | 서브 랩 |
| 비주얼 | 그룹의 얼굴 |
| 막내 | 그룹의 에너지 |

K-pop 팬이라면 익숙한 포지션 체계를 그대로 사용하여 진입 장벽을 낮춥니다.

### 2.4 그룹 기획

| 항목 | 내용 |
|------|------|
| 그룹명 | 사용자가 직접 결정 (필수) |
| 엠블럼 | AI가 그룹 콘셉트에 맞게 자동 생성 (512x512 PNG) |
| 콘셉트 | 청량 / 걸크러쉬 / 다크 / 레트로 등 (선택) |
| 인사 문구 | 그룹 공식 인사 (예: "안녕하세요, OOO입니다!") |

### 2.5 하이라이트 콘텐츠

그룹이 완성되면 자동으로 하이라이트 콘텐츠가 생성됩니다.

| 유형 | 설명 |
|------|------|
| 데뷔 발표 | 그룹 결성 알림 콘텐츠 |
| 멤버 소개 | 각 멤버의 프로필 카드 |
| 콘셉트 공개 | 시즌별 콘셉트 비주얼 |
| 피드 형식 | 인스타그램 피드와 유사한 UI로 표시 |

이 콘텐츠는 사용자가 SNS에 공유할 수 있는 형태로, 자연스러운 바이럴 루프를 만듭니다.

### 2.6 멤버 간 관계성 + 친밀도

| 항목 | 내용 |
|------|------|
| 관계 설정 | 멤버 간 케미 (절친, 라이벌, 선후배 등) |
| 친밀도 수치 | 대화 및 활동에 따라 변동 |
| 관계 이벤트 | 특정 친밀도 달성 시 히든 대화/이벤트 해금 |

관계성 시스템은 K-pop 팬덤에서 가장 인기 있는 "케미" 문화를 게임 메카닉으로 구현한 것입니다.

### 2.7 AI 채팅

AIoIA의 핵심 경험 중 하나입니다.

| 항목 | 내용 |
|------|------|
| 구조 | 사용자 ↔ 멤버 1:N (그룹 채팅방) |
| 페르소나 | 멤버 생성 시 설정한 MBTI + 포지션 기반 |
| 대화 기억 | 전체 대화 히스토리 유지 |
| 맥락 이해 | 멤버 간 관계, 그룹 내 역할 반영 |
| 말투 분화 | 리더는 듬직, 막내는 애교, 래퍼는 쿨 등 |

**1대다 채팅 구조**: Character.AI 등 기존 서비스는 1:1 채팅만 가능합니다. AIoIA는 그룹 채팅방에서 여러 멤버가 동시에 대화하며, 멤버 간 상호작용도 발생합니다. 이것이 "팬↔아이돌" 채팅이 아닌 "기획사 대표↔그룹" 채팅인 이유입니다.

### 2.8 육성 시스템

| 항목 | 내용 |
|------|------|
| 연습 | 보컬/댄스/랩 등 항목별 연습 실행 |
| 능력치 상승 | 연습 결과에 따라 육각형 차트 수치 변동 |
| 레벨업 | 누적 연습량 기반 등급 상승 (F → C → B → A → S) |
| 시즌 연동 | 시즌마다 새로운 콘셉트 → 다시 육성 사이클 시작 |

"키운다"는 행위가 핵심입니다. 시간과 노력을 투자한 아이돌은 버릴 수 없어요 — 이것이 리텐션의 본질입니다.

### 2.9 다른 그룹 탐색

| 항목 | 내용 |
|------|------|
| 탐색 피드 | 인스타그램 Explore 페이지와 유사한 UI |
| 그룹 카드 | 그룹명, 멤버 프로필, 콘셉트 미리보기 |
| 팔로우 | 마음에 드는 그룹 팔로우 |
| 조회 전용 | 다른 사용자의 그룹은 조회만 가능, 채팅 불가 |
| CTA | "나도 만들기" 버튼으로 전환 유도 |

다른 사용자의 그룹을 탐색하는 것은 소셜 락인의 핵심입니다. 커뮤니티가 형성되면 네트워크 효과가 발생합니다.

## 3. 기술 스택 상세

### 3.1 AI 대화 엔진

| 구성 요소 | 기술 | 용도 |
|-----------|------|------|
| 대화 생성 | OpenAI GPT API | 멤버 페르소나 기반 텍스트 생성 |
| 프롬프트 구조 | System + Character Profile + Context | 멤버별 말투/성격 분화 |
| 대화 기억 | PostgreSQL + 벡터 검색 | 장기 대화 히스토리 유지 |
| 다중 멤버 | 멀티 에이전트 오케스트레이션 | 1대N 그룹 채팅 동시 처리 |

### 3.2 이미지 생성

| 구성 요소 | 기술 | 용도 |
|-----------|------|------|
| 프로필 이미지 | GPT Image 1.5 / Gemini Pro | MBTI 기반 멤버 외모 생성 |
| 엠블럼 | GPT Image 1.5 | 그룹 콘셉트 기반 로고 생성 |
| 콘셉트 비주얼 | GPT Image 1.5 | 시즌별 콘셉트 이미지 |
| 해상도 | 512x512 PNG | 프로필 / 공유용 |

### 3.3 음성 / 영상 (Phase 2)

| 구성 요소 | 기술 | 용도 |
|-----------|------|------|
| TTS | Fish Audio | 멤버별 목소리 합성 |
| 영상통화 | LiveKit Agents | AI 아이돌과 실시간 영상 대화 |
| 실시간 처리 | WebSocket / Socket.IO | 저지연 양방향 통신 |

### 3.4 인프라

| 구성 요소 | 기술 |
|-----------|------|
| Frontend | Next.js 13, TypeScript, Tailwind CSS, DaisyUI |
| Backend | Python (FastAPI), SQLAlchemy, PostgreSQL |
| 비동기 처리 | Celery, Redis |
| 실시간 | Socket.IO |
| 배포 | Google Cloud, Kubernetes |
| 모바일 | Flutter (예정) |
