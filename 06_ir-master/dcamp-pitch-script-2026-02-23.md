## Slide 4. 문제 — 구조적 공백 (1분)

### 화면
| | AI 생성 툴 | 팬 플랫폼 | AIOIA |
|---|---|---|---|
| 유저 역할 | 입력자 | 소비자 | **설계자** |
| 시간 구조 | 1회성 | 소비 | **시즌 루프** |
| 데이터 축적 | 없음 | 없음 | **있음** |

### 스크립트
AI 생성 비용은 급락했습니다. 이미지, 음성, 영상 다 commodity가 되고 있어요.

그런데 **"생성 이후"가 없습니다**.  
만들고 끝. 이전 행동이 다음 결과에 영향을 주지 않아요.

버추얼 아이돌 시장은 연 18% 성장 중이지만,  
유저에게 **설계 권한**이 없습니다. 기획사가 다 정합니다.

캐릭터 AI 채팅 앱은 1억 명이 쓰지만,  
**관계가 얕습니다**. 롤플레이 중심이에요.

**AI 캐릭터를 반복 설계하고, 결과를 확인하고, 데이터가 쌓이는 구조**가 없습니다.

## Slide 5. 솔루션 — Core Game Loop (1.5분)

### 화면
```
드래프트 → 빌드 → 시뮬레이션 → 시즌 결과 → 반복
(선점 경쟁)  (조합 설계)  (AI 상호작용)  (시장 피드백)
```

### 스크립트
저희 솔루션은 **시즌 기반 게임 루프**입니다.

1. **드래프트**: 한정된 AI 캐릭터 풀에서 선점 경쟁. 선택에 기회비용이 생깁니다.
2. **빌드**: 세계관, 콘셉트, 멤버 조합을 설계합니다. 정답이 없는 설계 공간이에요.
3. **시뮬레이션**: AI 채팅으로 캐릭터 반응을 관찰하고 관계 로그가 쌓입니다.
4. **시즌 결과**: 투표나 평가로 내 설계의 시장 반응을 확인합니다.

그리고 **다음 시즌**.

매 시즌 축적되는 것들:
- 조합 히스토리
- 관계 로그  
- 성장 데이터

이게 **유저 소유 IP**가 됩니다.

시즌이 쌓일수록 전환 비용이 올라가고, 추천이 개인화되고, 네트워크 효과가 강해집니다.

## Slide 6. Why Now — 타이밍 (1분)

### 화면
- AI 생성 비용 급락 → 경쟁은 "생성 후"로 이동
- K-pop 오디션이 "드래프트→육성→투표" 문법을 글로벌에 학습시킴
- 버추얼 아이돌 성장 중, 유저가 만드는 플랫폼 부재

### 스크립트
왜 지금이냐면요.

첫째, **AI 생성이 commodity가 됐습니다**.  
경쟁은 "생성 후 무엇을 하는가"로 이동하고 있어요.

둘째, **K-pop 오디션 프로그램이 문법을 학습시켰습니다**.  
보이즈플래닛 2: 222개국, 3200만 표, 556만 명 참여.  
전 세계가 "캐스팅→육성→투표" 구조를 이해합니다.

셋째, **수요측 플랫폼이 없습니다**.  
버추얼 아이돌은 공급(기획사 IP)만 늘고 있어요.  
유저가 IP를 만드는 플랫폼은 저희가 처음입니다.

## Slide 7. Traction — 검증 (1.5분)

### 화면
| 지표 | 수치 |
|---|---|
| Trial 신청 (이메일) | ~5,000명 |
| 실제 채팅 사용자 | ~500명 |
| 고객 획득 비용 | ~70원/사용자 |
| 1개월 리텐션 | ~5% |
| 마케팅 비용 | ~36,000원 |

### 스크립트
지금까지 검증한 것들입니다.

마케팅 비용 약 3만 6천원으로 **5천 명의 Trial 신청**을 받았습니다.  
AI 생성 아이돌 이미지 10초 영상을 틱톡에 올렸더니 바이럴이 됐어요.

실제 채팅 사용자 약 500명.  
1개월 리텐션은 5%인데, 이건 시뮬레이션 루프가 없었을 때 수치입니다.

스프린트별로 보면:
- **Sprint 1**: 드래프트 선점 심리 확인. 유저들이 경쟁적으로 캐스팅함.
- **Sprint 2**: 빌드 인프라 확립. ERD, API, 홈 구조 완성.
- **Sprint 3**: 지금 채팅 시뮬레이션 + 지불 의향 테스트 중.

핵심 발견 후 **2주 만에 피벗**해서 스프린트에 반영했습니다.

굿즈, 앨범, 팬미팅, 영상통화.

셋째, **캐릭터 AI 채팅에서 잘 되는 것들**.  
버블처럼 아티스트-팬 메시징 구독,  
메시지 수 기반 과금.

투표 참여가 늘면 시즌 패스 가치가 올라가는 구조예요.

# 부록: Q&A 예상 질문

## 1. 비즈니스 모델 — "구독이 어렵지 않나요?"

**답변**:
맞습니다. 그래서 **세 가지 수익원**을 준비했습니다.

1. **게임 과금**: 원신이 캐릭터 뽑기+육성에 연 5조원 벌어요
2. **K-pop 업계 모델**: 아이돌 굿즈, 영통에 팬들이 연간 수십만원 씀
3. **채팅 구독**: Character.ai가 월 145억원 매출, 버블은 아티스트당 월 4,500원

저희는 이 세 가지가 합쳐진 구조입니다.

## 2. 경쟁 — "캐릭터 AI가 이미 1억 유저인데?"

**답변**:
캐릭터 AI는 **롤플레이 채팅**입니다. 남이 만든 캐릭터와 1:1 대화.

저희는:
- 내가 캐스팅하고 조합함 → **소유감**
- 멤버 간 관계 + 그룹 다이나믹 → **복잡한 관계망**
- 시즌 결과로 시장 피드백 → **게임 루프**
- 시즌 축적 → **떠날 수 없는 데이터**

범용 LLM(ChatGPT, Claude)은 도구일 뿐이고,  
기획사가 대표 페르소나를 설정하면 리스크가 커서 빅테크는 못 합니다.

## 3. 리텐션 — "5%면 낮지 않나요?"

**답변**:
맞습니다. 그래서 피벗했습니다.

5%는 **시뮬레이션 루프가 없었을 때** 수치입니다.  
채팅만 있고, 결과 확인이 없었어요.

Sprint 3에서 시뮬레이션 + 시즌 결과를 추가하고 있고,  
Phase 2 목표는 **40% 시즌 리텐션**입니다.

## 4. 타겟 — "K-pop 팬이 타겟인가요?"

**답변**:
K-pop 팬이긴 한데, 더 정확히는 **크리에이터**입니다.

유튜버, 틱톡커처럼 콘텐츠 만드는 사람인데,  
자기 얼굴 안 내밀고 **AI 아이돌을 가상 인플루언서로 내세우고 싶은 사람들**이에요.

"AI 아이돌 공모전"에 참여하고 싶은 사람들.  
지금 틱톡에서 AI 아이돌 이미지만 올려도 바이럴 됩니다.

## 5. 팀 — "왜 이 팀이어야 하나요?"

**답변**:
세 가지입니다.

1. **속도**: UT 발견 → 피벗 → 스프린트 반영까지 2주
2. **팬덤 이해**: 20년 K-pop 팬덤 + BL/팬픽 서브컬처까지 아는 여성 창업자
3. **기술**: 스캐터랩 AI 챗봇(이루다) + 네이버 스트리밍 경험

팬덤 UX는 직접 겪어보지 않으면 설계할 수 없습니다.
