# 데이터 B2B 전략 (Data & B2B Strategy)

> 김민철님 피드백: "처음부터 세팅을 해두면 나중에 데이터 판매할 때 더 설득력 있고 비싼 가격으로 판매할 수 있다"

## 1. 전략 개요

AIoIA는 크리에이터가 AI 아이돌을 만드는 과정에서 **K-pop 팬 선호도 데이터**를 자연스럽게 수집합니다.
이 데이터는 엔터테인먼트사의 캐스팅, 콘셉트 기획, 마케팅 전략에 직접적인 가치를 제공합니다.

**핵심**: B2C 서비스 사용 데이터 → B2B 데이터 상품으로 전환

## 2. 수집 데이터 종류

### 2.1 크리에이터 행동 데이터

| 데이터 | 수집 시점 | 엔터사 가치 |
|--------|----------|------------|
| 선호 외모 | 멤버 생성 시 MBTI/이미지 선택 | 캐스팅 전략 |
| 선호 성격 | MBTI 설정, 채팅 패턴 | 컨셉 기획 |
| 선호 콘셉트 | 그룹 콘셉트 선택 (청량/걸크/다크/레트로) | 컴백 전략 |
| 포지션 선호 | 포지션 배정 패턴 | 그룹 구성 전략 |
| 관계성 선호 | 멤버 간 관계 설정 패턴 | 유닛/콘텐츠 기획 |
| 의상/헤어 선호 | 유료 아이템 구매 패턴 | MD/스타일링 전략 |

### 2.2 팬 소비 데이터

| 데이터 | 수집 시점 | 엔터사 가치 |
|--------|----------|------------|
| 지출 패턴 | 과금 금액, 빈도, 아이템 유형 | 가격 전략, 굿즈 기획 |
| 채팅 패턴 | 대화 주제, 빈도, 길이 | 팬 소통 전략 |
| 시즌 반응 | 시즌별 활동량 변화 | 컴백 타이밍 |
| 공유 패턴 | SNS 공유 콘텐츠 유형 | 마케팅 전략 |

### 2.3 트렌드 데이터

| 데이터 | 분석 방법 | 엔터사 가치 |
|--------|----------|------------|
| 외모 트렌드 변화 | 월별/분기별 생성 데이터 추이 | 캐스팅 방향 예측 |
| 콘셉트 트렌드 | 인기 콘셉트 변화 추이 | 컴백 전략 수립 |
| 지역별 선호 차이 | 국가/지역별 데이터 비교 | 글로벌 전략 |
| 연령별 선호 차이 | 세그먼트별 패턴 비교 | 타깃 마케팅 |

## 3. 회원가입 시 세그먼트 분류 설계

김민철님 조언: "처음부터 세팅을 해둬야 나중에 비싼 가격으로 팔 수 있다"

### 3.1 온보딩 시 수집 (자연스럽게)

| 질문 | 선택지 | 목적 |
|------|--------|------|
| "나는..." | 크리에이터 / 팬 / 둘 다 | 사용자 유형 분류 |
| "좋아하는 그룹은?" | 최애 그룹 선택 (복수) | 취향 프로필 |
| "관심 있는 콘셉트는?" | 청량/걸크/다크/레트로/Y2K | 콘셉트 선호 |
| "나이대는?" | 10대/20대/30대/40대+ | 세그먼트 |
| "어디서 왔나요?" | 국가 선택 | 지역 데이터 |

### 3.2 사용 중 수집 (행동 기반)

별도 질문 없이, 사용자의 행동 패턴에서 자동 수집:
- 멤버 생성 시: 외모/성격 선호
- 과금 시: 지출 패턴
- 채팅 시: 관심사, 대화 스타일
- 공유 시: 콘텐츠 선호

## 4. 엔터테인먼트사 가치 제안

### 4.1 하이브 인터뷰 기반 인사이트

하이브(HYBE) 관계자 인터뷰에서 확인된 니즈:

| 니즈 | 현재 상황 | AIoIA 제공 가치 |
|------|-----------|----------------|
| 캐스팅 데이터 | 감에 의존, 오디션 결과만 | "팬이 선호하는 외모/성격" 데이터 |
| 컴백 전략 | 트렌드 분석팀 운영 (고비용) | 실시간 콘셉트 트렌드 리포트 |
| 팬 세그먼트 | 자체 앱 (Weverse) 데이터만 | 크로스 플랫폼 팬 행동 데이터 |
| 실패 리스크 | 컴백 실패 시 수십억 손실 | 사전 검증 데이터로 리스크 감소 |

핵심 인용: "이런 데이터가 있으면 컴백 기획 실패 리스크를 크게 줄일 수 있다"

## 5. 데이터 상품 설계

### 5.1 트렌드 리포트 (정기 발행)

| 항목 | 내용 |
|------|------|
| 주기 | 월간/분기별 |
| 내용 | 외모 트렌드, 콘셉트 인기 변화, 지역별 선호, 연령별 차이 |
| 형태 | PDF 리포트 + 대시보드 접근 |
| 가격 | 월 300~500만원 (구독) |
| 타깃 | 기획사 A&R, 마케팅팀 |

### 5.2 세그먼트 분석 (맞춤형)

| 항목 | 내용 |
|------|------|
| 주기 | 프로젝트 기반 |
| 내용 | 특정 아티스트/그룹 타깃 팬의 선호 분석 |
| 형태 | 맞춤 리포트 + 컨설팅 |
| 가격 | 프로젝트당 1,000~3,000만원 |
| 타깃 | 기획사 전략팀 |

### 5.3 실시간 대시보드 (SaaS)

| 항목 | 내용 |
|------|------|
| 주기 | 실시간 |
| 내용 | 트렌드 모니터링, 팬 반응 실시간 추적, 컴백 시 반응 분석 |
| 형태 | 웹 대시보드 (SaaS) |
| 가격 | 월 1,000~2,000만원 (연간 계약) |
| 타깃 | 대형 기획사 |

## 6. 프라이버시 / 법적 고려사항

| 항목 | 대응 |
|------|------|
| 개인정보보호법 (한국) | 수집 시 동의, 익명화 처리 후 B2B 제공 |
| GDPR (유럽) | 명시적 동의, 데이터 삭제 권리 보장 |
| CCPA (미국) | 옵트아웃 권리 제공 |
| 약관 명시 | "서비스 개선 및 산업 트렌드 분석 목적 데이터 활용" 명시 |
| 익명화 | 개인 식별 불가능한 집계 데이터만 B2B 제공 |
| 데이터 최소 수집 | 서비스에 필요한 최소한의 데이터만 수집 |

**원칙**: 개인 데이터는 절대 B2B로 판매하지 않음. 집계/익명화된 트렌드 데이터만 상품화.

## 7. B2B 매출 프로젝션

### 7.1 전제 조건

| 항목 | 가정 |
|------|------|
| B2B 시작 시점 | M12 (MAU 20,000+ 달성 후) |
| 초기 고객 | 2~3개 기획사 (파일럿) |
| 데이터 가치 | MAU에 비례 (더 많은 데이터 = 더 높은 신뢰도) |

### 7.2 매출 프로젝션

| 시기 | 고객 수 | 상품 | 연간 매출 |
|------|---------|------|-----------|
| M12 | 2개사 | 트렌드 리포트 (파일럿) | 1,200만원 |
| M18 | 5개사 | 리포트 + 세그먼트 분석 | 5,000만원 |
| M24 | 10개사 | 리포트 + 대시보드 | 2억원 |
| M36 | 15개사 | 풀 패키지 | 5억원 |

### 7.3 장기 비전

MAU가 100만 이상이 되면:
- 데이터 정확도와 신뢰도가 업계 최고 수준
- 기획사뿐 아니라 광고주, 브랜드, 미디어도 고객
- B2B 매출이 B2C 매출을 초과할 가능성

## 8. 실행 로드맵

| 시기 | 액션 | 목표 |
|------|------|------|
| M1 (현재) | 온보딩 세그먼트 분류 설계 + 데이터 수집 시작 | 데이터 파이프라인 구축 |
| M3 | 내부 트렌드 대시보드 구축 | 자체 분석 역량 확보 |
| M6 | 첫 번째 트렌드 리포트 발행 (내부용) | 데이터 품질 검증 |
| M9 | 기획사 2곳에 파일럿 제안 | B2B 검증 |
| M12 | 유료 리포트 출시 | B2B 매출 시작 |

**김민철님 조언 핵심**: "나중에가 아니라 지금부터 데이터 수집 구조를 세팅해야 한다."
온보딩 세그먼트 분류와 행동 데이터 수집은 M1부터 시작합니다.
