# 솔루션 (Solution)

## 핵심 가치 제안

> **"AI 아이돌을 캐스팅하고 키우는 시뮬레이션 게임"**

사용자가 프로듀서가 되어 AI 아이돌 그룹을 드래프트하고, 빌드하고, 시뮬레이션하고, 시즌 결과를 확인하는 반복 루프. 소비가 아니라 **설계와 육성**이 핵심 경험이다.

### 제품명
- **AIdol**: AI 아이돌 시뮬레이션 플랫폼

## 1. 핵심 게임 루프 (Core Game Loop)

```
드래프트 (Draft)
  → 연습생 풀에서 AI 아이돌을 캐스팅. 등급, 능력치, MBTI, 포지션을 보고 전략적으로 선택
  → 경쟁 캐스팅: 한 번 선택된 연습생은 다른 사용자가 선택 불가

빌드 (Build)
  → 그룹 콘셉트 설정, 포지션 배정, 관계 구조 설계
  → 멤버 조합에 따라 케미/시너지가 달라짐

시뮬레이션 (Simulate)
  → AI 아이돌과 1:1 대화, 그룹 내 관계 진행
  → 멤버 성장, 이벤트 발생, 팬 반응 시뮬레이션

시즌 결과 (Season Result)
  → 시즌 종료 시 성과 평가 (인기, 실력 성장, 팬 수)
  → 랭킹, 히스토리 기록

반복 (Next Season)
  → 새 시즌에 신규 연습생 드래프트, 멤버 교체/영입
  → 이전 시즌 데이터 축적 → 그룹 IP 진화
```

## 2. MVP 테스트 결과 (Sprint 1~2, 8명 심층 UT)

### 2.1 AHA Moment — 사용자가 핵심 가치를 체감한 순간

**AHA 1: AI 이미지가 상상과 일치하는 순간 (8명 중 6명)**

> **"it made the image like exactly what I was thinking."**
> — Sogand, 20대, 캐나다 K-POP 팬 (Sprint 1 UT)

AI 이미지 생성 결과가 기대를 초과할 때 강한 감정 반응 발생. 한 피실험자는 결과물 확인 직후 자세를 고쳐앉으며 다음 멤버 생성에 몰입.

**AHA 2: 프로듀서 정체성 동일시 (8명 중 2명, B그룹)**

> **"지금 내가 지금 바로 민희진이다."**
> — U3 장쏘피, 35세, 파인아트 작가 (Sprint 1 UT)

캐스팅 완료 후 자발 발화. 아이돌 팬이 아닌 창작 성향 사용자에게서 "프로듀서" 정체성이 활성화됨.

**AHA 3: 예상을 초과한 몰입 (8명 중 4명)**

> **"오랜만에 몰입해 본 것 같아요."**
> — U8 김현조, 20대, 세븐틴 팬 9년차 (Sprint 1 UT)

### 2.2 정량 지표

| 지표 | 수치 | 의미 |
|------|------|------|
| 그룹 생성 완주율 | **83%+ (5/6)** | 플로우 장벽 낮음 |

### 2.3 세그먼트별 반응 차이

| 구분 | A그룹 (팬 성향, 5명) | B그룹 (창작 성향, 2명) |
|------|---------------------|---------------------|
| 핵심 욕구 | 외모 완성도 높은 프리셋 + 선점 경쟁 | 직접 설정하는 창작 경험 + 육성 |
| AHA Moment | 선점 경쟁 긴장감 | 프로듀서 정체성 동일시 |
| WTP | 100~500원 찍먹 | 구독 가능성 (사용 후 판단) |
| 이탈 위험 | 프리셋 품질 미달 시 즉시 이탈 | 텍스트 세계관 설정에서 이탈 |

### 2.4 위험 신호 및 대응

| 위험 | 근거 | 대응 |
|------|------|------|
| 프리셋 외모 기대 미달 | 8명 중 7명 불만 | 프리셋 품질 향상 + 신규 생성 강화 |
| 관계성 가치 전달 실패 | "이걸 꼭 해야 하나?" → 설명 후 완주 | 진입 화면에서 목적 안내 UX 추가 |
| 1회성 체험 종료 | 성장/시즌 없으면 재방문 동인 부재 | Phase 2 시즌 루프 설계 |
| 캐스팅만으로 결제 전환 부족 | U8: "키우는 느낌이 부족" → 결제 의향 없음 | 육성 기능 = 수익화 전제 조건 |

### 2.5 결론

> **"실제 아이돌을 대체하지 않는다. 팬이 실험하고 확장할 수 있는 팬덤 시뮬레이션 공간이다."**

핵심 차별점은 "AI 친구"가 아니라 **"내가 캐스팅하고 키우는 시뮬레이션"**이라는 카테고리. 사용자가 이 카테고리를 직접 정의했다.

## 3. 시즌 축적 → 플랫폼화

### 3.1 시즌 축적

| 축적 데이터 | 가치 |
|------------|------|
| 조합 히스토리 | 어떤 멤버 조합이 성공하는지 학습 |
| 관계 로그 | 사용자-아이돌 간 대화/이벤트 히스토리 |
| 성장 데이터 | 시즌별 성과, 전략 패턴 |
| 사용자 소유 IP | 사용자가 만든 그룹 = 사용자의 자산 |

### 3.2 플랫폼 효과

```
시즌 축적 → 전환비용 상승 (내 그룹 히스토리를 버릴 수 없다)
         → 추천 개인화 (사용자 취향 학습 → 더 정확한 연습생 추천)
         → 네트워크 효과 (사용자 그룹 간 경쟁/비교 → 커뮤니티 형성)
```

## 4. 현재 MVP 기능

### 4.1 아이돌 생성

#### 그룹 기획
- 그룹명 (필수), 콘셉트 (선택)
- 엠블럼 이미지 AI 생성 (512x512 PNG)
- 이메일 수집 (그룹의 다음 소식을 받을 이메일)

#### 기본 제공 연습생 캐스팅
- 시스템 제공 연습생 풀
- 성별 필터 (남성/여성/혼성)
- 등급 (A/B/C/F), 능력치 (육각형 차트)
- 경쟁 캐스팅: 선착순, 한 번 캐스팅되면 다른 사용자 선택 불가

#### 신규 멤버 생성
- 성별 선택, MBTI 4개 슬라이더
- 이름 입력, 프로필 이미지 AI 생성

#### 포지션 배정
- 리더, 메인보컬, 서브보컬, 메인댄서, 서브댄서, 메인래퍼, 서브래퍼, 비주얼, 막내

### 4.2 채팅
- 사용자 <> 멤버 1:1 대화
- 생성 시 입력한 성격 기반 페르소나 응답
- 대화 히스토리 유지

### 4.3 관리/공유
- 내 그룹 목록 관리
- URL 공유 (조회만, 대화 불가)
- "나도 만들기" CTA

## 6. Why Now

### 6.1 AI 생성 비용 보편화
- 이미지 생성, 음성 합성, 텍스트 생성 비용이 급격히 하락
- 2년 전에는 불가능했던 "사용자별 고유 AI 캐릭터 생성"이 경제적으로 가능해짐

### 6.2 K-POP 드래프트 문화 글로벌 학습 완료
- 프로듀스 101 → 보이즈 플래닛 → 글로벌 팬들이 "투표하고 뽑는" 문화를 이미 체득
- "국민 프로듀서"라는 역할에 대한 사전 학습이 끝남

### 6.3 수요 측 IP 플랫폼 부재
- 공급자(기획사) 주도 IP만 존재
- 사용자가 만들고 소유하는 IP 플랫폼은 아직 없음
- PLAVE($32.5M 매출)가 버추얼 아이돌 수요를 검증했으나, 여전히 기획사 소유 모델
