# D-Camp 오피스아워 피드백 디브리핑

- 일시: 2026-02-24
- 참석자: 제이(회의 진행), 소연, 수지, 제니
- 관련: D-Camp 오피스아워 발표 후 팀 내부 피드백 논의

> **TL;DR**
> D-Camp 오피스아워 피드백 디브리핑: "실재감 부족"이 핵심 이슈이며, 콘텐츠 자동 생성 / AI 영상 / 타 유저 노출(투표/팔로우)을 Phase 2 핵심 과제로 논의

## 1. 요약

### 1.1 D-Camp 멘토 피드백 공유 및 공감

- 멘토 핵심 피드백: 서비스가 "프린세스 메이커"처럼 보인다. AI 아이돌의 실재감이 부족하다.
- IP로서의 가치가 안 드러남 ("먼 미래 같고 정말 돈을 벌 수 있을까"라는 인상)
- 채팅만으로는 부족하고, AI 아이돌이 진짜 존재하는 것처럼 보여야 한다는 의견
- 채팅 기능을 실제로 시연하지 못한 것이 아쉬운 점 (스프린트 3 미완성)

### 1.2 "현타 지점" 분석 - 팀 전원 공감

- 스프린트 1 완료 후 "완성되었습니다" 프로필 공유 페이지에서 현타가 온다
- 사용자 테스트에서도 동일: 신나게 만들고 나서 "그래서 뭐?" 반응
- 핵심 문제: "인형놀이에 불과하구나"라는 느낌을 어떻게 극복할 것인가

### 1.3 해결 아이디어 3가지 논의

**1.3.1 콘텐츠 자동 생성 및 플로우 변경 (소연 제안)**

- 현재: 아이돌 그룹 생성 완료 후 일주일 뒤 수동으로 콘텐츠 생성
- 제안: 생성 완료 직후 해당 그룹의 콘텐츠가 바로 하나라도 나오게 함
- 스켈레톤 UI로 로딩 시간 처리 가능
- 가설: 콘텐츠가 연결되면 실재감과 리텐션이 생길 것이다

**1.3.2 AI 영상/인사 영상 생성 (제니 제안)**

- 멤버들의 단체 사진 또는 인사 영상 생성
- AI 영상 기술이 급격히 발전 중 - 인스타그램 AI 인플루언서 "돈홍이" 사례 (팔로워 약 18~20만, 매우 자연스러운 AI 영상)
- 영상이 주는 리얼함이 이미지보다 훨씬 강한 임팩트
- Phase 2에서 최소 한 명이라도 영상으로 보여주는 플로우 검토
- 기술적 가능성 테스트 필요

**1.3.3 타 유저 존재감 노출 (제이 제안)**

- 다른 유저들이 만든 아이돌 그룹을 볼 수 있는 목록/랭킹 페이지
- 간단한 투표 기능 추가 -> "나만의 인형놀이"가 아닌 시장 안에 있는 느낌
- 다른 사람 그룹의 아이돌 얼굴을 크게 볼 수 있게 -> 보는 재미
- 사용자 인터뷰에서 "자기가 만든 아이돌 사진을 크게 볼 곳이 없다"는 피드백 반영
- 팔로우/좋아요 기능 Phase 2에서 활성화 검토 (로그인 전제)

### 1.4 데모 시 발견된 버그

- 아이돌 목록에 40개 콘텐츠 중 4개만 표시 (빈 아이돌 그룹 ID가 DB에 6개 쌓여 있어서)
- 탭 전환 시 콘텐츠 미표시 문제
- 자잘한 에러 다수 존재

### 1.5 기타

- 서비스 이름 아직 미확정 (확정 필요)
- 에너지 레벨 체크 아이스브레이킹: 당분간 하지 않기로 결정

## 2. 결정 사항

- D-Camp 멘토 피드백 "실재감 부족"에 팀 전원 공감 -> Phase 2 핵심 과제로 설정
- 해결 방향 3가지를 아이디어 후보로 두고 기술적 가능성 검토 후 결정:
  - (1) 콘텐츠 자동 생성 플로우
  - (2) AI 영상(인사 영상) 생성
  - (3) 타 유저 노출 + 투표/팔로우 기능
- 회의 시작 시 에너지 레벨 체크 활동은 당분간 하지 않기로 함
