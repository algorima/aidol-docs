## AI-Native 조직

9명의 소규모 팀이지만, AI를 10번째 팀원으로 활용하여 대기업 수준의 생산성을 낸다. Day 1부터 모든 프로세스에 AI를 내재화했다.

### AI 활용 영역

| 영역 | 도구 | 방식 |
|------|------|------|
| **코드 작성** | Claude Code, Cursor | CLAUDE.md + 규칙 파일로 코드 스타일 학습, 실시간 피드백 |
| **코드 리뷰** | Copilot + Gemini + Claude | 3중 AI 리뷰 통과 후 사람 리뷰. PR마다 자동 실행 |
| **문서 검토** | Gemini Code Assist | YC 파트너 관점으로 IR/지원서 자동 리뷰 (17개 연속 PR 리뷰) |
| **사용자 분석** | PostHog + AI | 세션 리플레이를 AI에게 제공, 이탈 지점/UX 개선점 자동 도출 |
| **버그 대응** | Sentry + Slack + Claude | 오류 발생 → Slack 알림 → AI 즉시 원인 분석 + 해결책 제안 |
| **업무 관리** | Notion/Asana/Figma MCP | Claude가 MCP로 노션 재무 데이터, Figma 디자인, Asana 업무에 직접 접근 |

### 실측 생산성

| 작업 | AI 활용 | 수동 (추정) |
|------|---------|------------|
| iMessage 채널 연결 (설치+설정+테스트) | 5분 | 35분 |
| 문서 레포 구조 개편 + PR (폴더 6개, README 7개) | 3분 | 75분 |
| Slack 공지 작성 + 전송 | 1분 | 10분 |
| **합계** | **10분** | **130분** |

**생산성 향상: 13배**. 사람이 늘지 않아도 AI가 함께 자라면서 생산성이 복리로 증가하는 구조.

### 핵심 구조

회사의 모든 지식(코드, 문서, 재무, IR, PRD)이 GitHub에서 버전 관리된다. AI 설정 파일(CLAUDE.md, .gemini/styleguide.md, copilot-instructions.md)이 시간이 지날수록 축적되어, AI가 우리 방식을 더 잘 이해하게 된다.
