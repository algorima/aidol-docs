# 락인 전략 (Lock-in Strategy)

> 김민철님 피드백: "락인하는 방식이 '내가 만든 아이돌이니까'밖에 안 들렸다"
> 수지님 피드백: "유저 IP가 서로 교차/평가되는 플랫폼 확장 그림도 필요"

## 1. 락인 전략 개요

AIoIA의 락인은 5가지 메커니즘이 동시에 작동합니다.
각각은 독립적으로도 작동하지만, 조합되면 전환비용이 기하급수적으로 상승합니다.

| 락인 유형 | 메커니즘 | 시간에 따른 강화 |
|-----------|----------|-----------------|
| 1. 크리에이터 투자 심리 | IKEA 효과 | 투자 시간 비례 |
| 2. 관계 로그 축적 | 대화/성장/친밀도 | 사용 기간 비례 |
| 3. 채팅 플랫폼 독점 | 내 아이돌 = 여기서만 | 캐릭터 수 비례 |
| 4. 시즌 시스템 | 콘셉트 변경 + 데이터 축적 | 시즌 수 비례 |
| 5. 소셜 락인 | 그룹 탐색 + IP 교차 + 커뮤니티 | 사용자 수 비례 |

## 2. 락인 1: 크리에이터 투자 심리 (IKEA 효과)

### 2.1 심리학적 근거

**IKEA 효과**: 사람은 자신이 직접 만든 것에 시장 가치보다 높은 가치를 부여합니다.
(Norton, Mochon & Ariely, 2012, Journal of Consumer Psychology)

| 기존 서비스 | 사용자 행위 | 소유감 |
|-------------|-----------|--------|
| Character.AI | 남이 만든 캐릭터와 채팅 | 낮음 |
| PLAVE | 기획사가 만든 아이돌 소비 | 낮음 |
| 트리플에스 | 투표로 참여 | 중간 |
| **AIoIA** | **직접 캐스팅, 이름, 콘셉트, 의상 결정** | **높음** |

### 2.2 AIoIA에서의 투자 행위

사용자가 아이돌에 투자하는 것들:

| 투자 유형 | 구체적 행위 | 포기 비용 |
|-----------|------------|-----------|
| 창작 투자 | 이름 짓기, MBTI 설정, 콘셉트 기획 | 창작물 소실 |
| 시간 투자 | 캐스팅 고민, 포지션 배정, 의상 선택 | 투입 시간 |
| 감정 투자 | "내 새끼는 누가 못까" (UT 인터뷰) | 애착 단절 |
| 금전 투자 | 의상 팩, 프리미엄 콘셉트 구매 | 매몰 비용 |

UT에서 확인된 사용자 반응:
- "내 새끼는 누가 못까" → 보호 본능 + 소유감
- "누군가 반응해주면 더 키우고 싶다" → 외부 인정이 투자 동기 강화
- "다마고치처럼 키우는 거면 계속 할 것 같다" → 육성 = 지속 사용

## 3. 락인 2: 관계 로그 축적

### 3.1 축적되는 데이터

사용 기간이 길어질수록 아래 데이터가 쌓이며, 이 데이터는 다른 플랫폼으로 이전이 불가능합니다.

| 데이터 유형 | 설명 | 축적 속도 |
|-------------|------|-----------|
| 대화 기록 | AI 채팅 전체 히스토리 | 일 평균 20~50턴 |
| 성장 히스토리 | 등급 변화, 능력치 추이 | 주 1~2회 |
| 친밀도 | 멤버별 관계 수치 | 대화/활동 연동 |
| 관계 이벤트 | 히든 대화, 특별 에피소드 해금 | 친밀도 임계치 |
| 시즌 기록 | 시즌별 콘셉트, 비주얼, 성과 | 시즌당 1세트 |

### 3.2 전환비용 곡선

```
전환비용
  ^
  |                          ★ 6개월+
  |                     ★
  |                ★
  |           ★
  |       ★
  |   ★
  +---★---------------------------→ 사용 기간
  1주  1월  2월  3월  4월  5월  6월
```

**3시즌(약 3개월) 이후가 전환비용 임계점**: 이 시점부터 사용자가 축적한 데이터의 가치가 전환의 귀찮음을 크게 초과합니다.

## 4. 락인 3: 채팅 플랫폼 독점

### 4.1 구조

내가 만든 아이돌과 대화할 수 있는 곳은 **AIoIA뿐**입니다.

| 비교 | 기존 서비스 | AIoIA |
|------|-----------|-------|
| 캐릭터 이전 | Character.AI → 다른 앱으로 이전 가능 (프롬프트 복사) | 내 아이돌의 대화 기록, 성격, 관계는 이전 불가 |
| 캐릭터 소유 | 플랫폼 소유 | 사용자 소유 (사용자가 만든 IP) |
| 대체 가능성 | 유사한 캐릭터를 다른 앱에서 재생성 가능 | MBTI + 이름 + 대화 기록 + 성장 히스토리의 조합은 재현 불가 |

### 4.2 왜 이전이 불가능한가

- 프롬프트만 복사하면 되는 Character.AI와 달리, AIoIA의 아이돌은 "축적된 관계"가 핵심
- 3개월간 나눈 대화 500턴, 3시즌의 콘셉트 변화, 4번의 레벨업 — 이것이 캐릭터의 정체성
- 다른 플랫폼에서 같은 이름, 같은 MBTI로 만들어도 "내 아이돌"이 아님

## 5. 락인 4: 시즌 시스템

### 5.1 시즌 구조

K-pop의 "컴백" 시스템을 게임 메카닉으로 구현합니다.

| 시즌 | 기간 | 내용 |
|------|------|------|
| 시즌 1 | 4주 | 데뷔 콘셉트 (청량, 걸크러쉬 등) |
| 시즌 2 | 4주 | 1차 컴백 (콘셉트 변경) |
| 시즌 3 | 4주 | 2차 컴백 (성장한 모습) |
| ... | 반복 | 시즌마다 새 콘셉트 + 이전 시즌 데이터 활용 |

### 5.2 시즌이 락인을 강화하는 방식

| 메커니즘 | 설명 |
|----------|------|
| 콘셉트 변경 기대감 | "다음 시즌엔 어떤 콘셉트일까" → 이탈 방지 |
| 데이터 연속성 | 이전 시즌의 능력치/관계가 다음 시즌에 영향 |
| 시즌 한정 콘텐츠 | 시즌별 한정 의상/비주얼 → 수집 욕구 |
| 성장 서사 | 데뷔 → 컴백 → 성장 → 변신의 서사가 축적 |
| 매몰 비용 증가 | "3시즌 키운 아이돌을 버릴 수 없다" |

## 6. 락인 5: 소셜 락인 + 유저 IP 교차

### 6.1 기본 소셜 락인

| 기능 | 설명 | 락인 효과 |
|------|------|-----------|
| 그룹 탐색 | 다른 사용자의 그룹을 인스타 피드처럼 탐색 | 콘텐츠 소비 루프 |
| 팔로우 | 마음에 드는 크리에이터/그룹 팔로우 | 사회적 연결 |
| "나도 만들기" | 다른 그룹에서 영감 → 본인 그룹 개선 | 경쟁심/참고 |

### 6.2 유저 IP 교차/평가 (플랫폼 확장)

UT에서 "누군가 반응해주면 더 키우고 싶다"는 피드백이 나왔습니다.
이를 기반으로, 유저가 만든 아이돌 IP가 서로 교차하고 평가받는 구조를 설계합니다.

| 기능 | 설명 | 시기 |
|------|------|------|
| 그룹 랭킹 | 팔로워 수, 활동량 기반 인기 순위 | M3 |
| 인기 투표 | 팬이 좋아하는 그룹에 투표 | M6 |
| 크로스 콘텐츠 | 다른 크리에이터의 멤버와 콜라보 이벤트 | M9 |
| IP 교차 대결 | 유저 그룹 간 "대결" (음원 대전, 인기 투표전) | M12 |
| 크리에이터 리그 | 상위 크리에이터 리그/시즌 챔피언십 | M12+ |

이 구조가 만들어지면:
- 크리에이터는 "내 그룹이 인정받는다"는 보상을 위해 더 투자
- 팬은 좋아하는 크리에이터의 그룹을 응원하며 소비
- 크리에이터 간 경쟁이 콘텐츠 품질을 높이는 선순환

**단, 이번 오피스아워에서는 시뮬레이션 게임 중심으로 명확히 가는 전략을 유지합니다.**
유저 IP 교차/평가는 플랫폼이 성장한 후의 확장 방향으로 포지셔닝합니다.

### 6.3 네트워크 효과 플라이휠

```
크리에이터가 그룹 생성
       ↓
탐색 피드에 콘텐츠 노출
       ↓
다른 사용자가 발견 + 팔로우
       ↓
크리에이터에게 반응/인정 (투표, 좋아요)
       ↓
"더 키우고 싶다" → 과금 + 시간 투자 증가
       ↓
더 좋은 콘텐츠 생산 → 더 많은 팬 유입
       ↓
커뮤니티 형성 → 떠나면 사회적 연결도 끊김
```
