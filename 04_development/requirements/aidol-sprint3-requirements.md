# AIdol Sprint 3 - 프론트엔드 요구사항

## 1. Companion 조회 API (기존)

> `GET /companions` — 이미 존재하는 API

| 항목        | 필드명            | 타입     | 설명                   |
| ----------- | ----------------- | -------- | ---------------------- |
| 멤버 ID     | `id`              | `string` | 멤버 고유 식별자       |
| 멤버 이름   | `name`            | `string` | 멤버 표시 이름         |
| 프로필 사진 | `profileImageUrl` | `string` | 멤버 프로필 이미지 URL |

- **사용처**: 채팅방 목록 페이지 (멤버 정보 표시), 채팅방 페이지 (대상 멤버 헤더)

<br/>

## 2. 채팅방 목록 조회 API (신규)

> `GET /chatrooms` — **신규 API**

| 항목                   | 필드명              | 타입                 | 설명                              |
| ---------------------- | ------------------- | -------------------- | --------------------------------- |
| 채팅방 ID              | `id`                | `string`             | 채팅방 고유 식별자                |
| **대상 멤버 ID**       | **`companionId`**   | **`string`**         | **채팅 대상 companion ID**        |
| **마지막 메시지**      | **`lastMessage`**   | **`string \| null`** | **가장 최근 메시지 내용**         |
| **마지막 메시지 시간** | **`lastMessageAt`** | **`string \| null`** | **마지막 메시지 시각 (ISO 8601)** |

- **사용처**: 채팅방 목록 페이지
- 랜덤 1명에 대해서만 채팅방 자동 생성
- 나머지 멤버는 **잠금 상태**로 표시 (채팅방 생성 불가)

<br/>

## 3. 채팅 메시지 조회 API (기존)

> `GET /chatrooms/:chatroomId/messages` — 이미 존재하는 API

| 항목             | 필드명            | 타입                      | 설명                        |
| ---------------- | ----------------- | ------------------------- | --------------------------- |
| 메시지 ID        | `id`              | `string`                  | 메시지 고유 식별자          |
| 보낸 사람        | `senderType`      | `"user"` \| `"companion"` | 메시지 발신자 유형          |
| **보낸 멤버 ID** | **`companionId`** | **`string` \| `null`**    | **메시지를 보낸 멤버의 ID** |
| 메시지 내용      | `content`         | `string`                  | 메시지 본문                 |
| 보낸 시간        | `createdAt`       | `string (ISO 8601)`       | 메시지 전송 시각            |

- **사용처**: 채팅방 페이지
