# UT 분석 → IR/사업계획서 활용 가이드

- 목적: ut-transcript-analyzer 에이전트가 생성한 분석 보고서를 IR 자료와 사업계획서에 활용하는 방법
- 작성일: 2026-03-06

## 1. 전체 흐름

```
녹취록 (raw)
    |
    v
ut-transcript-analyzer 에이전트
    |
    v
분석 보고서 (01_research/user-interviews/{sprint}-analysis/)
    |
    +--→ [팀 내부] PRD/기능명세서 업데이트
    |
    +--→ [IR] 06_ir-master/ 섹션별 업데이트
    |
    +--→ [지원사업] 사업계획서 근거 작성
```

## 2. 분석 보고서에서 IR로 가져오는 법

### 2.1 태그별 → IR 섹션 매핑

에이전트가 생성한 보고서의 태그를 IR 섹션에 직접 매핑합니다.

| 보고서 태그 | IR 섹션 | 활용 방법 |
|------------|---------|----------|
| `[PAIN]` 발화 | `01-problem.md` "팬의 고충" | 사용자가 직접 말한 문제 인용 |
| `[AHA]` 발화 | `02-solution.md` 검증 근거 | "이 기능에 이렇게 반응했다" |
| `[WTP]` 발화 | `05-business-model.md` | 결제 의향, 가격 민감도 근거 |
| `[COMP]` 발화 | `01-problem.md` 기존 서비스 한계 | "버블은 ~인데" 비교 발화 |
| `[NEED]` 발화 | `02-solution.md` 로드맵 근거 | 사용자가 원하는 기능 = 제품 방향 검증 |
| 가설 판정 결과 | `03-traction.md` | PMF 신호로 변환 |
| `[BEHAV]` 관찰 | `03-traction.md` | 정량 지표 (체류 시간, 전환율 등) |

### 2.2 정량 지표 집계 방법

여러 보고서의 데이터를 종합할 때:

| 지표 | 집계 방법 | IR 반영 형식 |
|------|----------|-------------|
| 캐스팅 몰입도 | 각 보고서 `[AHA]` 발화 중 점수 응답 평균 | "N명 중 M명이 4점 이상" |
| 결제 의향 | `[WTP]` 발화에서 긍정/부정 카운트 | "N명 중 M명 결제 의향 (X%)" |
| NPS | 추천 의향 점수 평균 | "NPS X점" |
| PMF ("매우 실망") | 설문 응답 비율 | "N명 중 M명 (X%)" |

### 2.3 인용문 선정 기준

IR Deck에 넣을 사용자 인용문 고를 때:

| 우선순위 | 기준 | 예시 |
|---------|------|------|
| 1순위 | 감정이 강하고 구체적 | "오랜만에 몰입해 본 것 같아요" |
| 2순위 | 경쟁사 대비 차별점 언급 | "버블은 일방적인데 이건 내가 만든 느낌" |
| 3순위 | 결제/추천 의향이 명확 | "친구한테 바로 보여줬어요" |
| 피하기 | 맥락 없이 짧은 감탄 | "오 좋다" (근거 불충분) |

## 3. 사업계획서에서 활용하는 법

### 3.1 초기창업패키지 등 정부 지원사업

| 사업계획서 항목 | 분석 보고서에서 가져올 것 | 작성 형식 |
|---------------|------------------------|----------|
| 고객 발굴/검증 | 누적 UT 인원, 세그먼트 분포, 주요 발견 | "총 N명 대상 M회 스프린트 UT 진행" |
| 제품 차별성 | `[COMP]` 발화 + 가설 검증 결과 | "기존 서비스 대비 ~ 반응 확인" |
| 사업화 전략 | `[WTP]` 집계 + 전환율 | "타겟 세그먼트 X% 결제 의향" |
| 시장 진입 | 세그먼트별 판정 비교 | "A그룹에서 가장 강한 반응" |
| 성장 전략 | 추천/공유 관련 발화 | "N명 중 M명 자발적 공유 행동" |

### 3.2 작성 원칙

- **숫자로 말하기**: "사용자들이 좋아했다" X → "10명 중 7명이 4점 이상 몰입도 응답" O
- **발화 직접 인용**: 심사위원이 읽었을 때 생생한 사용자 목소리
- **세그먼트 명시**: "모든 사용자"가 아닌 어떤 유형의 사용자인지
- **Before/After**: 스프린트 간 개선 데이터가 있으면 반드시 포함

### 3.3 템플릿 문장

보고서 데이터를 채워 넣으면 되는 형식:

**고객 검증:**
> "{총 N}명 대상 {M}회 스프린트 사용자 테스트 진행. {세그먼트}에서 {핵심 지표} {수치} 확인."

**차별화:**
> "기존 {경쟁사} 사용 경험이 있는 {N}명 중 {n}명이 '{차별점}'에 긍정 반응."

**시장 반응:**
> "{세그먼트} 사용자의 {%}가 {행동/의향} 확인. 특히 '{대표 발화}' 등 구체적 피드백 확보."

## 4. 종합 분석 시 체크리스트

여러 보고서를 모아서 IR/사업계획서에 반영할 때:

- [ ] 각 보고서의 태그별 발화를 스프레드시트에 모으기
- [ ] 세그먼트별로 판정 결과 집계 (검증/부분 검증/기각/미검증)
- [ ] `03-traction.md`의 수치 업데이트 (누적 UT 인원, PMF 지표)
- [ ] IR Deck Best Quote 3개 선정 (Pain/Aha/WTP 각 1개)
- [ ] GO/STOP 판단 근거 업데이트 (`aidol-prd.md` 2.3절 기준)
