## 0. 한 줄 핵심

## 2. 가장 인상 깊은 사용자 발화 (Raw Voice)

- "지금 아이돌도 서사 있는 팀들이 더 인기가 많은 것 같아서, 그 소설을 직접 만들기까지 하면 더 애정이 커지지 않을까 생각했어요" (세계관·서사 = 애착 강화 논리를 사용자 스스로 제시)
- "약간 게임 요소, 컴퓨터 키우는 것처럼" (원하는 인터랙션 방식을 직접 표현)
- "만드는 거에 관여를 하고 그 이후부터 안 하면 점점 관심이 떨어질 것 같아요. 그래서 로그인을 해서 지속적으로 뭔가 주도적인 행동을 해야 꾸준히 들여다볼 것 같습니다"
- "여기도 이제 정해놓지 않고 원하는 만큼 고를 수 있는 것도 있고, 첫 화면에서부터 마음에 들지 않으면 배제시키고 할 수 있어서 궁금한 사람만 눌러오면 되니까" (자유 선택 구조가 "내가 선택했다"는 느낌의 근거)
- "이 관계성이 어느 정도 100퍼센트인지 선택하는 칸이 있었는데, 그거를 직접 선택하지 않고 어떤 선택지를 고르면 얘네가 얼마큼의 퍼센트 관계성이 나오는지를 랜덤으로 하면 좋을 것 같아요" (케미 결과를 예측할 수 없는 랜덤 요소 선호)
- "만드는 거에 관여를 하고 그 이후부터 안 하면 점점 관심이 떨어질 것 같아요" (이메일 단방향 콘텐츠만으로는 재방문 동기 부족)
- "선택 못할 거면 차라리 안 보는 것도 괜찮은 것 같아요" (다른 그룹 캐스팅 이미지 공개에 대해 — 보고 나서 질투심 유발 우려)

## 3. 행동 기반 관찰

### 3.1 멤버 캐스팅

- 선택 기준: 비주얼 + 능력치(보컬/댄스/랩) 균형. 비주얼이 낮으면 "아이돌 자질이 부족하다"고 판단하여 배제
- 포지션 6개 → 정확히 6명 캐스팅. 포지션 수에 맞춰 인원을 맞추는 행동 패턴 관찰
- 원하는 포지션(담당)에 능력치 높고 비주얼도 좋은 멤버를 찾지 못하자 캠트(센터/비주얼)를 따로 뽑고 나머지를 서브로 배정: "담당을 뽑으려고 하니 비주얼이 없어서 그냥 캠트로 새 뽑고 하나는 서브에 벌로 넣었어요"
- Sprint 1 대비 비주얼 개선을 자발적으로 언급: "저번에 비주얼이 됐는데 이번에는 훨씬 낫다"

### 3.2 그룹 엠블럼 생성

- 한국어 입력 시 이미지 생성 3~4회 반복 실패
- 영어 입력으로 전환 후 성공 ("나왔어요 나왔어" 발화로 확인)
- 인터뷰어가 현장에서 버그로 리포팅하겠다고 언급 → 인터뷰 흐름이 중단됨

### 3.3 관계성(케미) 설정

- 비주얼 첫인상("고양이 같은 사람", "강아지 같은 사람") + 코멘트의 성격 정보("센 느낌", "순둥순둥한 느낌")를 종합해 관계성 지정
- 어울리지 않을 것 같은 멤버끼리 엮는 방식 선택: "제일 안 어울릴 것 같은 비주얼끼리"
- 포지션 배정 시 능력치가 화면에 표시되지 않아 기억에 의존 → 어려움 호소: "기억할 수가 없죠"

### 3.4 콘텐츠 반응 (홈 탭)

- 텍스트 채팅 형태 콘텐츠: 웹소설 미독자라 직접 읽을 의향 낮음
- "수다 타임" 형태(가벼운 티키타카)는 볼 의향 있음: "키키카카만 하는 거는 채팅창으로만 봐도 충분하니까"
- 마피아/MBTI 토론 형태의 게임성 콘텐츠는 텍스트로도 괜찮다고 판단: "시각적인 게 별로 필요하지 않은 거면은 괜찮을 것 같아요"
- 썸네일의 궁금증 유발 5/5, AI 느낌 인지하나 거부감은 없음: "AI 같긴 한데... 난 괜찮은 것 같아"
- "콘텐츠 다음 이야기가 궁금하다" 5/5

## 4. 가설 검증 결과

### 가설 1: 사용자가 직접 아이돌 멤버를 생성하는 과정에서 몰입감을 느낀다

**결과: 있음**

- "내가 선택했다"는 느낌 5/5, "고민이 되었다" 4/5
- 이유: 원하는 수 만큼 자유롭게 고를 수 있고 마음에 안 들면 배제할 수 있는 구조 → 선택권이 몰입감의 핵심 동인
- Sprint 1 대비 비주얼 개선에 긍정 반응 → 이미지 품질 개선이 몰입에 직접 기여
- 한계: 엠블럼 생성 오류(3~4회 실패)가 경험을 중단시켜 몰입이 끊기는 구간 발생

### 가설 2: 멤버 간 관계성(케미) 설정이 그룹에 대한 애착을 높인다

**결과: 약함 → 잠재력 있음**

- "세계관이 애정을 강화한다"에 긍정 반응. "서사 있는 팀이 인기 많은 것처럼 직접 만들면 더 애정이 커질 것" → 논리적 이해는 있음
- 그러나 현재 관계성 UI에서 능력치가 표시되지 않아 "기억할 수가 없다"는 실질적 장벽 발생
- "랜덤으로 결과가 나오면 좋겠다" → 현재 수치 직접 입력 방식보다 선택지 기반 랜덤 결과를 선호
- "게임 요소, 컴퓨터 키우는 것처럼" → 관계성이 정적 설정이 아닌 동적 게임 루프여야 효과적

### 가설 3: 콘텐츠(하이라이트/숏폼)가 다른 그룹에 대한 관심을 유도한다

**결과: 모호 (콘텐츠 형식 의존)**

- "콘텐츠 다음 이야기가 궁금하다" 5/5, 썸네일 궁금증 유발 5/5
- 그러나 텍스트 채팅 형태는 직접 읽을 의향 낮음 (웹소설 비독자)
- "수다 타임"처럼 가벼운 대화 형식, 또는 마피아/토론처럼 게임성 있는 텍스트는 수용 가능하다고 판단
- 형식에 따라 다른 그룹 콘텐츠 소비 여부가 갈림 → 콘텐츠 주제에 맞는 형식 매칭이 핵심

## 6. 다음 스프린트 수정 제안 (3개 이하)

1. **엠블럼 이미지 생성 오류 수정 (버그)**: 한국어 입력 시 이미지 생성 실패가 3~4회 반복됨. 사용자가 영어로 전환해야 성공 → 언어 전처리 또는 오류 안내 메시지 개선. 몰입이 중단되는 가장 직접적인 장벽.

2. **관계성 설정 UX 개선 — 선택지 기반 랜덤 결과 방식 검토**: "선택지를 고르면 관계성이 랜덤으로 결정되는 방식"을 사용자가 직접 제안. 관계성 설정 시 상대 멤버 능력치가 보이지 않아 기억에 의존하는 문제도 함께 해결 필요.

3. **콘텐츠 재참여 루프 설계 검토**: "만드는 데 관여하지 않으면 관심이 떨어진다"는 발화는 이메일 기반 단방향 수신만으로는 재방문이 어렵다는 신호. 로그인 기반 능동적 참여(케미 추가, 세계관 결정) 루프가 리텐션의 핵심임을 확인.

## 7. 남은 질문 / 리스크

- **게임 루프 설계 우선순위**: "컴퓨터 키우기" 비유처럼 지속적 상호작용이 없으면 이탈 가능. 현재 스프린트에서 이 루프를 만들지 못하면 이탈률 측정이 어려움.
- **케미 기능의 결과물이 아직 경험되지 않음**: 관계성을 설정했지만 콘텐츠에 어떻게 반영되는지 확인하지 못함 → "기대가 된다"는 발화는 있지만 실제 만족감으로 이어질지 미검증.
- **비주얼 품질 기대값 상승 지속**: Sprint 1보다 개선됐다고 느꼈으나, 직접 생성 이미지와 기본 템플릿 이미지의 품질 격차가 지속되면 생성 기능 사용 의향 하락 가능. (김현조 UT에서도 유사 패턴 관찰)
- **공유 의향 조건부**: "쓰는 친구들한테는 공유하겠다" → 친구 네트워크에 기존 사용자가 없으면 공유 고리가 형성되지 않음. 초기 확산 전략에서 네트워크 씨앗 사용자 확보 필요.
