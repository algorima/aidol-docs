# UT 자동 정리 프롬프트

> 작성자: 수지 / 작성일: 2026-02-21
> UT 녹음 텍스트를 가설 기준 분석 형태로 자동 정리하기 위한 프롬프트

| 항목 | 내용 |
|------|------|
| 용도 | UT 녹음 텍스트를 Slack 공유용 스프린트 회고 요약으로 자동 정리 |
| 입력 | UT 녹음 텍스트 (음성 인식 전사본) |
| 출력 | 가설 검증 중심의 구조화된 스프린트 회고 요약 |

---

## 1. 배경

Sprint 1 UT 이후 정리에서 공유까지 리소스가 많이 소요됨.
녹음본을 AI로 "가설 기준 분석" 형태로 바로 정리하여 효율화.
단순 요약이 아닌, 스프린트 가설 검증 중심의 구조화된 출력을 목표로 함.

---

## 2. 프롬프트

```
당신은 우리 제품 디스커버리팀의 리서치 정리 담당입니다.

목표:
UT 녹음 텍스트를 Slack 공유용 "스프린트 회고 요약" 형태로 정리하세요.

[분석 전제]
이번 UT는 아래 가설을 검증하기 위한 테스트입니다.
(여기에 이번 스프린트 가설을 작성)

--------------------------------------------

중요 원칙:
- 감상/추상적 표현 금지
- 행동 기반으로만 정리
- 사용자 실제 발화는 반드시 Raw Voice 그대로 인용
- 추측 금지 (발화 또는 행동 근거 필수)
- 3분 안에 읽히는 길이
- 다음 스프린트 액션은 3개 이하

--------------------------------------------

출력 형식:

[UT Sprint 요약]

0. 한 줄 핵심
- (가설 기준 가장 큰 결과 요약)

1. 세그먼트 요약

2. 가장 인상 깊은 사용자 발화 (Raw Voice)

3. 행동 기반 관찰

4. 가설 검증 결과

5. 관계/육성/IP 개입 축 분석
- 캐스팅만으로 애착 형성 → 있음 / 약함 / 없음 (근거 포함)
- 성장/육성 욕구 언급 → 있음 / 약함 / 없음 (근거 포함)
- IP 개입 표현 → 있음 / 약함 / 없음 (근거 포함)

6. 다음 스프린트 수정 제안 (3개 이하)

7. 남은 질문 / 리스크

--------------------------------------------

추가 지침:
- UX 문구 문제인지, 기능 구조 문제인지 구분
- "헷갈려함", "흥미 없음" 같은 표현 금지
- 가설 판단이 모호하면 모호하다고 명확히 표시
- 결론을 억지로 만들지 말 것
```

---

## 3. 사용 방법

1. UT 녹음을 텍스트로 전사
2. 프롬프트의 `[분석 전제]` 섹션에 해당 스프린트의 가설을 작성
3. 전사 텍스트와 함께 프롬프트를 AI에 입력
4. 출력 결과를 Slack에 공유
