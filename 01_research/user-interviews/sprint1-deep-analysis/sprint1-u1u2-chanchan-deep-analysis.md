## 3. Behavioral Sequence Analysis

사용자의 행동 흐름을 시간순으로 매핑.

### 3.1 은찬 (U1) 행동 시퀀스

```
설문 완료 (07:00)
→ 링크 접속 (07:08) "너무 힘들어" (부담 표현)
→ 연습생 목록 탐색 → 계약완료 다수 확인 → "고를 애가 없네" (선택지 부족 인지)
→ 걸그룹 탭에서 남자 발견 → 혼성 가능성 질문
→ 신규 멤버 생성 기능 발견 (10:15)
→ 포지션 설정 → "메인 보컬" 배정 시도
→ 그룹 이름 "3리 보이즈" 입력
→ 서사 작성 시도 → "아비로" 이름, 서사 작성
→ 이미지 생성 실패 (13:50~14:08) → "너무 힘들다"
→ 프로필 공유 (17:40) → 완료
→ 설문 응답: 고민 4점 ("처음 하는 거니까")
→ 심즈 비유 → 외모 기준 강조 → 선점 아쉬움 강하게 표현
→ 바이럴/댓글/경쟁 아이디어 다수 제안
→ 마무리: "무조건 잘 생긴 애들로만 꽉 채워놔야 돼요"
```

### 3.2 예찬 (U2) 행동 시퀀스

```
플랫폼 설명 요청 (02:50~03:04) "언니 플랫폼이 뭔지 그냥 설명해줘"
→ 연습생 캐스팅 시작 (07:35)
→ 계약완료 다수 → "구리에밖에 없잖아. 어머 안 하고 싶어" (실망)
→ 실존 아이돌 닮은 꼴 인식 (08:03) "보면 알지 그렇게 생겼잖아"
→ 능력치 확인 → "이게 왜 C등급이야?" (등급 시스템 의아)
→ 포지션 설정 시 능력치 정보 재확인 요청 (10:42) "육각형을 나 기억 안 나는데"
→ 그룹 이름 "엠블럼" 입력 → 로고 품질 불만 (12:19~14:27)
→ 대충 포지션 배정 (16:14) "난 그냥 대충 했어"
→ 프로필 공유 (14:52)
→ 설문 응답: 고민 4점
→ 외모 부족 피드백, 서사/데뷔 구분 제안
→ 콘텐츠 구조 (채팅 캡처 → SNS 공유) 아이디어
→ 수익모델: 찍먹은 저가/무료, 프리미엄은 단계적
```

### 3.3 공통 행동 패턴

1. **탐색 시 계약완료 연습생 먼저 인지** → 선택지 부족 불만 선행
2. **외모로 빠르게 필터링** → 능력치는 2차 기준
3. **포지션 설정 시 정보 부족**으로 의미 있는 선택 불가 → "대충" 처리
4. **로고/이미지 생성 품질 불만** → 기대 대비 결과물 괴리
5. **설문 이후 자유 토론에서 폭발적 아이디어 제안** → 핵심 인사이트는 태스크 이후에 등장

## 10. Mental Model Gap Analysis

사용자의 멘탈 모델과 서비스의 실제 모델 간 차이 분석.

### Gap 1: 연습생 캐스팅 ≠ 데뷔

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 연습생 캐스팅 + 그룹 생성 = 완성 | 연습생 캐스팅 = 계약. 데뷔는 별도 단계 |
| 생성 후 바로 콘텐츠 제공 | 연습 → 데뷔곡 → 앨범 → 활동의 단계적 서사 기대 |

**영향**: 사용자는 "숙소 스토리" 단계에서 아직 연습생이라 콘텐츠에 몰입하기 어려움. "데뷔를 해야 된다고"라는 반복 발화.

### Gap 2: 프리셋 = 선택지 vs 프리셋 = 걸러야 할 것

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 175명의 다양한 프리셋 제공 | "쟁쟁한 애들로 20~30명"이면 충분. 나머지는 노이즈 |
| 다양성 = 가치 | 품질 기준 미달 = 방해물 |

**영향**: 양보다 질. 매력 없는 프리셋은 서비스 전체 인상을 깎음.

### Gap 3: 포지션 설정 = 의미 있는 선택 vs 정보 없는 작업

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 포지션을 유저가 자유롭게 배정 | 능력치(육각형)를 보고 합리적으로 배정해야 의미 있음 |
| 창의적 자유 제공 | 정보 없이 선택 = "대충" = 의미 없음 |

**영향**: 포지션 설정 UX에서 능력치 정보가 동시에 보여야 함. 현재는 별도 화면이라 접근 불가.

### Gap 4: 서비스 = 크리에이터 플랫폼 vs 서비스 = 게임

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 아이돌 창작/소비 플랫폼 | "이거 그냥 게임 아니야?" (심즈, 동물의 숲, 미연시) |
| 유저 = 크리에이터 | 유저 = 프로듀서 겸 팬 (경쟁/소유/키움) |

**영향**: 게임적 요소(경쟁, 보상, 레벨업, 이벤트)가 없으면 동기 유지 어려움.

## 11. Norman's 7 Stages of Action Analysis

### 11.1 Goal (목표)
- **유저 목표**: 마음에 드는 연습생을 골라 나만의 아이돌 그룹을 만들기
- **시스템 목표**: 연습생 캐스팅 → 포지션 설정 → 그룹명/로고 → 완성

### 11.2 Execution Side (실행)

| 단계 | 분석 |
|------|------|
| Plan (계획) | 연습생 목록에서 마음에 드는 외모를 고르겠다 → 명확 |
| Specify (구체화) | 최소 3명 선택 필요 → 인지됨. 단, 어떤 기준으로 고를지 정보 부족 (능력치 비교 어려움) |
| Perform (수행) | 연습생 클릭 → 캐스팅 → 포지션 배정 → 이름/로고 생성. **문제**: 포지션 배정 시 능력치 참조 불가, 로고 생성 실패 |

### 11.3 Evaluation Side (평가)

| 단계 | 분석 |
|------|------|
| Perceive (인지) | 결과물(로고, 그룹 카드) 확인 가능 → OK |
| Interpret (해석) | 로고가 "풍경 사진 같은 것"으로 나옴 → "이게 로고여야 되는데" 해석 혼란 |
| Compare (비교) | 기대(세련된 아이돌 로고) vs 결과(랜덤 이미지) → 큰 괴리 |

### 11.4 Gulf of Execution (실행의 간극)
- 포지션 설정 시 능력치 정보가 같은 화면에 없어서 합리적 선택 불가
- 이미지/로고 생성 품질이 의도와 다르게 출력

### 11.5 Gulf of Evaluation (평가의 간극)
- 로고 생성 결과가 "로고"로 인식되지 않음
- "계약 완료"의 의미가 직관적이지만, 전체 선택지 대비 비율이 높아 실망

## 12. Norman's 3 Levels of Design

### 12.1 Visceral Level (본능적 반응)

- **부정**: 프리셋 외모 품질 → "이게 너무 현실적인", "못 생겼었고"
- **부정**: 로고 이미지 품질 → "왜 이상한 사진", "풍경 사진 같은 것"
- **긍정**: 잘 생긴 프리셋을 발견했을 때 → 빠른 선택 반응
- **시사점**: Visceral 레벨에서 실패하면 이후 단계 진입 자체가 막힘. 외모 품질은 최소 기준.

### 12.2 Behavioral Level (행동적 반응)

- **불만**: 포지션 설정 시 능력치 정보 접근 불가 → "대충" 처리
- **불만**: 이미지 생성 실패 → 재시도 부담
- **긍정**: 캐스팅 자체의 선택 행위 → "꽤 고민이 되었다" (4점)
- **시사점**: 핵심 태스크(캐스팅)의 기본 플로우는 이해 가능하나, 보조 태스크(포지션, 로고)에서 마찰 발생.

### 12.3 Reflective Level (반성적 반응)

- **강한 긍정**: 선점 경쟁에서의 아쉬움 → 소유감 형성 증거
- **강한 긍정**: "심즈 하는 것처럼" → 서비스를 통한 자기 표현 및 놀이로 의미 부여
- **강한 긍정**: 바이럴/팬덤 경쟁 상상 → 서비스의 사회적 가치 인식
- **부정**: "게임 아니야?" → 서비스 정체성에 대한 혼란
- **시사점**: Reflective 레벨에서 가장 강한 반응. 경쟁/소유/바이럴에 대한 상상이 핵심 동기. 단, 서비스가 게임인지 플랫폼인지 포지셔닝이 필요.

## 15. Cross-Analysis Summary

### 15.1 핵심 인사이트 5선

**1. 외모 품질은 Visceral Level의 게이트키퍼**
- 외모 기준 미달 → 나머지 모든 경험 무의미
- "무조건 잘 생긴 애들로만 꽉 채워놔야 돼요"는 서비스 존폐 조건
- 액션: 쟁쟁한 외모의 프리셋 최소 20~30명 확보, 글로벌/혼혈 다양성 추가

**2. 애착의 진짜 드라이버는 "캐스팅 행위"가 아니라 "경쟁/선점"**
- H1은 조건부 지지: 캐스팅 자체보다 "다른 유저가 가져가면 아쉬운" 경쟁 구조가 애착의 원천
- 계약 만료, 장바구니, 타임어택 등 게임화 요소가 핵심
- 액션: 선점/경쟁 메커니즘 설계가 애착 전략의 우선순위

**3. "데뷔"라는 전환점이 없으면 콘텐츠 소비 동기 부재**
- 연습생 → 데뷔를 명확히 분리해야 사용자 멘탈 모델에 부합
- 연습생 단계: 캐스팅/성장 콘텐츠
- 데뷔 이후: 숙소, 콘서트, 앨범, 여행 콘텐츠
- 액션: 데뷔 이벤트를 명시적 전환점으로 설계

**4. 프로듀서 ≠ 모든 것을 직접 만드는 사람**
- 유저는 핵심 선택(멤버, 콘셉트)만 하고, 나머지는 자동 생성 + 랜덤 이벤트를 기대
- "너무 내가 설정해 버리면 나 혼자 노는 느낌"
- 액션: 유저 선택 3~4개 + 시스템 자동 생성의 밸런스. 랜덤 이벤트(악플 터짐, 캐스팅 제의 등)로 서사 활성화

**5. 2차 콘텐츠 생산 = 바이럴 성장 엔진**
- 채팅 캡처 → SNS 공유, 팬덤 간 댓글 경쟁 → 자연 바이럴
- 사용자가 직접 바이럴 전략을 제안할 만큼 강한 동기
- 액션: 캡처/공유 기능 우선 구현, 댓글 시스템 설계, 그룹 간 비교 콘텐츠 자동 생성
