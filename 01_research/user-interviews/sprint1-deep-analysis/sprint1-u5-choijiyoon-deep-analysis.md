## 3. Behavioral Sequence Analysis

### 3.1 최지윤 (U5) 행동 시퀀스

```
접속 (00:00) → "잘생겨서 다행이네요" (초기 반응)
→ 그룹 유형 선택: 걸그룹 (01:07)
→ 연습생 탐색 → 계약 완료 다수 확인 → "A등급은 다 팔려나간 거네" (선점 인지)
→ F등급이지만 비주얼로 1명 캐스팅 (01:29) → 등급보다 외모 우선
→ 전략 전환: "포지션이 필요해요" → 능력치(육각형) 확인 (02:58)
→ 6각형 연습생 캐스팅 → "확실한 실력 하나 필요해" (03:27)
→ 남은 연습생 상태 판단 → "비주얼로 가야겠다" (03:40)
→ 댄스 능력치 확인 → 4번째 캐스팅 (03:47~04:19)
→ "보컬이 좀 있어야 되는데" → 보컬 1명 추가 (04:19~04:34)
→ 5명 캐스팅 완료 (05:04) → "홀수가 걸그룹이 하기는 좋을 것 같아서"
→ 포지션 배정 시작 (05:20) → "육각형 화면 있었으면 좋겠어" (정보 부족 피드백)
→ 포지션 중복 불가 발견 (06:29) → "왜 뽑았는지가 모르겠네" (06:52)
→ 그룹명/엠블럼 설정 (07:27~09:43) → "촌스러워요" (엠블럼 불만)
→ 프롬프트 입력 방식 혼란 (09:49~10:34) → "단어를 치면 되는 건 줄 알아서"
→ 결과 확인 (11:08) → "끝이네 하고 나갔을 것 같아"
→ 이메일 입력 화면 (12:10~13:37) → 거부 ("매일 잘 열어보지도 않을뿐더러")
→ 아쉬움 3점 응답 (21:23)
→ 키우기 서비스 관심 표명 (23:42)
→ 커뮤니티 경쟁 시나리오에서 소유 심리 발현 (25:25~28:19)
→ 마무리 (30:54)
```

### 3.2 행동 패턴 요약

1. **외모 우선, 등급 무시**: F등급이어도 비주얼이 마음에 들면 즉시 캐스팅
2. **전략적 포지션 밸런스 시도**: 능력치를 확인하고 보컬/댄스/비주얼을 고르게 배치하려는 의도
3. **정보 부족으로 전략 좌절**: 포지션 배정 시 능력치 재참조 불가 → "왜 뽑았는지 모르겠네"
4. **선택지 부족 인지 → 타협**: "남은 애들 중에 그나마 고른 것" → 퍼즐 끼워 맞추기
5. **이메일 거부**: 캐스팅 체험으로 끝, 후속 콘텐츠에 대한 기대 없음
6. **키우기/성장 시나리오에서 관심 전환**: 단순 캐스팅보다 발전/경쟁 관찰에 흥미

## 6. Thematic Analysis

녹취록에서 도출된 핵심 테마를 코드 기반으로 분류.

### Theme 1: 선택지 부족이 애착 형성의 구조적 장벽

- 관련 코드: `선택지부족`, `선점경쟁-인지`, `타협감`, `퍼즐끼워맞추기`, `선택지부족=애착부재`
- 근거: VC-03, VC-10
- 요약: "A등급은 다 팔려나간 거네"라는 인식이 캐스팅 초기부터 형성. 남은 연습생 중에서 "그나마 고른" 느낌이 들면 선택이 아닌 소거법이 되고, 애착은 형성되지 않음. "선택권이 많았으면 조금 더 애착이 갔을 수도 있는데"라는 명시적 발화가 이를 확인.

### Theme 2: 외모 우선, 등급은 부차적

- 관련 코드: `선택기준-외모우선`, `등급무시-비주얼`, `프리셋불만-외모`, `아이돌기준-미달`
- 근거: VC-01, VC-04
- 요약: F등급이어도 비주얼이 마음에 들면 캐스팅. 반면 비주얼이 "당황스러운" 연습생은 등급과 무관하게 즉시 배제. 아이돌 서비스에서 외모는 선택의 필수 조건.

### Theme 3: 정보 부족이 선택의 의미를 파괴

- 관련 코드: `UX-정보접근성`, `정보부족-포지션`, `기억부재`, `선택-무의미화`
- 근거: VC-05, VC-06, VC-07
- 요약: 포지션 배정 시 캐스팅 화면의 능력치(육각형)를 참조할 수 없어 "왜 뽑았는지 모르겠네"라는 상태에 빠짐. 선택에 충분한 정보가 주어지지 않으면 해당 선택은 의미 없는 작업이 됨.

### Theme 4: 일회성 체험으로 끝나는 현재 서비스

- 관련 코드: `이메일-거부`, `관심부재`, `일회성체험`, `채팅-무관심`, `갑자기-튀어나옴`
- 근거: VC-09, VC-17
- 요약: 캐스팅 후 "해봤구나 끝"이라는 인식. 채팅 콘텐츠에 대해서도 "갑자기 내 인생에 튀어나온 애들"이라 관심 부재. 현재 서비스는 유저와 아이돌 간 관계를 형성하는 구조가 아님.

### Theme 5: 키우기/성장이 진짜 동기

- 관련 코드: `배정+교체-선호`, `키우기-기대`, `내부경쟁-재미`, `성장관찰-동기`, `키우기>채팅`
- 근거: VC-13, VC-18
- 요약: 처음부터 만드는 것보다 "있던 애를 빼고 능력치를 향상하는 걸 보고 비주얼이 발전되는 걸 보는" 것이 더 재미있다고 명시. D등급을 A등급으로 키우고, 멤버를 교체하고, 내부 경쟁을 관찰하는 것이 핵심 동기.

### Theme 6: 잠재적 소유 심리와 방어 본능

- 관련 코드: `소유심리-방어`, `내새끼-논리`, `타유저-비교`, `잠재적애착`
- 근거: VC-19, VC-20
- 요약: 직접 질문하면 "사라져도 상관없다"(VC-15)고 하지만, 다른 유저가 깔 때는 "내 새끼는 내가 까도 내가 까지 남이 갈 수는 없는 거야"라는 강한 방어 본능 발현. 애착은 의식적으로는 부정하지만 경쟁 맥락에서 잠재적으로 작동.

## 10. Mental Model Gap Analysis

사용자의 멘탈 모델과 서비스의 실제 모델 간 차이 분석.

### Gap 1: 캐스팅 = 적극적 선택 vs 캐스팅 = 소거법

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 다양한 연습생 중 원하는 멤버를 골라 캐스팅 | "남은 애들 중에 그나마 고른 것" = 퍼즐 끼워 맞추기 |
| 선택 행위 자체에 가치 | 선택지가 충분해야 선택에 의미가 있음 |

**영향**: 선택지 부족은 단순 불편이 아니라 애착 형성의 구조적 장벽. "선택권이 많았으면 애착이 갔을 수도 있는데"라는 명시적 인과관계 발화.

### Gap 2: 캐스팅 완료 = 서비스 시작 vs 캐스팅 완료 = 체험 종료

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 캐스팅 후 이메일로 후속 콘텐츠 연결 | "캐스팅을 해봤구나 끝" → 일회성 체험 |
| 이메일 = 기대 행동 | 이메일 = 불필요한 개인정보 제공 |

**영향**: 현재 캐스팅에서 후속 콘텐츠로의 연결 고리가 없음. 유저가 왜 돌아와야 하는지에 대한 명확한 가치 제안이 부재.

### Gap 3: 유저 = 창작자 (모든 것을 만듦) vs 유저 = 감독 (배정된 것을 조율)

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 유저가 처음부터 멤버 선택, 포지션, 이름, 로고 모두 결정 | 이미 구성된 그룹을 받고, 멤버 교체/트레이닝/능력치 향상 |
| 백지에서 시작 | 기본 세팅에서 미세 조정 |

**영향**: "결정 장애"를 가진 유저에게 처음부터 모든 것을 결정하게 하는 것은 부담. 기본 배정 + 교체/발전이라는 가벼운 진입 방식이 이 유저 유형에 더 적합.

### Gap 4: 소유 = 만들었으니까 vs 소유 = 남이 건드리면 안 되니까

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 캐스팅/생성 = 소유 | 소유감은 생성이 아니라 경쟁/방어 맥락에서 발생 |
| 만든 것에 대한 애착 | "사라져도 상관없다"이면서 "남이 까면 싸우고 싶다" |

**영향**: 애착/소유감의 트리거가 "만드는 행위"가 아니라 "타인과의 관계(경쟁/비교/방어)"임. 커뮤니티/소셜 기능이 없으면 소유 심리가 활성화되지 않음.

## 11. Norman's 7 Stages of Action Analysis

### 11.1 Goal (목표)
- **유저 목표**: 마음에 드는 연습생을 골라 걸그룹을 만들기
- **시스템 목표**: 연습생 캐스팅 → 포지션 설정 → 그룹명/로고 → 완성

### 11.2 Execution Side (실행)

| 단계 | 분석 |
|------|------|
| Plan (계획) | 걸그룹, 홀수 인원, 비주얼+실력 밸런스 → 명확한 전략 수립 |
| Specify (구체화) | 외모 1순위로 필터링, 부족한 포지션(보컬, 댄스) 전략적 보충 → 구체화 성공 |
| Perform (수행) | 캐스팅 자체는 원활. **문제**: 포지션 배정 시 능력치 정보 미노출 → "왜 뽑았는지 모르겠네". 엠블럼 프롬프트 입력 방식 혼란 ("단어 치면 되는 줄") |

### 11.3 Evaluation Side (평가)

| 단계 | 분석 |
|------|------|
| Perceive (인지) | 결과 화면 확인 가능 → OK |
| Interpret (해석) | 엠블럼이 "촌스럽다" → 기대와 결과의 괴리. "끝이네 하고 나갔을 것 같아" → 완료 상태를 체험 종료로 해석 |
| Compare (비교) | 기대(세련된 아이돌 그룹 엠블럼) vs 결과(촌스러운 AI 생성물) → 괴리 |

### 11.4 Gulf of Execution (실행의 간극)
- 포지션 설정 화면에서 능력치(육각형) 정보가 보이지 않아 합리적 판단 불가
- 엠블럼 프롬프트 입력이 단어인지 문장인지 불명확
- 포지션 중복 배정 불가 제약이 사전에 안내되지 않음

### 11.5 Gulf of Evaluation (평가의 간극)
- 엠블럼 AI 생성 결과물이 "촌스럽다"고 느끼나 이를 개선할 명확한 방법을 모름
- 캐스팅 완료 후 "끝이네" → 서비스가 제공하는 후속 가치를 인지하지 못함

## 12. Norman's 3 Levels of Design

### 12.1 Visceral Level (본능적 반응)

- **긍정**: 첫 화면 "잘생겨서 다행이네요" → 첫 연습생 비주얼에 안도
- **부정**: "당황스러운 비주얼도 몇 명이 있어가지고" → 아이돌 기준 미달 프리셋에 충격
- **부정**: 엠블럼 "촌스러워요" → AI 생성물 품질 불만
- **시사점**: 첫 노출 연습생의 비주얼이 Visceral Level의 게이트키퍼. "잘생겨서 다행"은 곧 "못생겼으면 바로 나갔을 것"을 함의.

### 12.2 Behavioral Level (행동적 반응)

- **긍정**: 캐스팅 플로우 자체는 이해 가능 ("연습생 찾기" → "캐스팅" → "캐스팅 보드")
- **불만**: 포지션 배정 시 능력치 참조 불가 → "왜 뽑았는지 모르겠네" → 합리적 판단 방해
- **불만**: 엠블럼 프롬프트 입력 → "단어를 치면 되는 건 줄 알아서" → 인터랙션 모델 혼란
- **불만**: 포지션 중복 불가 → 예상치 못한 제약
- **시사점**: 캐스팅 핵심 플로우는 OK이나, 부가 태스크(포지션, 엠블럼)에서 마찰 발생. 능력치 정보 동시 노출이 최우선 UX 과제.

### 12.3 Reflective Level (반성적 반응)

- **긍정**: "국민 프로듀스 그런 느낌" → 서비스를 통한 역할 놀이로 의미 부여
- **강한 긍정**: "내 새끼는 내가 까도 내가 까지" → 경쟁 맥락에서 깊은 감정적 연결
- **부정**: "캐스팅을 해봤구나 끝" → 체험 후 지속적 가치를 느끼지 못함
- **부정**: "퍼즐을 끼워 맞춘 느낌" → 선택의 주체성 결여에 대한 메타 인지
- **시사점**: Reflective Level에서 핵심 갈등이 드러남. 캐스팅 행위 자체는 재밌지만(국프 비유), 선택지 부족으로 진정한 주체적 선택이 아니었다는 자각이 애착 형성을 차단. 그러나 경쟁/소유 시나리오에서는 잠재된 감정이 폭발 → 소셜 기능이 Reflective Level의 킥.

## 13. Hypothesis Validation

### H1: 유저는 캐스팅만으로도 애착을 느낀다

| 판정 | 조건부 반대 |
|------|-----------|
| 근거 - 지지 | 고민 점수 4/5, 아쉬움 3/5, "내가 고른 애들이었으니까" → 최소한의 소유감 존재 |
| 근거 - 반대 | "퍼즐을 끼워 맞춘 느낌" → 진정한 선택이 아님. "캐스팅을 해봤구나 끝" → 일회성 체험. 이메일 거부. "사라져도 상관없다". |
| 조건 | **선택지가 충분히 풍부하고 매력적일 때만** 캐스팅으로 애착이 발생할 가능성. 현재 수준에서는 애착 미형성. 다만 경쟁/소유 맥락이 추가되면 잠재적 애착이 활성화될 수 있음. |

| 판정 | 혼재 (과정 의미 있으나, 조건 미충족 시 무의미) |

## 15. Cross-Analysis Summary

### 15.1 핵심 인사이트 5선

**1. 선택지 부족은 "불편함"이 아니라 "애착 차단기"**
- "선택권이 많았으면 애착이 갔을 수도 있는데 지금은 퍼즐을 끼워 맞춘 느낌" → 애착 미형성의 직접적 원인을 유저가 스스로 진단
- U1U2와 공통: 선택지 부족 불만. 그러나 U5는 더 명시적으로 "선택지 부족 → 애착 미형성"의 인과관계를 언급
- 액션: 매력적인 프리셋 풀 확대. 혹은 선점 시스템을 조정하여 접속 시점과 무관하게 충분한 선택지 보장

**2. 캐스팅 완료 = 체험 종료. 리텐션 고리가 없다**
- "캐스팅을 해봤구나 끝" → 현재 서비스는 일회성 체험에 그침
- 이메일은 채널 자체에 대한 거부감 + 후속 콘텐츠에 대한 기대 부재로 이중 실패
- 액션: 캐스팅 직후 "다음에 무엇이 일어나는지"를 즉각적으로 보여주는 온보딩 필요. 이메일이 아닌 즉시 접근 가능한 채널(앱 푸시, 인앱 알림)로 전환 검토

**3. "키우기"가 캐스팅보다 강력한 동기**
- 처음부터 만들기 < 배정받은 그룹의 멤버 교체/능력치 향상/비주얼 발전 관찰
- "진짜 그건 경쟁인 거잖아. 걔네들끼리의" → 멤버 간 내부 경쟁 관찰이 핵심 재미
- U1U2의 "데뷔 전환점" 인사이트와 연결: 연습생 → 데뷔 성장 과정 자체가 콘텐츠
- 액션: 육성/트레이닝 시스템 설계. 캐스팅을 진입점으로, 육성을 리텐션 엔진으로 분리

**4. 소유 심리는 "만들기"가 아닌 "경쟁"이 트리거**
- "사라져도 상관없다" (직접 질문) vs "내 새끼는 내가 까도 내가 까지 남이 갈 수는 없는 거야" (경쟁 시나리오)
- 동일 유저가 맥락에 따라 정반대 반응 → 소유 심리의 트리거는 타유저와의 관계
- U1U2의 "완전 아쉬워"(선점 경쟁)와 일맥상통하지만, U5는 더 구체적인 방어 행동까지 상상
- 액션: 커뮤니티/소셜 기능이 애착의 필수 인프라. 혼자 하는 캐스팅만으로는 소유 심리 활성화 불가

**5. 결정 장애 유저를 위한 "배정 + 미세 조정" 모델**
- "결정 장애를 가진 입장에서는 선택권이 너무 많아지면 힘들거든"
- "배정되어 있어도 상관없어" → 그러나 교체/발전의 선택지는 원함
- 전체를 고르는 것이 아니라 일부를 조율하는 것이 이 유저 유형의 선호
- 액션: 기본 배정 모드(디폴트 그룹 제공 + 멤버 교체 가능) 도입 검토. 캐스팅 진입 허들 낮춤.

### 15.2 Sprint 1 가설 검증 총괄 (U5 단독)

| 가설 | 판정 | 조건/한계 |
|------|------|----------|
| H1 캐스팅만으로 애착 | 조건부 반대 | 고민 4점, 아쉬움 3점이나, "퍼즐 끼워 맞추기"/"사라져도 상관없다"/"캐스팅 해봤구나 끝". 선택지 풍부 + 경쟁 맥락 필요. |
| H2 과정 > 결과 | 혼재 | 전략적 캐스팅 시도(지지) vs "배정받아도 상관없다"(약화). 정보 충분 + 선택지 풍부 시에만 과정에 의미. |
| H3 이메일 = 기대 행동 | 반대 | 명확한 거부. 이메일 채널 자체의 거부감 + 후속 콘텐츠 기대 부재. 키우기 맥락 + 즉시 채널이면 전환 가능성. |

### 15.3 U1U2 vs U5 비교 인사이트

| 항목 | U1U2 (은찬/예찬) | U5 (최지윤) |
|------|-----------------|-------------|
| 애착 수준 | 조건부 지지 (선점 경쟁에서 강한 반응) | 조건부 반대 (퍼즐 끼워 맞추기, 사라져도 OK) |
| 핵심 동기 | 경쟁/선점/바이럴 | 키우기/성장 관찰/내부 경쟁 |
| 이메일 | 미입력 (콘텐츠 매력 부족) | 거부 (채널 + 콘텐츠 모두 거부) |
| 프리셋 불만 | "쟁쟁한 애들로 20~30명" 요구 | "A등급 다 팔려나감", 퍼즐 끼워 맞추기 |
| 역할 인식 | 심즈 + 프로듀서 | 국민 프로듀스 |
| 소유 심리 트리거 | 선점 경쟁 ("완전 아쉬워") | 타유저 비교/방어 ("내 새끼") |
| 배정 수용도 | 낮음 (직접 고르고 싶어함) | 높음 ("배정받아도 상관없다") |
| 결정 부담 | 낮음 (적극적 선택) | 높음 (결정 장애 자칭) |
