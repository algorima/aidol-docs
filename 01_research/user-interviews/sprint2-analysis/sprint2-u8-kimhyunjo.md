## 3. 행동 기반 관찰

### 3.1 멤버 생성 (프로필 이미지)

- 신규 멤버 3명을 직접 생성함
- 프롬프트에 "증명 사진"을 입력하자 퀄리티가 개선됨을 확인
- 원하는 이미지가 나올 때까지 재생성을 반복하려는 의향을 보임: "네 맞습니다. 사실 뭔가 계속 해봤을 것 같긴 해요"
- 뒤로 가기 시 생성된 이미지가 초기화되는 문제를 직접 경험: "뒤로 가니까 다시 생성을 해야 되는 것 같네요"
- 홈 탭의 기존 연습생 사진(템플릿 기반)이 자신이 직접 생성한 것보다 더 퀄리티가 좋다고 느낌

### 3.2 MBTI/포지션 설정

- MBTI 조합을 신중하게 고민하며 설정: ENFJ(리더) → INFP(케미 좋은 조합) → 마지막 멤버는 N 계열로 통일
- 포지션은 골고루 배치, 국적 기반으로 포지션을 연결 (일본인 멤버 → 메인 댄서)
- 얼굴 이미지 보고 포지션을 결정하는 행동 관찰됨: "다른 멤버의 그냥 얼굴상 그냥 느낌으로 봤던 것 같습니다"

### 3.3 관계성(케미) 설정

- 단방향 관계 설정 구조를 확인한 후 "별칭은 양방향 느낌"이라는 인식 불일치를 지적
- 관계 그래프 UI를 자발적으로 제안: "멤버가 3명이면 동그라미에서 화살표로 왔다 갔다 하는 그런 UI"
- 현재 리스트 형태에서는 어떤 멤버가 됐는지 왔다 갔다 보게 된다고 지적
- 멤버 수가 많아지면 더 혼란스러울 것이라 예상

### 3.4 콘텐츠/숏폼 반응

- 채팅 형태 콘텐츠: "실제 대화 느낌... 웹 소설 많이 읽는 사람한테 맞는 콘텐츠" (본인은 해당 안 됨)
- 다른 그룹 콘텐츠: "관심 이전 단계 좀 알아보는 정도로 한번 들어가 보는 정도"
- 그룹 한 줄 소개가 필요하다고 제안: "그룹명 밑에 한 줄 소개라든지... 이 그룹이 어떤 그룹인지 표현할 수 있는 문구"
- 숏폼 영상에 긍정적 반응: "확실히 움직이는 숏폼 있는 게 좀 재밌을 것 같은 생각이 들어요"
- 숏폼 영상의 자연스러움에 놀람: "진짜 자연스럽다... AI인지 아닌지 좀 더 봐야지 좀 알 것 같은 느낌"

## 4. 가설 검증 결과

### 가설 1: 사용자가 직접 아이돌 멤버를 생성하는 과정에서 몰입감을 느낀다

**결과: 약함**

- MBTI/포지션 설정 시 몰입감 있게 조합을 고민함 (근거: ENFJ-INFP 케미, 포지션 골고루 배치)
- 그러나 프로필 이미지 생성에서 프롬프트 정밀도 부족으로 만족감이 저해됨
- 직접 발화: "재미있긴 한데 뭔가 계속 잘 안 맞는다는 느낌은 좀 드는 것 같긴 해요"
- 기존 캐스팅 보드(템플릿 기반 이미지)가 더 매력적이었다고 평가

### 가설 2: 멤버 간 관계성(케미) 설정이 그룹에 대한 애착을 높인다

**결과: 모호**

- 관계성 기능 자체에 대한 관심은 있었으나, "별칭=양방향" vs "실제 설정=단방향"의 인식 불일치 발생
- 관계성을 설정하면서 애착이 높아졌다는 직접적 행동/발화 근거 부족
- 오히려 UI/UX 혼란이 경험을 방해: "어떤 멤버가 됐는지 좀 왔다 갔다 하면서 보게 되는 것 같아요"

### 가설 3: 콘텐츠(하이라이트/숏폼)가 다른 그룹에 대한 관심을 유도한다

**결과: 약함 (숏폼은 있음)**

- 채팅 형태 콘텐츠: 웹소설 비독자인 본인에게는 약간 어색 (근거: "이제 웹 소설을 안 읽어가지고 살짝 조금 어색하다는 느낌")
- 다른 그룹 관심: "관심 이전 단계 좀 알아보는 정도" → 강한 관심 전환 근거 부족
- 숏폼 영상: "확실히 움직이는 숏폼 있는 게 좀 재밌을 것 같은 생각" → 텍스트 콘텐츠보다 효과적

## 6. 다음 스프린트 수정 제안 (3개 이하)

1. **프롬프트 템플릿 도입**: 이미지 생성 시 "증명 사진 스타일", "연습생 프로필" 등 프리셋을 제공하여 프롬프트 진입 장벽 낮추기. 사용자가 직접 프롬프트를 작성할 때 기대와 결과의 괴리가 몰입감을 저해하는 것이 반복 관찰됨 (Sprint 1에서도 유사 피드백).

2. **케미 설정 UI 개선**: 별칭(양방향 인식)과 관계 서사(단방향)를 분리하여 입력 유도. 멤버 수 증가 시 관계 파악이 어려우므로 그래프/다이어그램 형태 UI 검토.

3. **숏폼 영상 콘텐츠 우선 개발**: 텍스트 기반 하이라이트 콘텐츠보다 숏폼 영상이 비독자 세그먼트에서도 관심을 끌었음. 숏폼을 홈 피드의 메인 콘텐츠 포맷으로 우선 검토.

## 7. 남은 질문 / 리스크

- **타겟 세그먼트 재검토 필요**: "케이팝 팬덤은 실제 사람의 팬덤이라 경향이 다르다"는 지적. 버추얼 아이돌/2D 창작물 팬덤을 별도 세그먼트로 인터뷰하여 검증 필요.
- **뒤로 가기 시 이미지 초기화 버그**: 재생성 시 기존 결과가 날아가는 UX 문제 확인됨. 기능 구조 문제인지 구현 버그인지 개발팀 확인 필요.
- **"꾸미기 vs 수익화" 방향성 결정**: 사용자가 꾸미기의 한계를 느낄 것이라 예측하며 수익화 모델에 더 관심을 보임. 이 방향이 다른 세그먼트에서도 동일한지 추가 검증 필요.
- **AI 인플루언서/광고 모델에 대한 여론 리스크**: "커뮤니티에서 AI 인플루언서에 대한 의견 대립이 분명히 있을 수도 있다"는 지적. 광고 상품 카테고리 선정 시 고관여 제품(화장품 등) 회피 필요.
