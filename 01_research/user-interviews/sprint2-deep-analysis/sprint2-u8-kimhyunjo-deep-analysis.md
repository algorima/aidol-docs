### 3.2 핵심 행동 패턴

1. **MBTI를 전략적 도구로 활용**: 리더십/케미/분위기를 고려한 의도적 조합 (단순 설정이 아닌 설계 행위)
2. **반복 피드백의 일관성**: 프롬프트 예시 필요 (Sprint 1에서도 동일 언급 → 미해결 Pain)
3. **뒤로 가기 = 데이터 유실**: 두 번 발생 (08:15, 16:25) → UX 구조적 문제
4. **관계성 설정에서 능동적 설계 제안**: 단순 피드백이 아닌 구조적 대안 제시 (별칭/서사 분리, 그래프 UI)
5. **직접 생성보다 캐스팅 보드 선호**: 프롬프트 품질 한계로 인해 기존 프리셋의 완성도를 더 높이 평가
6. **비즈니스 관점의 평가**: 팬덤, 바이럴, 수익화까지 스스로 논의 확장

- **답답함**: 프롬프트가 의도대로 작동하지 않을 때, 뒤로 가기 데이터 유실 시
- **놀라움**: 캐스팅 보드의 리얼리티, 쇼폼 영상의 자연스러움
- **분석적 관심**: 팬덤 구조, 바이럴 가능성, 수익화 모델에 대한 비즈니스적 호기심
- **아쉬움**: 직접 생성의 품질이 기대에 미치지 못한 것

## 6. Thematic Analysis

### Theme 1: MBTI = 관계성 설계 도구 (세계관 개입의 핵심 수단)

- 관련 코드: `MBTI-전략적구성`, `관계케미-설계의도`, `리더십밸런스`, `MBTI-다양성재미`, `그룹케미-분위기설계`
- 근거: VC-04, VC-05
- 요약: 김현조는 MBTI를 단순 성격 태그가 아닌 그룹 내 관계 구조를 설계하는 전략적 도구로 활용. ENFJ(리더) + INFP(소심/귀여움) + N유형 통일(분위기 맞춤)이라는 의도적 조합. 이는 "세계관 수정이 제작 행위인가?"라는 Sprint 2 핵심 질문에 대해 **MBTI 설정 자체가 이미 제작 행위의 초기 형태**임을 시사.

### Theme 2: 관계성 설정의 구조적 설계 (양방향 별칭 + 단방향 서사)

- 관련 코드: `관계성-양방향기대`, `별칭-양방향본질`, `서사입력-단방향보완`, `UX-구조제안`, `단방향-이질감`
- 근거: VC-07, VC-08
- 요약: 유닛명/별칭은 본질적으로 양방향인데 단방향으로 설정하게 하면 이질감 발생. "별칭은 양방향으로 두고, 하단에 서사를 단방향으로 추가 입력"하는 구조를 직접 설계 제안. 이는 단순 UI 피드백이 아니라 **정보 구조(IA) 레벨의 설계 행위**.

### Theme 3: 프롬프트 품질의 한계 → 캐스팅 보드 선호

- 관련 코드: `프롬프트-예시필요`, `직접생성-프롬프트불일치`, `캐스팅보드-선호`, `캐스팅보드-리얼리티`
- 근거: VC-03, VC-06, VC-14, VC-19
- 요약: 프롬프트 기반 직접 생성은 "재미있긴 한데 잘 안 맞는다"는 반복적 좌절. 반면 큐레이션된 캐스팅 보드의 연습생 사진은 "진짜 리얼리티스러워서 놀랐다". 프롬프트 만족 시 높은 만족감 가능하나, 현재 수준으로는 캐스팅 보드가 더 강력한 경험.

### Theme 4: 콘텐츠 형태별 몰입 차이 (텍스트 < 이미지 < 쇼폼 영상)

- 관련 코드: `채팅스토리-실제감`, `쇼폼-자연스러움`, `동영상-정적콘텐츠대비우위`, `쇼폼-생동감`
- 근거: VC-15, VC-16, VC-20
- 요약: 채팅형 텍스트 콘텐츠는 "재밌지만 웹소설 독자 한정", 쇼폼 영상은 "진짜 자연스럽다" + "생동감 넘친다"로 감정 최고점. 콘텐츠 형태에 따라 몰입 수준이 명확하게 차이남.

### Theme 5: 타겟 세그먼트 재정의 (K-POP 팬 → 버추얼/2D 창작물 팬)

- 관련 코드: `타겟세그먼트-재정의`, `버추얼팬덤-제안`, `2D창작팬덤`, `케이팝vs가상-차이`
- 근거: VC-13
- 요약: "케이팝 팬덤은 실제 사람의 팬덤이라 경향이 다를 것", "2D 창작물 팬덤 분들이 더 인사이트를 주실 수 있을 것". 가상 아이돌 서비스의 핵심 사용자가 기존 K-POP 팬이 아닌 버추얼/창작물 팬일 수 있다는 전략적 인사이트.

### Theme 6: 비즈니스 모델 - 꾸미기 과금의 한계와 크리에이터 수익화 가능성

- 관련 코드: `꾸미기-한계인식`, `수익화모델-선호`, `심즈수준-미달`, `크리에이터경제-관심`
- 근거: VC-18
- 요약: 꾸미기(과금) 모델은 "심즈처럼 언리얼 엔진 수준이 아닌 이상 한계". 오히려 사용자가 만든 아이돌로 수익을 창출하는 크리에이터 경제 모델에 관심. 이는 과금 vs 수익 공유 모델의 전략적 분기점.

## 10. Mental Model Gap Analysis

### Gap 1: 별칭 = 양방향 vs 시스템 = 단방향

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 관계성을 A→B 단방향으로 설정 | 별칭/유닛명은 본질적으로 양방향 ("안영주" = 장원영+안유진 공유) |
| 한 화면에서 A→B 관계를 입력 | 별칭(양방향)과 개별 감정/서사(단방향)는 다른 정보 |

**영향**: "유닛명은 서로 상호 관계 양방향이라서 느낌이 좀 다를 수 있을 것 같다" → 현재 단방향 구조가 K-POP 팬덤의 관계성 문법과 불일치. 별칭/서사를 분리하지 않으면 지속적 혼란 발생.

### Gap 2: 직접 생성 = 창의적 자유 vs 프롬프트 = 통제 불가

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 프롬프트로 원하는 이미지 생성 (창의적 자유) | 프롬프트가 의도대로 작동하지 않아 "원하는 캐릭터"를 못 만듦 |
| 직접 생성이 캐스팅보다 더 높은 만족감 | 현재로서는 캐스팅 보드(큐레이션)가 더 낫다 |

**영향**: 직접 생성 기능의 가치 명제(value proposition)가 "내가 원하는 대로 만든다"인데, 프롬프트 품질이 이를 뒷받침하지 못함. 프롬프트 템플릿/예시가 없으면 캐스팅 보드 대비 열위.

### Gap 3: 관계성 시각화 = 리스트 vs 그래프

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 멤버별 리스트 형태로 관계 설정 | 관계는 "동그라미에서 화살표"로 한눈에 봐야 이해됨 |
| 멤버 3명이면 3개 항목 | 멤버 수 증가 시 리스트로는 관계 파악 불가능 |

**영향**: 현재 리스트 UI는 3명에서도 "왔다 갔다 하면서 보게 된다"는 불만. 25명까지 가능한 시스템에서 리스트 UI는 확장 불가.

### Gap 4: 서비스 = 소비 플랫폼 vs 사용자 = 수익 창출 기대

| 서비스 모델 | 사용자 멘탈 모델 |
|------------|----------------|
| 사용자가 과금하여 아이돌을 꾸미는 모델 | 심즈 수준 아닌 이상 꾸미기에 한계. 오히려 수익을 나눠 받고 싶음 |
| 유저 = 소비자 | 유저 = 크리에이터 (내 아이돌이 돈을 벌면 나도 수익) |

**영향**: 과금 중심 모델만으로는 "전략적 설계자" 세그먼트의 장기 retention 확보 어려움. 크리에이터 수익 공유 모델이 차별화 동력.

## 11. Norman's 7 Stages of Action Analysis

### 11.1 Goal (목표)
- **유저 목표**: 전략적으로 설계한 MBTI/관계성을 가진 나만의 아이돌 그룹을 만들고, 생동감 있는 콘텐츠로 확인하기
- **시스템 목표**: 신규 멤버 생성 → MBTI/포지션 설정 → 관계성 설정 → 콘텐츠 소비

### 11.2 Execution Side (실행)

| 단계 | 분석 |
|------|------|
| Plan (계획) | MBTI 조합을 전략적으로 구성하겠다 (ENFJ 리더 + INFP 소심 + N유형 통일) → 명확한 의도 |
| Specify (구체화) | 이미지 프롬프트에 "증명 사진" 입력, MBTI 선택, 관계성 별칭 입력 → 구체화 가능하나 프롬프트 가이드 부재 |
| Perform (수행) | 이미지 생성 → 배경 과다 / 뒤로 가기 → 데이터 유실 / 관계성 설정 → 단방향 혼란. **문제**: 실행 과정에서 반복적 마찰 |

### 11.3 Evaluation Side (평가)

| 단계 | 분석 |
|------|------|
| Perceive (인지) | 생성된 이미지, 관계성 설정 결과, 캐스팅 보드, 쇼폼 영상 → 인지 가능 |
| Interpret (해석) | 이미지: "배경이 너무 많다" → 프로필 사진이 아님. 관계성: "유닛명은 양방향인데" → 구조적 불일치 인지 |
| Compare (비교) | 직접 생성 vs 캐스팅 보드 → 캐스팅 보드 우위. 텍스트 콘텐츠 vs 쇼폼 영상 → 쇼폼 압도적 |

### 11.4 Gulf of Execution (실행의 간극)
- 프롬프트 예시/가이드 부재 → 원하는 이미지 출력까지의 시행착오 과다
- 뒤로 가기 시 데이터 유실 → 재작업 강제
- 관계성 설정의 단방향/양방향 혼재 → 입력 구조와 멘탈 모델 불일치

### 11.5 Gulf of Evaluation (평가의 간극)
- 이미지 생성 결과가 "프로필 사진"으로 인식되지 않음 (배경 과다)
- 관계성 설정 결과가 한눈에 파악되지 않음 (리스트 UI의 한계)
- MBTI 조합의 효과가 콘텐츠에 어떻게 반영되는지 확인 불가 (피드백 루프 부재)

## 12. Norman's 3 Levels of Design

### 12.1 Visceral Level (본능적 반응)

- **강한 긍정**: 캐스팅 보드의 연습생 사진 → "진짜 리얼리티스러워서 놀랐다" (1:02:33)
- **강한 긍정**: 쇼폼 영상 → "진짜 자연스럽다", "생동감 넘치고" (54:03, 1:03:02)
- **부정**: 이미지 생성 결과 → "배경도 너무 많이 보여" (05:02)
- **부정**: 로고 생성 → "어울리지 않는 일러스트 형태" (24:21)
- **시사점**: Visceral 레벨에서 큐레이션된 콘텐츠(캐스팅 보드, 쇼폼)는 성공하고 있으나, 사용자 생성 콘텐츠(프롬프트 이미지, 로고)는 실패. 생성 품질 개선 또는 큐레이션 강화 필요.

### 12.2 Behavioral Level (행동적 반응)

- **긍정**: MBTI 조합의 전략적 설정 → 몰입/재미 (13:27~14:00)
- **불만**: 프롬프트 예시 없음 → Sprint 1부터 반복 피드백
- **불만**: 뒤로 가기 → 데이터 유실 (08:15, 16:25)
- **불만**: 관계성 설정 → 단방향/양방향 혼란, 리스트 UI 비효율
- **시사점**: MBTI/관계성 설정이라는 핵심 태스크의 개념적 재미는 있으나, 실행 UX에서 마찰이 반복. 특히 프롬프트와 관계성 UI가 핵심 병목.

### 12.3 Reflective Level (반성적 반응)

- **강한 긍정**: 타겟 세그먼트 재정의 제안 → 서비스의 본질을 깊이 성찰 (47:04~47:43)
- **강한 긍정**: 크리에이터 수익화 모델 선호 → 서비스의 장기적 가치에 대한 비전 (1:00:04)
- **분석적**: 바이럴 가능성 + AI 논쟁 리스크 균형 잡힌 인식 (55:08)
- **분석적**: "팬덤이 붙을지 여부가 되게 중요한 프로젝트" (46:50)
- **시사점**: Reflective 레벨에서 가장 깊이 있는 반응. 서비스의 전략적 방향에 대한 통찰이 풍부. "전략적 설계자" 유형은 서비스의 방향성 자체에 관심을 가지며, 이 세그먼트의 피드백이 전략적 의사결정에 높은 가치.

## 15. Cross-Analysis Summary

### 15.1 핵심 인사이트 5선

**1. 세계관 수정은 "전략적 설계자" 유형에서 명확한 제작 행위로 인식된다 (H7 강한 지지)**

김현조는 MBTI를 리더십/케미/분위기 관점에서 의도적으로 조합하고, 관계성 설정의 정보 구조(별칭 양방향 + 서사 단방향 분리)를 직접 설계하며, UI 확장성(그래프 형태)까지 제안했다. 이는 세계관 수정이 단순 설정 편집이 아니라 **"내 아이돌 그룹의 관계 구조를 설계하는 창작 행위"**로 인식되고 있음을 보여준다. 다만 PM/기획 경험자라는 특수성이 있어, 일반 팬 유형에서의 재검증이 필요하다.

**2. 프롬프트 품질이 직접 생성의 가치를 결정한다 (캐스팅 보드 vs 직접 생성의 분기점)**

프롬프트가 의도대로 작동하지 않으면 사용자는 직접 생성을 포기하고 큐레이션(캐스팅 보드)을 선호한다. "오히려 홈탭에 있는 기존 연습생들 사진이 더 뽑고 싶은 마음이 더 들었다"(VC-14)는 현재 프롬프트 품질이 직접 생성의 가치 명제를 훼손하고 있음을 의미. 프롬프트 템플릿(채현 님 작업 중)의 도입이 이 Gap을 해소할 핵심 액션이다.

**3. 관계성 설정 UI는 "별칭(양방향) + 서사(단방향) 분리" 및 "그래프 시각화"로 재설계 필요**

현재 리스트 기반 단방향 설정은 K-POP 팬덤의 유닛명 문화(양방향)와 불일치하며, 멤버 3명에서도 "왔다 갔다" 비효율이 발생한다. 별칭은 두 멤버가 공유하는 양방향 이름으로, 서사/감정은 A→B 단방향으로 분리하는 2-layer 구조와, 노드/엣지 기반 그래프 UI가 사용자 멘탈 모델에 부합한다.

**4. 쇼폼 영상이 현재 서비스에서 가장 강력한 감정 반응을 유도한다**

감정 최고점(5점)이 모두 쇼폼 영상 관련 발화에서 발생. "진짜 자연스럽다"(54:03), "생동감 넘치고"(1:03:02). 채팅형 텍스트 콘텐츠는 "웹소설 독자 한정"이라는 한계가 있지만, 쇼폼 영상은 세그먼트를 초월하는 범용적 매력. 바이럴 채널(틱톡)과의 시너지도 높다.

**5. 핵심 타겟 세그먼트 재검토: K-POP 팬 vs 버추얼/2D 창작물 팬**

"케이팝 팬덤은 실제 사람의 팬덤이라 경향이 다를 것"(VC-13)이라는 지적은 전략적으로 중요한 인사이트. 가상 아이돌 서비스의 핵심 사용자가 기존 K-POP 팬(실제 인간 아이돌 팬)이 아니라, 버추얼/2D 창작물(웹툰, 포스타입 등) 팬일 수 있다. 다음 인터뷰에서 해당 세그먼트 포함을 검토해야 한다.
