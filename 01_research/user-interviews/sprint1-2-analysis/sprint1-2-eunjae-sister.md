# SPRINT 1-2 - UT Sprint 요약: 은재님 언니

> Sprint 1(캐스팅/그룹 생성) + Sprint 2(관계성 설정, 하이라이트 콘텐츠)까지 포함하여 테스트 진행.
> 은재님이 직접 UT를 진행하고, 소연님과 피드백 미팅에서 결과를 공유한 녹취록 기반 분석.

## 피실험자 정보

- K-POP 팬 / AI 채팅 경험 적음 / ChatGPT, Gemini 등 생성형 AI 도구 사용 경험 있음 / 이미지 생성 AI 경험 많음 (프롬프트 작성 숙련)
- MBTI 과몰입 성향 / 실제 아이돌 자체 콘텐츠는 잘 안 보는 타입
- 세그먼트: **B그룹(창작 성향) + K-POP 팬 + AI 이미지 생성 숙련자**

## 0. 한 줄 핵심

프리셋 캐스팅보다 신규 생성에 강한 창작 욕구를 보였고, 이미지 생성 결과물에서 몰입이 급격히 상승했으나, 관계성/콘텐츠의 필요성은 설명 없이는 자발적으로 인지하지 못함

## 1. 태깅된 주요 발화

### [AHA] 긍정 반응/몰입

> "처음에 언니가 기대를 안 하고 있다가 이제 결과물이 딱 나오니까 엄청 집중도 확 올라가면서 이렇게까지 잘 나올 줄 몰랐다고 본인이 상상하던 그대로 나왔다고" (17:28)

> "누가 갑자기 자세도 고쳐 앉으면서 이렇게 다음 거를 엄청 열심히 상상을 했는데" (17:28)

> "굉장히 마음에 들어 하더라고" — 최종 결과물 반응 (11:36)

> "결국엔 끝까지 다 만들더라고요 모든 관계성을" — 처음에 거부하다가 5명 전원 관계성 완성 (24:12)

### [PAIN] 불만/불편

> "이미지를 재생성하면 이제 기존 이미지가 없어져 버리니까 이것보다 좋은 결과가 나올지 아님 더 악화가 될지 고민이 돼가지고 이걸 누를지 말지도 되게 고민을 이렇게 많이 망설여지는 것 같았어요" (18:13)

> "이미지 생성이 근데 조금 느려서 아쉬웠다" (17:28)

> "그룹 이름 정할 때가 이제 엄청 너무 고민이 많이 되는데 나중에 수정도 안 될 것 같아서 이때 약간 포기할까 조금 생각을 했었던 것 같아요" (19:27)

> "너무 많은 그룹을 보기보다는 적은 그룹 안에서 콘텐츠를 많이 접하면서 더 친숙해지는 게 좋을 것 같다" (30:48)

> "사진이 있는데 잘 안 보여서 이제 이 멤버들도 잘 모르는데 얼굴이 너무 작게 보이니까 구분이 더 잘 안된다" (30:48)

### [NEED] 기능 요청

> "여성 남성을 이렇게 두 개로 구분하기보다는 그 외 상관없음 아니면 정하지 않음 이런 식으로 추가가 되면 어떨까" (16:27)

> "AI라서 오히려 이런 게 또 좀 더 자유롭고 도전적인 걸 만들고 싶은 사람이 있을 수 있겠다" (16:27)

> "나중에 스타일링도 자유롭게 바꿀 수 있으면 좋겠다" — 얼굴은 OK, 의상 커스텀 원함 (18:16)

> "라이벌이면서도 친한 약간 이런 것도 원하더라고요" / "동경하지만 약간 어렵게 생각하는 이런 식으로" — 관계성 복합 유형 요청 (25:45, 25:58)

> "본인이 만든 그룹의 콘텐츠는 언제 볼 수 있느냐" — 내 그룹 콘텐츠 우선 (28:42)

### [COMP] 경쟁사 비교

> "플레이브도 이런 글 같은 콘텐츠를 즐길 수 있을 것 같다" — 플레이브 기준으로 콘텐츠 품질 비교 (29:19)

### [BEHAV] 행동 관찰

> 프리셋 연습생 목록: "한 3초" 보고 바로 신규 생성으로 넘어감 (09:17)

> 5명 전원 신규 생성으로 캐스팅 (08:38)

> MBTI 슬라이더 퍼센트 단위까지 신중하게 설정 (09:56)

> 멤버 얼굴을 미리 상상하고 성격/팀 역할 고려하며 MBTI 설정 (10:17)

> 이미지 생성 시 구체적 프롬프트 ("아랍상에 눈썹이 진하고 턱이 있는 편이고 쌍이지만 눈이 큰 편이고 포마드 헤어에 20대 남성 한국 남자 아이돌") (16:54)

> 엠블럼도 여러 번 재생성 (20:27)

> 관계성: "얘네의 관계성을 다 만들어야 돼?" → 설명 후 → 5명 전원 완성 (24:12)

> 완성 화면에서 URL 공유만 요청, 이메일 미입력, 새 그룹 미생성 (20:54)

## 3. 가설 검증 결과

**가설**: 캐스팅 + 관계성 설정만으로 사용자가 AI 아이돌 그룹에 애착을 형성하고, 콘텐츠 소비 및 결제 의향으로 이어지는가

**판정: 부분 검증 (창작 몰입 강함, 애착 전 단계 확인, 콘텐츠 소비 연결은 조건부)**

| 축 | 판정 | 근거 발화/행동 |
|----|------|--------------|
| 캐스팅 몰입 | 있음 | 5명 전원 신규 생성, MBTI 퍼센트 단위 설정, 이미지 프롬프트 구체적 작성, 자세 고쳐앉음 |
| 애착 형성 | 약함~있음 | 자세한 서사 작성, 결과물 만족, 내 그룹 콘텐츠 기대. 단, 다른 사용자에 뺏기는 것에 대한 발화 없음 |
| 관계성 자발적 동기 | 없음 | "얘네의 관계성을 다 만들어야 돼?" — 설명 전에는 필요성 인지 못함 |
| 콘텐츠 소비 | 조건부 | 타 그룹 텍스트 콘텐츠 관심 없음. 내 그룹 콘텐츠만 기대. 글보다 비주얼 선호 |
| 결제 의향 | 미검증 | 결제 관련 질문/발화 없음 |

## 4. 기능 수정 제안

| 현재 | 제안 | 유형 | 근거 발화 |
|------|------|------|----------|
| 이미지 재생성 시 기존 이미지 유실 | 이전 이미지 히스토리 유지 또는 비교 선택 | 기능 구조 | "재생성하면 기존 이미지가 없어져 버리니까 누를지 말지 망설여지는 것 같았어요" |
| 성별 선택: 남성/여성 2택 | "상관없음" / "정하지 않음" 옵션 추가 | 기능 구조 | "AI라서 오히려 자유롭고 도전적인 걸 만들고 싶은 사람이 있을 수 있겠다" |
| 그룹명 수정 불가 (인지) | 수정 가능하게 하거나, 수정 가능함을 명시 | UX | "수정도 안 될 것 같아서 포기할까 생각" — 이탈 위험 |
| 관계성 설정 진입 시 목적 불명확 | 진입 화면에서 "이 관계성이 채팅/콘텐츠에 반영됩니다" 안내 | UX | "이게 내가 즐길 수 있을까" → 설명 후에야 동기 부여 |
| 관계성 유형: 8가지 단순 분류 | 복합 관계성 추가 (라이벌+친함, 동경+어려움) | 기능 구조 | "라이벌이면서도 친한", "동경하지만 어렵게 생각하는" |
| 채팅 뷰에서 멤버 프로필 사진 작음 | 프로필 사진 크기 확대 | UX | "얼굴이 너무 작게 보이니까 구분이 안된다" |
| 타 그룹 하이라이트: 텍스트 중심 | 비주얼/프로필 중심으로 변경 (인스타 피드형) | 기능 구조 | "글보다 어떤 멤버가 있는지가 궁금하고" |
| 스타일링 고정 | 의상/스타일 커스텀 옵션 | 기능 구조 (P2) | "스타일링도 자유롭게 바꿀 수 있으면 좋겠다" |

## 5. 다음 스프린트 제안 (3개 이하)

1. **관계성 진입 UX 개선** — "왜 해야 하는지"를 화면 내에서 설명 (채팅/콘텐츠 반영 안내). 설명 없이는 이탈 위험
2. **이미지 재생성 히스토리** — 기존 이미지 유지 + 비교 선택 기능. 재생성 망설임 = 이탈 포인트
3. **내 그룹 콘텐츠 우선 노출** — Sprint 3 채팅 진입 시 내가 만든 그룹부터 보여주기. 타 그룹 콘텐츠는 후순위

## 6. 미검증/추가 확인 필요

- 결제 의향 미질문 — Sprint 3 UT에서 반드시 확인 필요
- 은재님이 UT를 진행하고 소연님에게 전달하는 구조 — 피실험자 직접 발화가 아닌 2차 전달이므로 뉘앙스 손실 가능성
- "내 그룹 콘텐츠를 보면 재방문하는가" — Sprint 3에서 실제 콘텐츠 제공 후 확인 필요
- 이 피실험자는 AI 이미지 생성 숙련자 — 일반 사용자는 프롬프트 작성에서 막힐 수 있음 (세그먼트 편향 주의)
