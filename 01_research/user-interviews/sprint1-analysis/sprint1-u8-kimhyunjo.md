## 3. 가설 검증 결과

**가설**: 캐스팅 + 관계성 설정만으로 사용자가 AI 아이돌 그룹에 애착을 형성하고, 콘텐츠 소비 및 결제 의향으로 이어지는가

**판정: 부분 검증 (캐스팅 몰입 O, 애착 전환 X, 결제 전환 X)**

- 캐스팅 과정에서의 몰입: 검증됨 (고민 4-5점, "열심히 찾게 된다")
- 캐스팅 결과에 대한 소유감: 약함 (3-4점, "빌링 필대로 간 것도 있었다")
- 다른 사용자에게 뺏기는 것에 대한 아쉬움: 약함 (2-3점)
- 결제 의향: 없음에 가까움 ("지금 정도면 충분하다")
- 전환 조건으로 직접 언급한 것: **육성 요소** (다마고치 비유)

## 4. 관계/육성/IP 개입 축 분석

### 4.1 캐스팅만으로 애착 형성: 약함

- 근거: 소유감 3-4점, 뺏김 아쉬움 2-3점
- "직접 트레이닝하고 좀 키우는 느낌은 조금 부족"
- 캐스팅 과정 자체에 몰입은 했으나 결과물에 대한 지속적 애착으로는 미전환

### 4.2 성장/육성 욕구 언급: 있음

- 다마고치 비유로 직접 육성 니즈 표현
- 보컬 트레이닝 -> 스탯 상승에 긍정 반응 ("좀 더 게임 그런 요소가 들어가서 괜찮을 것 같습니다")
- 심즈 경험 언급하며 성장 시스템 기대

### 4.3 IP 개입 표현: 있음

- 컨셉포토 AI 생성 제안: "AI로 뭔가 컨셉포토 이런 거 뽑으면 재밌을 것 같은 생각도 들었어요"
- 노래 제작 -> 퍼블리싱 -> 차트 진입까지 확장 상상
- 파피랜드 비유로 "저만의 영역을 꾸미는" 욕구 표현
- 다른 사용자와 경쟁/대결 언급: "경쟁 심리가 있으면은 좀 할 것 같아"

## 6. 추가 관찰: UX/기능 구조 이슈

### UX 문구 문제

- 엠블럼 프롬프트 입력창에 예시/placeholder 부재 -> 사용자가 뭘 입력해야 할지 막힘
- 관계성 설정의 "별칭" 용어가 유닛명/애칭/관계명 중 어떤 의미인지 불명확

### 기능 구조 문제

- 연습생 리스트 등급/능력치 정렬 기능 부재 -> 탐색 비용 높음
- 캐스팅 화면에서 프로필 요약 미노출 -> 하나씩 들어가서 확인해야 함
- 뒤로가기가 브라우저 뒤로가기에 의존
- 홈 탭 빈 화면에 안내 메시지 부재

## 7. 다음 스프린트 수정 제안 (3개)

1. **육성 루프 MVP 설계**: 캐스팅 후 사용자 개입 없이 콘텐츠만 수동 소비하는 현재 구조에서, 최소한의 "트레이닝" 인터랙션 추가 검토. 사용자가 직접 무언가를 해야 스탯이 오르는 구조가 애착 전환의 핵심 조건으로 반복 언급됨.

2. **캐스팅 UX 개선**: 등급/능력치 기반 정렬/필터 기능 추가. 프로필 화면에서 캐스팅 목록으로 돌아갈 때 프로필 요약 노출. 현재 "하나씩 다 들어가 봐야" 하는 탐색 비용이 높음.

3. **엠블럼 프롬프트 예시 추가**: 프롬프트 입력창에 placeholder 예시 문구 필요. 현재 사용자가 "뭘 입력해야 하는지 모름" 상태.
