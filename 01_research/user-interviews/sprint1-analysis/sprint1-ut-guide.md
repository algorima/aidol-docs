# SPRINT 1 - 캐스팅 인식 검증 UT 가이드

---

## 1. UT 설계 목적

1. 유저는 캐스팅을 단순 클릭이 아니라 **'선택 행위'로 인지하는가?**
2. 직접 생성 1명이 아니라, **캐스팅만으로도 애착이 발생하는가?**
3. 이메일 입력(1주 대기)이 기대 기반 전환으로 작동하는가?

핵심: "직접 생성"이 아니라 **캐스팅만으로 애착이 생기는지 확인**

---

## 2. 피실험자 세그먼트

### A그룹 - 팬 성향

- K-POP 팬 (10~20대 여성)
- 버추얼 아이돌 / AI 챗 경험 있음

### B그룹 - 창작 성향

- AI 이미지 생성툴 사용 경험자 (20~30대)
- 아이돌 팬은 아님

목적: 팬 몰입 vs 제작 몰입 차이 비교

---

## 3. 검증 가설 구조

- **H1:** 유저는 캐스팅만으로도 애착을 느낀다.
- **H2:** 유저는 결과물보다 선택 과정에 의미를 둔다.
- **H3:** 이메일 입력은 단순 알림이 아니라 기대 행동이다.

---

## 4. UT 구조

1. AI 연습생 캐스팅 (최소 3명)
2. 그룹 콘셉트 설정
3. 결과 카드 확인
4. 이메일 입력 선택

관찰 항목:

- 고민 시간
- 선택 변경 여부
- 특정 멤버에 대한 언급

---

## 5. 핵심 정량 지표

- 그룹 생성 완료율 %
- 평균 캐스팅 소요 시간
- 특정 멤버 선택 반복률 %
- 결과 화면 체류 시간
- 이메일 입력 전환율 %

---

## 6. 설문 문항

- 멤버를 고를 때 꽤 고민이 되었다 (1-5)
- 여러 후보 중에서 '내가 선택했다'는 생각이 들었다 (1-5)
- 완성된 그룹을 그냥 제공받았어도 괜찮았다 (1-5, 역문항)
- 이 멤버가 다른 유저에게 가면 아쉬울 것 같다 (1-5)

---

## 7. 결과 해석 시나리오

### 소비형 시그널

- 결과 만족 높음
- 과정 중요도 낮음
- 이메일 전환 낮음

### 참여형 IP 가능성

- 선택 과정 점수 4.0+
- 특정 멤버 집착 언급
- 이메일 전환 40%+
