## 0. 한 줄 핵심

온보딩/UI 구조 혼란으로 캐스팅이 '선택 행위'로 인지되지 못했고, 애착은 사실상 미형성. 단, 캐스팅 과정에서 몰입 경험

> "계약 완료 의미는 알겠는데 계약 미완료인 캐릭터를 바로 앞에서 보거나 필터 버튼이 있었으면 좋겠음"

> "연습생 캐스팅을 여러 명 해야 하는 지에 대한 설명이 없음"

> "등급에 대한 설명이 부족함"

## 3. 가설 검증 결과

**가설**: 캐스팅 과정에서 사용자가 몰입하여 연습생을 탐색하고 선택할 수 있는가

**판정: 부분 검증 (캐스팅 몰입 O, 애착 전환 X, 결제 전환 X)**

- 등급 체계 이해: 실패 (별도 설명 후 이해)
- 캐스팅만으로 애착 발생: 낮음
- 선택 과정에 의미 부여: 약함 (3점, 온보딩 부재로 선택 과정 자체에서의 인지 약화)
- 다른 사용자에게 뺏기는 것에 대한 아쉬움: 약함 (1-2점)
- 제작 인식 경험: 몰입감 느꼈으나 (4점) 발전 욕구 낮음 (2점)
- 스토리 소비 기대: (3점, 가벼운 호기심 수준)

## 4. 관계/육성/IP 개입 축 분석

### 4.1 캐스팅만으로 애착 형성: 매우 약함

- 근거: 아쉬움 1점 (전체 유저 중 최저)
- 후기 전체가 UX 개선 요청으로 구성 → 감정적 반응 자체가 부재

### 4.2 성장/육성 욕구 언급: 약함

- 그룹 발전 의향 2점

### 4.3 IP 개입 표현: 제작 인식만 존재

- 제작 인식 4점
- '내가 만들었다'는 인식은 형성됨
- 제작 인식 → 소유감·육성 욕구로의 전환이 발생하지 않은 상태
- K-pop 외국팬들이 컨텐츠 생성 요소들을 좋아할 것 같다고 표현

## 6. 추가 관찰: UX/기능 구조 이슈

### 6.1 온보딩 구조 문제

- 캐스팅 인원·목적·등급 체계에 대한 설명 부재 → 규칙 이해 실패 → 선택 행위 인지 차단
- "남은 캐스팅 완료" 문구 의미 불명확 → 사용자가 현재 진행 상태를 파악하지 못함
- U10(jhlee)에서도 동일 관찰: 등급 직관성 부족 + 계약완료 노출 문제

### 6.2 정보 구조 문제

- 계약완료(모자이크) 캐릭터가 리스트 상단에 노출 → 선택 불가 항목이 탐색 비용을 높임
- 미계약 캐릭터 우선 정렬 또는 필터 기능 부재
- U5(차유경)도 블러 리스트 상단 노출 문제를 지적한 바 있음

### 6.3 네비게이션 문제

- 캐스팅 보드에서 '다른 연습생 캐스팅'으로 바로 이동하는 동선 부재

### 6.4 기능 범위 문제

- 그룹 엠블럼 생성 시 로고 외 이미지도 출력 → 로고만 생성되도록 제한 필요

---

## 7. 다음 스프린트 수정 제안 (3개)

1. **캐스팅 온보딩 신설**: 캐스팅 진입 시 목적(그룹 구성), 인원, 등급 체계를 안내하는 최소한의 온보딩 추가. 현재 "남은 캐스팅 완료" 등 문구의 의미가 전달되지 않으며, U10에서도 동일 문제 반복 관찰됨. 선택 행위 인지의 선행 조건.

2. **계약완료 연습생 표시 개선**: 계약완료 캐릭터를 하단 분리 또는 필터링하여 선택 가능한 연습생만 우선 노출. 현재 모자이크 캐릭터가 상단에 위치하여 "왜 앞에 있는지 모르겠음"이라는 혼란 직접 발생. U5·U7·U10에서도 동일 이슈.

3. **캐스팅 보드 네비게이션 개선**: 캐스팅 보드 내 "캐스팅 하러가기" / "다른 연습생 보기" CTA 추가. 현재 캐스팅 보드에서 캐스팅 화면으로의 동선이 끊겨 있음.
